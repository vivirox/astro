Quote from \href{https://swr.vercel.app/}{\texttt{ vercel\textquotesingle{}s SWR}} for react\+:

\begin{quote}
The name “\+SWR” is derived from stale-\/while-\/revalidate, a HTTP cache invalidation strategy popularized by \href{https://tools.ietf.org/html/rfc5861}{\texttt{ HTTP RFC 5861}}. SWR is a strategy to first return the data from cache (stale), then send the fetch request (revalidate), and finally come with the up-\/to-\/date data.

With SWR, components will get a stream of data updates constantly and automatically. And the UI will be always fast and reactive. \end{quote}
