\hypertarget{Readme.md_autotoc_md21353}{}\doxysubsection{\texorpdfstring{Options}{Options}}\label{Readme.md_autotoc_md21353}
Options are defined with the {\ttfamily .option()} method, also serving as documentation for the options. Each option can have a short flag (single character) and a long name, separated by a comma or space or vertical bar (\textquotesingle{}\texorpdfstring{$\vert$}{|}\textquotesingle{}).

The parsed options can be accessed by calling {\ttfamily .opts()} on a {\ttfamily Command} object, and are passed to the action handler. (You can also use {\ttfamily .get\+Option\+Value()} and {\ttfamily .set\+Option\+Value()} to work with a single option value, and {\ttfamily .get\+Option\+Value\+Source()} and {\ttfamily .set\+Option\+Value\+With\+Source()} when it matters where the option value came from.)

Multi-\/word options such as "{}-\/-\/template-\/engine"{} are camel-\/cased, becoming {\ttfamily program.\+opts().template\+Engine} etc.

Multiple short flags may optionally be combined in a single argument following the dash\+: boolean flags, followed by a single option taking a value (possibly followed by the value). For example {\ttfamily -\/a -\/b -\/p 80} may be written as {\ttfamily -\/ab -\/p80} or even {\ttfamily -\/abp80}.

You can use {\ttfamily -\/-\/} to indicate the end of the options, and any remaining arguments will be used without being interpreted.

By default options on the command line are not positional, and can be specified before or after other arguments.\hypertarget{Readme.md_autotoc_md21354}{}\doxysubsubsection{\texorpdfstring{Common option types, boolean and value}{Common option types, boolean and value}}\label{Readme.md_autotoc_md21354}
The two most used option types are a boolean option, and an option which takes its value from the following argument (declared with angle brackets like {\ttfamily -\/-\/expect \texorpdfstring{$<$}{<}value\texorpdfstring{$>$}{>}}). Both are {\ttfamily undefined} unless specified on command line.

Configuration options can be passed with the call to {\ttfamily .command()} and {\ttfamily .add\+Command()}. Specifying {\ttfamily hidden\+: true} will remove the command from the generated help output. Specifying {\ttfamily is\+Default\+: true} will run the subcommand if no other subcommand is specified (\href{./examples/defaultCommand.js}{\texttt{ example}}).\hypertarget{Readme.md_autotoc_md21363}{}\doxysubsubsection{\texorpdfstring{Command-\/arguments}{Command-\/arguments}}\label{Readme.md_autotoc_md21363}
For subcommands, you can specify the argument syntax in the call to {\ttfamily .command()} (as shown above). This is the only method usable for subcommands implemented using a stand-\/alone executable, but for other subcommands you can instead use the following method.
