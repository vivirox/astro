\doxysubsection{Detailed Description}
\href{https://travis-ci.org/Qix-/color-convert}{\texttt{ }}

Color-\/convert is a color conversion library for Java\+Script and node. It converts all ways between {\ttfamily rgb}, {\ttfamily hsl}, {\ttfamily hsv}, {\ttfamily hwb}, {\ttfamily cmyk}, {\ttfamily ansi}, {\ttfamily ansi16}, {\ttfamily hex} strings, and CSS {\ttfamily keyword}s (will round to closest)\+:

\hypertarget{README.md_autotoc_md10695}{}\doxysubsection{\texorpdfstring{API}{API}}\label{README.md_autotoc_md10695}
Simply get the property of the {\itshape from} and {\itshape to} conversion that you\textquotesingle{}re looking for.

All functions have a rounded and unrounded variant. By default, return values are rounded. To get the unrounded (raw) results, simply tack on {\ttfamily .raw} to the function.

All \textquotesingle{}from\textquotesingle{} functions have a hidden property called {\ttfamily .channels} that indicates the number of channels the function expects (not including alpha).

\hypertarget{README.md_autotoc_md10697}{}\doxysubsubsection{\texorpdfstring{Routing}{Routing}}\label{README.md_autotoc_md10697}
Conversions that don\textquotesingle{}t have an {\itshape explicitly} defined conversion (in \href{conversions.js}{\texttt{ conversions.\+js}}), but can be converted by means of sub-\/conversions (e.\+g. XYZ -\/\texorpdfstring{$>$}{>} {\bfseries{RGB}} -\/\texorpdfstring{$>$}{>} CMYK), are automatically routed together. This allows just about any color model supported by {\ttfamily color-\/convert} to be converted to any other model, so long as a sub-\/conversion path exists. This is also true for conversions requiring more than one step in between (e.\+g. LCH -\/\texorpdfstring{$>$}{>} {\bfseries{LAB}} -\/\texorpdfstring{$>$}{>} {\bfseries{XYZ}} -\/\texorpdfstring{$>$}{>} {\bfseries{RGB}} -\/\texorpdfstring{$>$}{>} Hex).

Keep in mind that extensive conversions {\itshape may} result in a loss of precision, and exist only to be complete.
