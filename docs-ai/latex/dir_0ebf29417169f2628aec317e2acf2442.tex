This package is a \href{https://github.com/unifiedjs/unified}{\texttt{ unified}} (\href{https://github.com/remarkjs/remark}{\texttt{ remark}}) plugin to add support for math syntax. You can use this to add support for parsing and serializing this syntax extension.

As there is no spec for math in markdown, this extension follows how code (fenced and text) works in Commonmark, but uses dollars ({\ttfamily \$}).\hypertarget{readme.md_autotoc_md29158}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{readme.md_autotoc_md29158}
This project is useful when you want to support math in markdown. Extending markdown with a syntax extension makes the markdown less portable. La\+TeX equations are also quite hard. But this mechanism works well when you want authors, that have some La\+TeX experience, to be able to embed rich diagrams of math in scientific text.

If you {\itshape just} want to turn markdown into HTML (with maybe a few extensions such as math), we recommend \href{https://github.com/micromark/micromark}{\texttt{ {\ttfamily micromark}}} with \href{https://github.com/micromark/micromark-extension-math}{\texttt{ {\ttfamily micromark-\/extension-\/math}}} instead. If you don’t use plugins and want to access the syntax tree, you can use \href{https://github.com/syntax-tree/mdast-util-from-markdown}{\texttt{ {\ttfamily mdast-\/util-\/from-\/markdown}}} with \href{https://github.com/syntax-tree/mdast-util-math}{\texttt{ {\ttfamily mdast-\/util-\/math}}}.

This plugins adds \href{https://github.com/syntax-tree/mdast-util-to-hast\#fields-on-nodes}{\texttt{ fields on nodes}} so that the plugin responsible for turning markdown (mdast) into HTML (hast), \href{https://github.com/remarkjs/remark-rehype}{\texttt{ {\ttfamily remark-\/rehype}}}, will turn text math (inline) into {\ttfamily \texorpdfstring{$<$}{<}code class="{}language-\/math math-\/inline"{}\texorpdfstring{$>$}{>}…\texorpdfstring{$<$}{<}/code\texorpdfstring{$>$}{>}} and flow math (block) into {\ttfamily \texorpdfstring{$<$}{<}pre\texorpdfstring{$>$}{>}\texorpdfstring{$<$}{<}code class="{}language-\/math math-\/display"{}\texorpdfstring{$>$}{>}…\texorpdfstring{$<$}{<}/code\texorpdfstring{$>$}{>}\texorpdfstring{$<$}{<}/pre\texorpdfstring{$>$}{>}}.\hypertarget{readme.md_autotoc_md29159}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md29159}

\hypertarget{readme.md_autotoc_md29173}{}\doxysubsection{\texorpdfstring{Compatibility}{Compatibility}}\label{readme.md_autotoc_md29173}
Projects maintained by the unified collective are compatible with maintained versions of Node.\+js.

When we cut a new major release, we drop support for unmaintained versions of Node. This means we try to keep the current release line, {\ttfamily remark-\/math@\texorpdfstring{$^\wedge$}{\string^}6}, compatible with Node.\+js 16.

This plugin works with unified version 6+ and remark version 14+. The previous major (version 4) worked with remark 13.\hypertarget{readme.md_autotoc_md29174}{}\doxysubsection{\texorpdfstring{Security}{Security}}\label{readme.md_autotoc_md29174}
