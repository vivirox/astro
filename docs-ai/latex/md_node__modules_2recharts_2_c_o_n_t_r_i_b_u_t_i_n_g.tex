We do cherish tests. They help us move swiftly, and release with confidence. In our repo, you will find three types of tests\+: Unit tests, rendering tests with RTL, and user interaction tests in storybook. Wherever possible we prefer the simplest tests -\/ unit tests. Only where needed / useful we would use RTL or storybook tests.\hypertarget{md_node__modules_2recharts_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md27908}{}\doxysubsection{\texorpdfstring{Unit tests}{Unit tests}}\label{md_node__modules_2recharts_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md27908}

Storybook also has a great interface for adding tests. By default every story in storybook is a smoke test (rendering without error logs means the test passed). See for example {\ttfamily storybook/stories/\+Examples/cartesian/\+Reference\+Line/\+Reference\+Line\+If\+Overflow.\+stories.\+tsx}\hypertarget{md_node__modules_2recharts_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md27911}{}\doxysubsection{\texorpdfstring{Performance tests}{Performance tests}}\label{md_node__modules_2recharts_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md27911}
Recharts uses (\href{https://callstack.github.io/reassure/}{\texttt{ https\+://callstack.\+github.\+io/reassure/}})\mbox{[}Reassure\mbox{]} to run performance tests. Here is how to use\+:


\begin{DoxyEnumerate}
\item Before making code changes, run {\ttfamily npm run test\+:perf\+:baseline}. This will generate baseline data. You only do this once for each change.
\item Make your code changes now
\item Run {\ttfamily npm test\+:perf}. Reassure will run tests again, compare with the baseline from step 1. and print results to the console. If you wish you can also review the results in a markdown file in path {\ttfamily ./reassure/output.md}.
\item If you are satisfied with the performance impact then go ahead and commit! If you wish to make more changes go back to step 2.
\end{DoxyEnumerate}

Reassure will look for {\ttfamily \texorpdfstring{$\ast$}{*}.perf-\/test.\+tsx} files by default.

Also be aware that the performance tests might take a long time to run; up to several minutes.\hypertarget{md_node__modules_2recharts_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md27912}{}\doxysection{\texorpdfstring{\label{_pr}%

{\itshape Before} submitting a pull request, please make sure the following is doneâ€¦


\begin{DoxyItemize}
\item Search \href{https://github.com/recharts/recharts/pulls}{\texttt{ Git\+Hub}} for an open or closed Pull Request that relates to your submission. You don\textquotesingle{}t want to duplicate effort.
\item Fork the repo and create your branch from {\ttfamily master}.
\item If you have added functionality or changed existing functionality, be sure to add a test. Ideally a unit test for helper function, or a test that includes rendering with RTL.
\item If you\textquotesingle{}ve changed APIs, make sure that the stories in Storybook are working as expected.
\item Ensure the test suite passes ({\ttfamily npm run test}).
\item Make sure your code lints ({\ttfamily npm run lint}) -\/ we\textquotesingle{}ve done our best to make sure these rules match our internal linting guidelines.
