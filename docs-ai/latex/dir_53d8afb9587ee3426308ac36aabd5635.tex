Handle a node (matching {\ttfamily test}, if given) (Type\+Script type).

Visitors are free to transform {\ttfamily node}. They can also transform {\ttfamily parent}.

Replacing {\ttfamily node} itself, if {\ttfamily SKIP} is not returned, still causes its descendants to be walked (which is a bug).

When adding or removing previous siblings of {\ttfamily node} (or next siblings, in case of reverse), the {\ttfamily Visitor} should return a new {\ttfamily Index} to specify the sibling to traverse after {\ttfamily node} is traversed. Adding or removing next siblings of {\ttfamily node} (or previous siblings, in case of reverse) is handled as expected without needing to return a new {\ttfamily Index}.

Removing the children property of {\ttfamily parent} still results in them being traversed.\hypertarget{readme.md_autotoc_md17046}{}\doxysubsubsubsubsubsection{\texorpdfstring{Parameters}{Parameters}}\label{readme.md_autotoc_md17046}

What to do next.

An {\ttfamily Index} is treated as a tuple of {\ttfamily \mbox{[}CONTINUE, Index\mbox{]}}. An {\ttfamily Action} is treated as a tuple of {\ttfamily \mbox{[}Action\mbox{]}}.

Passing a tuple back only makes sense if the {\ttfamily Action} is {\ttfamily SKIP}. When the {\ttfamily Action} is {\ttfamily EXIT}, that action can be returned.
