Type\+: {\ttfamily object}

Define argument flags.

The key is the flag name in camel-\/case and the value is an object with any of\+:


\begin{DoxyItemize}
\item {\ttfamily type}\+: Type of value. (Possible values\+: {\ttfamily string} {\ttfamily boolean} {\ttfamily number})
\item {\ttfamily choices}\+: Limit valid values to a predefined set of choices.
\item {\ttfamily default}\+: Default value when the flag is not specified.
\item {\ttfamily short\+Flag}\+: A short flag alias.
\item {\ttfamily aliases}\+: Other names for the flag.
\item {\ttfamily is\+Multiple}\+: Indicates a flag can be set multiple times. Values are turned into an array. (Default\+: false)
\begin{DoxyItemize}
\item Multiple values are provided by specifying the flag multiple times, for example, {\ttfamily \$ foo -\/u rainbow -\/u cat}. Space-\/ or comma-\/separated values are \href{https://github.com/sindresorhus/meow/issues/164}{\texttt{ currently {\itshape not} supported}}.
\end{DoxyItemize}
\item {\ttfamily is\+Required}\+: Determine if the flag is required. (Default\+: false)
\begin{DoxyItemize}
\item If it\textquotesingle{}s only known at runtime whether the flag is required or not, you can pass a {\ttfamily Function} instead of a {\ttfamily boolean}, which based on the given flags and other non-\/flag arguments, should decide if the flag is required. Two arguments are passed to the function\+:
\item The first argument is the {\bfseries{flags}} object, which contains the flags converted to camel-\/case excluding aliases.
\item The second argument is the {\bfseries{input}} string array, which contains the non-\/flag arguments.
\item The function should return a {\ttfamily boolean}, true if the flag is required, otherwise false.
\end{DoxyItemize}
\end{DoxyItemize}

Note that flags are always defined using a camel-\/case key ({\ttfamily my\+Key}), but will match arguments in kebab-\/case ({\ttfamily -\/-\/my-\/key}).
