\doxysubsection{Detailed Description}
This module provides a Java\+Script implementation of the venerable \href{http://pubs.opengroup.org/onlinepubs/009695399/functions/strptime.html}{\texttt{ strptime}} and \href{http://pubs.opengroup.org/onlinepubs/007908799/xsh/strftime.html}{\texttt{ strftime}} functions from the C standard library, and can be used to parse or format \href{https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Date}{\texttt{ dates}} in a variety of locale-\/specific representations. To format a date, create a \doxylink{README.md_locale_format}{formatter} from a specifier (a string with the desired format {\itshape directives}, indicated by {\ttfamily \%}); then pass a date to the formatter, which returns a string. For example, to convert the current date to a human-\/readable string\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{const\ formatTime\ =\ d3.timeFormat("{}\%B\ \%d,\ \%Y"{});}
\DoxyCodeLine{formatTime(new\ Date);\ //\ "{}June\ 30,\ 2015"{}}

\end{DoxyCode}


Likewise, to convert a string back to a date, create a \doxylink{README.md_locale_parse}{parser}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{const\ parseTime\ =\ d3.timeParse("{}\%B\ \%d,\ \%Y"{});}

\# {\itshape locale}.{\bfseries{parse}}({\itshape specifier}) · \href{https://github.com/d3/d3-time-format/blob/main/src/locale.js}{\texttt{ Source}}

Returns a new parser for the given string {\itshape specifier}. The specifier string may contain the same directives as \doxylink{README.md_locale_format}{\texorpdfstring{$\ast$}{*}locale\texorpdfstring{$\ast$}{*}.format}. The {\ttfamily d} and {\ttfamily e} directives are considered equivalent for parsing.

The returned function parses a specified {\itshape string}, returning the corresponding \href{https://developer.mozilla.org/en/JavaScript/Reference/Global_Objects/Date}{\texttt{ date}} or null if the string could not be parsed according to this format’s specifier. Parsing is strict\+: if the specified {\itshape string} does not exactly match the associated specifier, this method returns null. For example, if the associated specifier is {\ttfamily Y-\/m-\/dTH\+:M\+:SZ}, then the string {\ttfamily "{}2011-\/07-\/01\+T19\+:15\+:28\+Z"{}} will be parsed as expected, but {\ttfamily "{}2011-\/07-\/01\+T19\+:15\+:28"{}}, {\ttfamily "{}2011-\/07-\/01 19\+:15\+:28"{}} and {\ttfamily "{}2011-\/07-\/01"{}} will return null. (Note that the literal {\ttfamily Z} here is different from the time zone offset directive {\ttfamily Z}.) If a more flexible parser is desired, try multiple formats sequentially until one returns non-\/null.
