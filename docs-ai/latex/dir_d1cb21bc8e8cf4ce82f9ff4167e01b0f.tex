This package is a hyperscript interface (like {\ttfamily create\+Element} from React and {\ttfamily h} from Vue and such) to help with creating hast trees.\hypertarget{readme.md_autotoc_md17647}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{readme.md_autotoc_md17647}
You can use this utility in your project when you generate hast syntax trees with code. It helps because it replaces most of the repetition otherwise needed in a syntax tree with function calls. It also helps as it improves the attributes you pass by turning them into the form that is required by hast.

You can instead use \href{https://github.com/syntax-tree/unist-builder}{\texttt{ {\ttfamily unist-\/builder}}} when creating any unist nodes and \href{https://github.com/syntax-tree/xastscript}{\texttt{ {\ttfamily xastscript}}} when creating xast (XML) nodes.\hypertarget{readme.md_autotoc_md17648}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md17648}

\hypertarget{readme.md_autotoc_md17654}{}\doxysubsubsubsubsubsection{\texorpdfstring{{\ttfamily selector}}{{\ttfamily selector}}}\label{readme.md_autotoc_md17654}
Simple CSS selector ({\ttfamily string}, optional). When string, builds an \href{https://github.com/syntax-tree/hast\#element}{\texttt{ {\ttfamily Element}}}. When nullish, builds a \href{https://github.com/syntax-tree/hast\#root}{\texttt{ {\ttfamily Root}}} instead. The selector can contain a tag name ({\ttfamily foo}), IDs ({\ttfamily \#bar}), and classes ({\ttfamily .baz}). If the selector is a string but there is no tag name in it then {\ttfamily h} defaults to build a {\ttfamily div} element and {\ttfamily s} to a {\ttfamily g} element. {\ttfamily selector} is parsed by \href{https://github.com/syntax-tree/hast-util-parse-selector}{\texttt{ {\ttfamily hast-\/util-\/parse-\/selector}}}.\hypertarget{readme.md_autotoc_md17655}{}\doxysubsubsubsubsubsection{\texorpdfstring{{\ttfamily properties}}{{\ttfamily properties}}}\label{readme.md_autotoc_md17655}

\hypertarget{readme.md_autotoc_md17667}{}\doxysubsection{\texorpdfstring{Compatibility}{Compatibility}}\label{readme.md_autotoc_md17667}
Projects maintained by the unified collective are compatible with maintained versions of Node.\+js.

When we cut a new major release, we drop support for unmaintained versions of Node. This means we try to keep the current release line, {\ttfamily hastscript@9}, compatible with Node.\+js 16.\hypertarget{readme.md_autotoc_md17668}{}\doxysubsection{\texorpdfstring{Security}{Security}}\label{readme.md_autotoc_md17668}
Use of {\ttfamily hastscript} can open you up to a \href{https://en.wikipedia.org/wiki/Cross-site_scripting}{\texttt{ cross-\/site scripting (XSS)}} when you pass user-\/provided input to it because values are injected into the syntax tree.
