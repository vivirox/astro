Look up info on a property.

In most cases, the given {\ttfamily schema} contains info on the property. All standard, most legacy, and some non-\/standard properties are supported. For these cases, the returned \`{}\+Info\`{} has hints about value of the property.

{\ttfamily name} can be a \href{https://developer.mozilla.org/en-US/docs/Web/API/HTMLElement/dataset}{\texttt{ valid data attribute or property}}, in which case an \`{}\+Info\`{} object with the correctly cased {\ttfamily attribute} and {\ttfamily property} is returned.

{\ttfamily name} can be an unknown attribute, in which case an \`{}\+Info\`{} object with {\ttfamily attribute} and {\ttfamily property} set to the given name is returned. It is not recommended to provide unsupported legacy or recently specced properties.\hypertarget{readme.md_autotoc_md28461}{}\doxysubsubsubsection{\texorpdfstring{Parameters}{Parameters}}\label{readme.md_autotoc_md28461}

\begin{DoxyItemize}
\item {\ttfamily schema} (\`{}\+Schema\`{}) — Either {\ttfamily property\+Information.\+html} or {\ttfamily property\+Information.\+svg}
\item {\ttfamily name} ({\ttfamily string}) — An attribute-\/like or property-\/like name that is passed through \`{}normalize\`{} to find the correct info
\end{DoxyItemize}\hypertarget{readme.md_autotoc_md28462}{}\doxysubsubsubsection{\texorpdfstring{Returns}{Returns}}\label{readme.md_autotoc_md28462}
