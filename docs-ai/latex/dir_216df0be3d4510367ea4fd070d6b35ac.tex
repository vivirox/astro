This package is a utility that takes markdown input and turns it into an \href{https://github.com/syntax-tree/mdast}{\texttt{ mdast}} syntax tree.

This utility uses \href{https://github.com/micromark/micromark}{\texttt{ {\ttfamily micromark}}}, which turns markdown into tokens, and then turns those tokens into nodes. This package is used inside \href{https://github.com/remarkjs/remark/tree/main/packages/remark-parse}{\texttt{ {\ttfamily remark-\/parse}}}, which focusses on making it easier to transform content by abstracting these internals away.\hypertarget{readme.md_autotoc_md22335}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{readme.md_autotoc_md22335}
If you want to handle syntax trees manually, use this. When you {\itshape just} want to turn markdown into HTML, use \href{https://github.com/micromark/micromark}{\texttt{ {\ttfamily micromark}}} instead. For an easier time processing content, use the {\bfseries{\href{https://github.com/remarkjs/remark}{\texttt{ remark}}}} ecosystem instead.

You can combine this package with other packages to add syntax extensions to markdown. Notable examples that deeply integrate with this package are \href{https://github.com/syntax-tree/mdast-util-gfm}{\texttt{ {\ttfamily mdast-\/util-\/gfm}}}, \href{https://github.com/syntax-tree/mdast-util-mdx}{\texttt{ {\ttfamily mdast-\/util-\/mdx}}}, \href{https://github.com/syntax-tree/mdast-util-frontmatter}{\texttt{ {\ttfamily mdast-\/util-\/frontmatter}}}, \href{https://github.com/syntax-tree/mdast-util-math}{\texttt{ {\ttfamily mdast-\/util-\/math}}}, and \href{https://github.com/syntax-tree/mdast-util-directive}{\texttt{ {\ttfamily mdast-\/util-\/directive}}}.\hypertarget{readme.md_autotoc_md22336}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md22336}
