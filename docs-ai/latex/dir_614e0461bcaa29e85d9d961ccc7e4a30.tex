\doxysubsection{Detailed Description}
\href{https://github.com/wooorm/refractor/actions}{\texttt{ }} \href{https://codecov.io/github/wooorm/refractor}{\texttt{ }} \href{https://www.npmjs.com/package/refractor}{\texttt{ }} \href{https://bundlephobia.com/result?p=refractor}{\texttt{ }}

Lightweight, robust, elegant virtual syntax highlighting using \href{https://github.com/PrismJS/prism}{\texttt{ Prism}}. Useful for virtual DOMs and non-\/\+HTML things. Perfect for \href{https://facebook.github.io/react/}{\texttt{ React}}, \href{https://github.com/Matt-Esch/virtual-dom}{\texttt{ VDOM}}, and others.

{\ttfamily refractor} is built to work with all syntaxes supported by \href{https://github.com/PrismJS/prism}{\texttt{ Prism}}, that’s \href{https://prismjs.com/\#languages-list}{\texttt{ 277 languages}} (as of \href{https://www.npmjs.com/package/prismjs}{\texttt{ {\ttfamily prism@1.\+27.\+0}}}) and all \href{https://prismjs.com/\#theme}{\texttt{ themes}}.

Want to use \href{https://github.com/isagalaev/highlight.js}{\texttt{ {\ttfamily highlight.\+js}}} instead? {\ttfamily refractor} does not support Prism plugins\+:


\begin{DoxyEnumerate}
\item Prism plugins often deal with the DOM, not Prism tokens
\item Prism is made using global variables instead of a module format, so all syntaxes below are custom built to work so you can {\ttfamily require} just what you need
\end{DoxyEnumerate}\hypertarget{readme.md_autotoc_md28513}{}\doxysubsection{\texorpdfstring{Syntaxes}{Syntaxes}}\label{readme.md_autotoc_md28513}
All syntaxes are included if you `require(\textquotesingle{}refractor'){\ttfamily . If you’re using}refractor/core{\ttfamily , checked syntaxes are always included, but unchecked syntaxes are not and must be}require{\ttfamily d and \mbox{[}}register\`{}\mbox{]}registered.

Unlike in Prism, {\ttfamily css\+Extras} and {\ttfamily php\+Extras} are camel-\/cased instead of dash-\/cased.

Only these custom built syntaxes will work with {\ttfamily refractor} because Prism’s own syntaxes are made to work with global variables and are not requirable.
