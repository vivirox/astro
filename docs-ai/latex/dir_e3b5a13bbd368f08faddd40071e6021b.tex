This package is a \href{https://github.com/unifiedjs/unified}{\texttt{ unified}} (\href{https://github.com/rehypejs/rehype}{\texttt{ rehype}}) plugin to render math. You can add classes to HTML elements, use fenced code in markdown, or combine with \href{../remark-math/}{\texttt{ {\ttfamily remark-\/math}}} for a {\ttfamily \$C\$} syntax extension.\hypertarget{readme.md_autotoc_md28809}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{readme.md_autotoc_md28809}
This project is useful as it renders math with Ka\+TeX at compile time, which means that there is no client side Java\+Script needed.

A different plugin, \href{../rehype-mathjax/}{\texttt{ {\ttfamily rehype-\/mathjax}}}, does the same but with \href{https://www.mathjax.org}{\texttt{ Math\+Jax}}.\hypertarget{readme.md_autotoc_md28810}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md28810}

The content of any element with a {\ttfamily language-\/math}, {\ttfamily math-\/inline}, or {\ttfamily math-\/display} class is transformed. The elements are replaced by what Ka\+TeX renders. Either a {\ttfamily math-\/display} class or using {\ttfamily \texorpdfstring{$<$}{<}pre\texorpdfstring{$>$}{>}\texorpdfstring{$<$}{<}code class="{}language-\/math"{}\texorpdfstring{$>$}{>}} will result in “display” math\+: math that is a centered block on its own line.\hypertarget{readme.md_autotoc_md28820}{}\doxysubsection{\texorpdfstring{CSS}{CSS}}\label{readme.md_autotoc_md28820}

Projects maintained by the unified collective are compatible with maintained versions of Node.\+js.

When we cut a new major release, we drop support for unmaintained versions of Node. This means we try to keep the current release line, {\ttfamily rehype-\/katex@\texorpdfstring{$^\wedge$}{\string^}7}, compatible with Node.\+js 16.

This plugin works with unified version 6+ and rehype version 4+.\hypertarget{readme.md_autotoc_md28823}{}\doxysubsection{\texorpdfstring{Security}{Security}}\label{readme.md_autotoc_md28823}
Assuming you trust Ka\+TeX, using {\ttfamily rehype-\/katex} is safe. A vulnerability in it could open you to a \href{https://en.wikipedia.org/wiki/Cross-site_scripting}{\texttt{ cross-\/site scripting (XSS)}} attack. Be wary of user input and use \href{https://github.com/rehypejs/rehype-sanitize}{\texttt{ {\ttfamily rehype-\/sanitize}}}.

When you don’t trust user content but do trust Ka\+TeX, run {\ttfamily rehype-\/katex} {\itshape after} {\ttfamily rehype-\/sanitize}\+:
