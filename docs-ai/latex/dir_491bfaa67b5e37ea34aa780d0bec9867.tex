\begin{DoxyCode}{0}

\DoxyCodeLine{}
\DoxyCodeLine{export\ default\ [}
\DoxyCodeLine{\ \ \{}
\DoxyCodeLine{\ \ \ \ files:\ ["{}tests/**"{}],\ //\ or\ any\ other\ pattern}

\DoxyCodeLine{\ \ \ \ \ \ "{}vitest/max-\/nested-\/describe"{}:\ ["{}error"{},\ \{\ "{}max"{}:\ 3\ \}]\ //\ you\ can\ also\ modify\ rules'\ behavior\ using\ option\ like\ this}

\hypertarget{README.md_autotoc_md7222}{}\doxysubsubsubsection{\texorpdfstring{Enabling with type-\/testing}{Enabling with type-\/testing}}\label{README.md_autotoc_md7222}
Vitest ships with an optional \href{https://vitest.dev/guide/testing-types}{\texttt{ type-\/testing feature}}, which is disabled by default.

If you\textquotesingle{}re using this feature, you should also enabled {\ttfamily typecheck} in the settings for this plugin. This ensures that rules like expect-\/expect account for type-\/related assertions in tests.


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ vitest\ from\ "{}@vitest/eslint-\/plugin"{};}
\DoxyCodeLine{}
\DoxyCodeLine{export\ default\ [}

\DoxyCodeLine{\ \ \ \ files:\ ["{}tests/**"{}],\ //\ or\ any\ other\ pattern}

\DoxyCodeLine{\ \ \ \ \ \ vitest,}
\DoxyCodeLine{\ \ \ \ \},}
\DoxyCodeLine{\ \ \ \ rules:\ \{}
\DoxyCodeLine{\ \ \ \ \ \ ...vitest.configs.recommended.rules,}
\DoxyCodeLine{\ \ \ \ \},}
\DoxyCodeLine{\ \ \ settings:\ \{}
\DoxyCodeLine{\ \ \ \ \ \ vitest:\ \{}
