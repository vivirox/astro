\hypertarget{README.md_autotoc_md34487}{}\doxysubsubsection{\texorpdfstring{Features}{Features}}\label{README.md_autotoc_md34487}
Inspired by \href{http://windicss.org/}{\texttt{ Windi CSS}}, \href{https://tailwindcss.com/}{\texttt{ Tailwind CSS}}, and \href{https://github.com/tw-in-js/twind}{\texttt{ Twind}}, but\+:


\begin{DoxyItemize}
\item \href{https://unocss.dev/config/}{\texttt{ Fully customizable}} -\/ no core utilities. All functionalities are provided via presets.
\item No parsing, no AST, no scanning, it\textquotesingle{}s {\bfseries{INSTANT}} (5x faster than Windi CSS or Tailwind JIT).
\item \texorpdfstring{$\sim$}{\string~}6kb min+brotli -\/ zero deps and browser friendly.
\item \href{https://unocss.dev/config/shortcuts}{\texttt{ Shortcuts}} -\/ aliasing utilities, dynamically.
\item \href{https://unocss.dev/presets/attributify/}{\texttt{ Attributify mode}} -\/ group utilities in attributes.
\item \href{https://unocss.dev/presets/icons/}{\texttt{ Pure CSS Icons}} -\/ use any icon as a single class.
\item \href{https://unocss.dev/transformers/variant-group}{\texttt{ Variant Groups}} -\/ shorthand for group utils with common prefixes.
\item \href{https://unocss.dev/transformers/directives}{\texttt{ CSS Directives}} -\/ reuse utils in CSS with {\ttfamily @apply} directive.
\item \href{https://unocss.dev/transformers/compile-class/}{\texttt{ Compilation mode}} -\/ synthesizes multiple classes into one at build time.
\item \href{https://unocss.dev/tools/inspector}{\texttt{ Inspector}} -\/ inspect and debug interactively.
\item \href{https://unocss.dev/integrations/runtime}{\texttt{ CSS-\/in-\/\+JS Runtime build}} -\/ use Uno\+CSS with one line of CDN import.
\item \href{https://marketplace.visualstudio.com/items?itemName=antfu.unocss}{\texttt{ VS Code extension}}
\item Code-\/splitting for CSS -\/ ships minimal CSS for MPA.
\end{DoxyItemize}\hypertarget{README.md_autotoc_md34488}{}\doxysubsubsection{\texorpdfstring{Documentation}{Documentation}}\label{README.md_autotoc_md34488}
