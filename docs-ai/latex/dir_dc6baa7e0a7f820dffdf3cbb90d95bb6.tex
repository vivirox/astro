\doxysubsection{Detailed Description}
An extremely fast CSS parser, transformer, and minifier written in Rust. Use it with \href{https://parceljs.org}{\texttt{ Parcel}}, as a standalone library or CLI, or via a plugin with any other tool.

 \hypertarget{README.md_autotoc_md21588}{}\doxysubsection{\texorpdfstring{Features}{Features}}\label{README.md_autotoc_md21588}

\begin{DoxyItemize}
\item {\bfseries{Extremely fast}} – Parsing and minifying large files is completed in milliseconds, often with significantly smaller output than other tools. See benchmarks below.
\item {\bfseries{Typed property values}} – many other CSS parsers treat property values as an untyped series of tokens. This means that each transformer that wants to do something with these values must interpret them itself, leading to duplicate work and inconsistencies. Lightning CSS parses all values using the grammar from the CSS specification, and exposes a specific value type for each property.
\item {\bfseries{Browser-\/grade parser}} – Lightning CSS is built on the \href{https://github.com/servo/rust-cssparser}{\texttt{ cssparser}} and \href{https://github.com/servo/stylo/tree/main/selectors}{\texttt{ selectors}} crates created by Mozilla and used by Firefox and Servo. These provide a solid general purpose CSS-\/parsing foundation on top of which Lightning CSS implements support for all specific CSS rules and properties.
\item {\bfseries{Minification}} – One of the main purposes of Lightning CSS is to minify CSS to make it smaller. This includes many optimizations including\+:
\begin{DoxyItemize}
\item Combining longhand properties into shorthands where possible.
\item Merging adjacent rules with the same selectors or declarations when it is safe to do so.
\item Combining CSS transforms into a single matrix or vice versa when smaller.
\item Removing vendor prefixes that are not needed, based on the provided browser targets.
\item Reducing {\ttfamily calc()} expressions where possible.
\item Converting colors to shorter hex notation where possible.
\item Minifying gradients.
\item Minifying CSS grid templates.
\item Normalizing property value order.
\item Removing default property sub-\/values which will be inferred by browsers.
\item Many micro-\/optimizations, e.\+g. converting to shorter units, removing unnecessary quotation marks, etc.
\end{DoxyItemize}
\item {\bfseries{Vendor prefixing}} – Lightning CSS accepts a list of browser targets, and automatically adds (and removes) vendor prefixes.
\item {\bfseries{Browserslist configuration}} – Lightning CSS supports opt-\/in browserslist configuration discovery to resolve browser targets and integrate with your existing tools and config setup.
\item {\bfseries{Syntax lowering}} – Lightning CSS parses modern CSS syntax, and generates more compatible output where needed, based on browser targets.
\begin{DoxyItemize}
\item CSS Nesting
\item Custom media queries (draft spec)
\item Logical properties
\item \href{https://drafts.csswg.org/css-color-5/}{\texttt{ Color Level 5}}
\begin{DoxyItemize}
\item {\ttfamily color-\/mix()} function
\item Relative color syntax, e.\+g. {\ttfamily lab(from purple calc(l \texorpdfstring{$\ast$}{*} .8) a b)}
\end{DoxyItemize}
\item \href{https://drafts.csswg.org/css-color-4/}{\texttt{ Color Level 4}}
\begin{DoxyItemize}
\item {\ttfamily lab()}, {\ttfamily lch()}, {\ttfamily oklab()}, and {\ttfamily oklch()} colors
\item {\ttfamily color()} function supporting predefined color spaces such as {\ttfamily display-\/p3} and {\ttfamily xyz}
\item Space separated components in {\ttfamily rgb} and {\ttfamily hsl} functions
\item Hex with alpha syntax
\item {\ttfamily hwb()} color syntax
\item Percent syntax for opacity
\item {\ttfamily \#rgba} and {\ttfamily \#rrggbbaa} hex colors
\end{DoxyItemize}
\item Selectors
\begin{DoxyItemize}
\item {\ttfamily \+:not} with multiple arguments
\item {\ttfamily \+:lang} with multiple arguments
\item {\ttfamily \+:dir}
\item {\ttfamily \+:is}
\end{DoxyItemize}
\item Double position gradient stops (e.\+g. {\ttfamily red 40\% 80\%})
\item {\ttfamily clamp()}, {\ttfamily round()}, {\ttfamily rem()}, and {\ttfamily mod()} math functions
\item Alignment shorthands (e.\+g. {\ttfamily place-\/items})
\item Two-\/value {\ttfamily overflow} shorthand
\item Media query range syntax (e.\+g. {\ttfamily @media (width \texorpdfstring{$<$}{<}= 100px)} or {\ttfamily @media (100px \texorpdfstring{$<$}{<} width \texorpdfstring{$<$}{<} 500px)})
\item Multi-\/value {\ttfamily display} property (e.\+g. {\ttfamily inline flex})
\item {\ttfamily system-\/ui} font family fallbacks
\end{DoxyItemize}
\item {\bfseries{CSS modules}} – Lightning CSS supports compiling a subset of \href{https://github.com/css-modules/css-modules}{\texttt{ CSS modules}} features.
\begin{DoxyItemize}
\item Locally scoped class and id selectors
\item Locally scoped custom identifiers, e.\+g. {\ttfamily @keyframes} names, grid lines/areas, {\ttfamily @counter-\/style} names, etc.
\item Opt-\/in support for locally scoped CSS variables and other dashed identifiers.
\item {\ttfamily \+:local()} and {\ttfamily \+:global()} selectors
\item The {\ttfamily composes} property
\end{DoxyItemize}
\item {\bfseries{Custom transforms}} – The Lightning CSS visitor API can be used to implement custom transform plugins.
\end{DoxyItemize}\hypertarget{README.md_autotoc_md21589}{}\doxysubsection{\texorpdfstring{Documentation}{Documentation}}\label{README.md_autotoc_md21589}
