Static HTML elements do not have semantic meaning. This is clear in the case of {\ttfamily \texorpdfstring{$<$}{<}div\texorpdfstring{$>$}{>}} and {\ttfamily \texorpdfstring{$<$}{<}span\texorpdfstring{$>$}{>}}. It is less so clear in the case of elements that {\itshape seem} semantic, but that do not have a semantic mapping in the accessibility layer. For example {\ttfamily \texorpdfstring{$<$}{<}a\texorpdfstring{$>$}{>}}, {\ttfamily \texorpdfstring{$<$}{<}big\texorpdfstring{$>$}{>}}, {\ttfamily \texorpdfstring{$<$}{<}blockquote\texorpdfstring{$>$}{>}}, {\ttfamily \texorpdfstring{$<$}{<}footer\texorpdfstring{$>$}{>}}, {\ttfamily \texorpdfstring{$<$}{<}picture\texorpdfstring{$>$}{>}}, {\ttfamily \texorpdfstring{$<$}{<}strike\texorpdfstring{$>$}{>}} and {\ttfamily \texorpdfstring{$<$}{<}time\texorpdfstring{$>$}{>}} -- to name a few -- have no semantic layer mapping. They are as void of meaning as {\ttfamily \texorpdfstring{$<$}{<}div\texorpdfstring{$>$}{>}}.

The \href{https://www.w3.org/TR/wai-aria-1.1/\#usage_intro}{\texttt{ WAI-\/\+ARIA {\ttfamily role} attribute}} confers a semantic mapping to an element. The semantic value can then be expressed to a user via assistive technology.

In order to add interactivity such as a mouse or key event listener to a static element, that element must be given a role value as well.\hypertarget{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2no-static-element-interactions_autotoc_md13946}{}\doxysection{\texorpdfstring{How do I resolve this error?}{How do I resolve this error?}}\label{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2no-static-element-interactions_autotoc_md13946}

Note\+: Adding a role to your element does {\bfseries{not}} add behavior. When a semantic HTML element like {\ttfamily \texorpdfstring{$<$}{<}button\texorpdfstring{$>$}{>}} is used, then it will also respond to Enter key presses when it has focus. The developer is responsible for providing the expected behavior of an element that the role suggests it would have\+: focusability and key press support.\hypertarget{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2no-static-element-interactions_autotoc_md13948}{}\doxysubsection{\texorpdfstring{Case\+: The event handler is only being used to capture bubbled events}{Case\+: The event handler is only being used to capture bubbled events}}\label{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2no-static-element-interactions_autotoc_md13948}
If your element is catching bubbled click or key events from descendant elements, there are no appropriate roles for your element\+: you will have to deactivate the rule. Consider explaining the reason for disabling the rule as well.
