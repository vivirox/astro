This package is a \href{https://github.com/unifiedjs/unified}{\texttt{ unified}} (\href{https://github.com/rehypejs/rehype}{\texttt{ rehype}}) plugin to format whitespace in HTML. In short, it works as follows\+:


\begin{DoxyItemize}
\item collapse all existing whitespace to either a line ending or a single space
\item remove those spaces and line endings if they do not contribute to the document
\item inject needed line endings
\item indent previously collapsed line endings properly
\end{DoxyItemize}

{\bfseries{unified}} is a project that transforms content with abstract syntax trees (ASTs). {\bfseries{rehype}} adds support for HTML to unified. {\bfseries{hast}} is the HTML AST that rehype uses. This is a rehype plugin that changes whitespace in hast.\hypertarget{readme.md_autotoc_md28788}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{readme.md_autotoc_md28788}
This package is useful when you want to improve the readability of HTML source code as it adds insignificant but pretty whitespace between elements. The package \href{https://github.com/syntax-tree/hast-util-format}{\texttt{ {\ttfamily hast-\/util-\/format}}} does the same as this plugin at the utility level. A different plugin, \href{https://github.com/rehypejs/rehype/tree/main/packages/rehype-stringify}{\texttt{ {\ttfamily rehype-\/stringify}}}, controls how HTML is actually printed\+: which quotes to use, whether to put a {\ttfamily /} on {\ttfamily \texorpdfstring{$<$}{<}img /\texorpdfstring{$>$}{>}}, etc. Yet another project, \href{https://github.com/rehypejs/rehype-minify}{\texttt{ {\ttfamily rehype-\/minify}}}, does the inverse\+: improve the size of HTML source code by making it hard to read.\hypertarget{readme.md_autotoc_md28789}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md28789}

Projects maintained by the unified collective are compatible with maintained versions of Node.\+js.

When we cut a new major release, we drop support for unmaintained versions of Node. This means we try to keep the current release line, {\ttfamily rehype-\/format@5}, compatible with Node.\+js 16.

This plugin works with {\ttfamily rehype-\/parse} version 3+, {\ttfamily rehype-\/stringify} version 3+, {\ttfamily rehype} version 5+, and {\ttfamily unified} version 6+.\hypertarget{readme.md_autotoc_md28802}{}\doxysubsection{\texorpdfstring{Security}{Security}}\label{readme.md_autotoc_md28802}
Use of {\ttfamily rehype-\/format} changes whitespace in the tree. Whitespace in {\ttfamily \texorpdfstring{$<$}{<}script\texorpdfstring{$>$}{>}}, {\ttfamily \texorpdfstring{$<$}{<}style\texorpdfstring{$>$}{>}}, {\ttfamily \texorpdfstring{$<$}{<}pre\texorpdfstring{$>$}{>}}, or {\ttfamily \texorpdfstring{$<$}{<}textarea\texorpdfstring{$>$}{>}} is not modified. If the tree is already safe, use of this plugin does not open you up for a \href{https://en.wikipedia.org/wiki/Cross-site_scripting}{\texttt{ cross-\/site scripting (XSS)}} attack. When in doubt, use \href{https://github.com/rehypejs/rehype-sanitize}{\texttt{ {\ttfamily rehype-\/sanitize}}}.\hypertarget{readme.md_autotoc_md28803}{}\doxysubsection{\texorpdfstring{Related}{Related}}\label{readme.md_autotoc_md28803}
