 \$ semver -\/h

A Java\+Script implementation of the \href{https://semver.org/}{\texttt{ https\+://semver.\+org/}} specification Copyright Isaac Z. Schlueter

Usage\+: semver \mbox{[}options\mbox{]} $<$version$>$ \mbox{[}$<$version$>$ \mbox{[}...\mbox{]}\mbox{]} Prints valid versions sorted by Sem\+Ver precedence

Options\+: -\/r --range $<$range$>$ Print versions that match the specified range.

-\/i --increment \mbox{[}$<$level$>$\mbox{]} Increment a version by the specified level. Level can be one of\+: major, minor, patch, premajor, preminor, prepatch, or prerelease. Default level is \textquotesingle{}patch\textquotesingle{}. Only one version may be specified.

--preid $<$identifier$>$ Identifier to be used to prefix premajor, preminor, prepatch or prerelease version increments.

-\/l --loose Interpret versions and ranges loosely

-\/p --include-\/prerelease Always include prerelease versions in range matching

-\/c --coerce Coerce a string into Sem\+Ver if possible (does not imply --loose)

--rtl Coerce version strings right to left

--ltr Coerce version strings left to right (default)

Program exits successfully if any valid version satisfies all supplied ranges, and prints all satisfying versions.

If no satisfying versions are found, then exits failure.

Versions are printed in ascending order, so supplying multiple versions to the utility will just sort them.

\DoxyCodeLine{A\ \`{}version\ range`\ is\ a\ set\ of\ \`{}comparators`\ which\ specify\ versions}

\DoxyCodeLine{}
\DoxyCodeLine{A\ \`{}comparator`\ is\ composed\ of\ an\ \`{}operator`\ and\ a\ \`{}version`.\ \ The\ set}
\DoxyCodeLine{of\ primitive\ \`{}operators`\ is:}
\DoxyCodeLine{}
\DoxyCodeLine{*\ \`{}<`\ Less\ than}
\DoxyCodeLine{*\ \`{}<=`\ Less\ than\ or\ equal\ to}

\DoxyCodeLine{*\ \`{}>=`\ Greater\ than\ or\ equal\ to}

\DoxyCodeLine{\ \ so\ this\ operator\ is\ optional,\ but\ MAY\ be\ included.}
\DoxyCodeLine{}
\DoxyCodeLine{For\ example,\ the\ comparator\ \`{}>=1.2.7`\ would\ match\ the\ versions}
\DoxyCodeLine{\`{}1.2.7`,\ \`{}1.2.8`,\ \`{}2.5.3`,\ and\ \`{}1.3.9`,\ but\ not\ the\ versions\ \`{}1.2.6`}

\DoxyCodeLine{}
\DoxyCodeLine{Comparators\ can\ be\ joined\ by\ whitespace\ to\ form\ a\ \`{}comparator\ set`,}
\DoxyCodeLine{which\ is\ satisfied\ by\ the\ **intersection**\ of\ all\ of\ the\ comparators}
\DoxyCodeLine{it\ includes.}
\DoxyCodeLine{}
\DoxyCodeLine{A\ range\ is\ composed\ of\ one\ or\ more\ comparator\ sets,\ joined\ by\ \`{}||`.\ \ A}
\DoxyCodeLine{version\ matches\ a\ range\ if\ and\ only\ if\ every\ comparator\ in\ at\ least}
\DoxyCodeLine{one\ of\ the\ \`{}||`-\/separated\ comparator\ sets\ is\ satisfied\ by\ the\ version.}
\DoxyCodeLine{}
\DoxyCodeLine{For\ example,\ the\ range\ \`{}>=1.2.7\ <1.3.0`\ would\ match\ the\ versions}
\DoxyCodeLine{\`{}1.2.7`,\ \`{}1.2.8`,\ and\ \`{}1.2.99`,\ but\ not\ the\ versions\ \`{}1.2.6`,\ \`{}1.3.0`,}

All methods and classes take a final {\ttfamily options} object argument. All options in this object are {\ttfamily false} by default. The options supported are\+:


\begin{DoxyItemize}
\item {\ttfamily loose} Be more forgiving about not-\/quite-\/valid semver strings. (Any resulting output will always be 100\% strict compliant, of course.) For backwards compatibility reasons, if the {\ttfamily options} argument is a boolean value instead of an object, it is interpreted to be the {\ttfamily loose} param.
\item {\ttfamily include\+Prerelease} Set to suppress the \href{https://github.com/npm/node-semver\#prerelease-tags}{\texttt{ default behavior}} of excluding prerelease tagged versions from ranges unless they are explicitly opted into.
\end{DoxyItemize}

Strict-\/mode Comparators and Ranges will be strict about the Sem\+Ver strings that they parse.

\begin{DoxyItemize}
\item {\ttfamily coerce(version, options)}\+: Coerces a string to semver if possible
\end{DoxyItemize}

This aims to provide a very forgiving translation of a non-\/semver string to semver. It looks for the first digit in a string, and consumes all remaining characters which satisfy at least a partial semver (e.\+g., {\ttfamily 1}, {\ttfamily 1.\+2}, {\ttfamily 1.\+2.\+3}) up to the max permitted length (256 characters). Longer versions are simply truncated ({\ttfamily 4.\+6.\+3.\+9.\+2-\/alpha2} becomes {\ttfamily 4.\+6.\+3}). All surrounding text is simply ignored ({\ttfamily v3.\+4 replaces v3.\+3.\+1} becomes {\ttfamily 3.\+4.\+0}). Only text which lacks digits will fail coercion ({\ttfamily version one} is not valid). The maximum length for any semver component considered for coercion is 16 characters; longer components will be ignored ({\ttfamily 10000000000000000.\+4.\+7.\+4} becomes {\ttfamily 4.\+7.\+4}). The maximum value for any semver component is {\ttfamily Integer.\+MAX\+\_\+\+SAFE\+\_\+\+INTEGER \texorpdfstring{$\vert$}{|}\texorpdfstring{$\vert$}{|} (2\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}53 -\/ 1)}; higher value components are invalid ({\ttfamily 9999999999999999.\+4.\+7.\+4} is likely invalid).

If the {\ttfamily options.\+rtl} flag is set, then {\ttfamily coerce} will return the right-\/most coercible tuple that does not share an ending index with a longer coercible tuple. For example, {\ttfamily 1.\+2.\+3.\+4} will return {\ttfamily 2.\+3.\+4} in rtl mode, not {\ttfamily 4.\+0.\+0}. {\ttfamily 1.\+2.\+3/4} will return {\ttfamily 4.\+0.\+0}, because the {\ttfamily 4} is not a part of any other overlapping Sem\+Ver tuple.\hypertarget{README.md_autotoc_md2490}{}\doxysubsubsection{\texorpdfstring{Clean}{Clean}}\label{README.md_autotoc_md2490}
