This package contains an extension that adds support for the tagfilter enabled by GFM to \href{https://github.com/micromark/micromark}{\texttt{ {\ttfamily micromark}}}. The tagfilter is kinda weird and kinda useless. This package exists for completeness. The tag filter is a naïve attempt at XSS protection. You should use a proper HTML sanitizing algorithm.\hypertarget{readme.md_autotoc_md23263}{}\doxysubsection{\texorpdfstring{When to use this}{When to use this}}\label{readme.md_autotoc_md23263}
This project is useful when you want to match how Git\+Hub works. You can use this extension when you are working with \href{https://github.com/micromark/micromark}{\texttt{ {\ttfamily micromark}}} already. When you do, you can instead use \href{https://github.com/micromark/micromark-extension-gfm}{\texttt{ {\ttfamily micromark-\/extension-\/gfm}}}, which includes this extension, to support all GFM features.

When you want to deal with syntax trees, you should instead use \href{https://github.com/syntax-tree/hast-util-sanitize}{\texttt{ {\ttfamily hast-\/util-\/sanitize}}}.

When you use remark and rehype, you should use \href{https://github.com/rehypejs/rehype-sanitize}{\texttt{ {\ttfamily rehype-\/sanitize}}}.\hypertarget{readme.md_autotoc_md23264}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md23264}
