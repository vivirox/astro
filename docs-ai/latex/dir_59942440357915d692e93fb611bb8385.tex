\hypertarget{README.md_autotoc_md5660}{}\doxysubsubsubsection{\texorpdfstring{create\+Filter}{create\+Filter}}\label{README.md_autotoc_md5660}
Constructs a filter function which can be used to determine whether or not certain modules should be operated upon.

Parameters\+: {\ttfamily (include?\+: \texorpdfstring{$<$}{<}picomatch\texorpdfstring{$>$}{>}, exclude?\+: \texorpdfstring{$<$}{<}picomatch\texorpdfstring{$>$}{>}, options?\+: Object)}~\newline
 Returns\+: {\ttfamily (id\+: string \texorpdfstring{$\vert$}{|} unknown) =\texorpdfstring{$>$}{>} boolean}\hypertarget{README.md_autotoc_md5661}{}\doxysubsubsubsubsection{\texorpdfstring{{\ttfamily include} and {\ttfamily exclude}}{{\ttfamily include} and {\ttfamily exclude}}}\label{README.md_autotoc_md5661}
Type\+: {\ttfamily String \texorpdfstring{$\vert$}{|} Reg\+Exp \texorpdfstring{$\vert$}{|} Array\mbox{[}...String\texorpdfstring{$\vert$}{|}\+Reg\+Exp\mbox{]}}~\newline


A valid \href{https://github.com/micromatch/picomatch\#globbing-features}{\texttt{ {\ttfamily picomatch}}} pattern, or array of patterns. If {\ttfamily options.\+include} is omitted or has zero length, filter will return {\ttfamily true} by default. Otherwise, an ID must match one or more of the {\ttfamily picomatch} patterns, and must not match any of the {\ttfamily options.\+exclude} patterns.

Note that {\ttfamily picomatch} patterns are very similar to \href{https://github.com/isaacs/minimatch\#readme}{\texttt{ {\ttfamily minimatch}}} patterns, and in most use cases, they are interchangeable. If you have more specific pattern matching needs, you can view \href{https://github.com/micromatch/picomatch\#library-comparisons}{\texttt{ this comparison table}} to learn more about where the libraries differ.\hypertarget{README.md_autotoc_md5662}{}\doxysubsubsubsubsection{\texorpdfstring{{\ttfamily options}}{{\ttfamily options}}}\label{README.md_autotoc_md5662}
\hypertarget{README.md_autotoc_md5663}{}\doxysubsubsubsubsubsection{\texorpdfstring{{\ttfamily resolve}}{{\ttfamily resolve}}}\label{README.md_autotoc_md5663}
Type\+: {\ttfamily String \texorpdfstring{$\vert$}{|} Boolean \texorpdfstring{$\vert$}{|} null}

Optionally resolves the patterns against a directory other than {\ttfamily process.\+cwd()}. If a {\ttfamily String} is specified, then the value will be used as the base directory. Relative paths will be resolved against {\ttfamily process.\+cwd()} first. If {\ttfamily false}, then the patterns will not be resolved against any directory. This can be useful if you want to create a filter for virtual module names.\hypertarget{README.md_autotoc_md5664}{}\doxysubsubsubsubsection{\texorpdfstring{Usage}{Usage}}\label{README.md_autotoc_md5664}
