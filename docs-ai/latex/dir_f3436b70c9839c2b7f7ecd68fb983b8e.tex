\doxysubsection{Detailed Description}
{\ttfamily string-\/argv} parses a string into an argument array to mimic {\ttfamily process.\+argv}. This is useful when testing Command Line Utilities that you want to pass arguments to and is the opposite of what the other argv utilities do.\hypertarget{README.md_autotoc_md31303}{}\doxysubsection{\texorpdfstring{Installation}{Installation}}\label{README.md_autotoc_md31303}

\hypertarget{README.md_autotoc_md31305}{}\doxysubsubsection{\texorpdfstring{params}{params}}\label{README.md_autotoc_md31305}
{\bfseries{required}}\+: {\bfseries{arguments}} String\+: arguments that you would normally pass to the command line.

{\bfseries{optional}}\+: {\bfseries{environment}} String\+: Adds to the environment position in the argv array. If ommitted then there is no need to call argv.\+split(2) to remove the environment/file values. However if your cli.\+parse method expects a valid argv value then you should include this value.

{\bfseries{optional}}\+: {\bfseries{file}} String\+: file that called the arguments. If omitted then there is no need to call argv.\+split(2) to remove the environment/file values. However if your cli.\+parse method expects a valid argv value then you should include this value.
