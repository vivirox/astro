Enforce that a control (an interactive element) has a text label.

There are two supported ways to supply a control with a text label\+:


\begin{DoxyItemize}
\item Provide text content inside the element.
\item Use the {\ttfamily aria-\/label} attribute on the element, with a text value.
\item Use the {\ttfamily aria-\/labelledby} attribute on the element, and point the IDREF value to an element with an accessible label.
\item Alternatively, with an {\ttfamily img} tag, you may use the {\ttfamily alt} attribute to supply a text description of the image.
\end{DoxyItemize}

The rule is permissive in the sense that it will assume that expressions will eventually provide a label. So an element like this will pass.

\begin{DoxyItemize}
\item {\ttfamily label\+Attributes} is a list of attributes to check on the control component and its children for a label. Use this if you have a custom component that uses a string passed on a prop to render an HTML {\ttfamily label}, for example.
\item {\ttfamily control\+Components} is a list of custom React Components names that will render down to an interactive element.
\item {\ttfamily ignore\+Elements} is an array of elements that should not be considered control (interactive) elements and therefore they do not require a text label.
\item {\ttfamily ignore\+Roles} is an array of ARIA roles that should not be considered control (interactive) roles and therefore they do not require a text label.
\item {\ttfamily depth} (default 2, max 25) is an integer that determines how deep within a {\ttfamily JSXElement} the rule should look for text content or an element with a label to determine if the interactive element will have an accessible label.
