This element is a stand-\/alone control like a button, a link or a form element. A user should be able to reach this element by pressing the tab key on their keyboard.

Add the {\ttfamily tab\+Index} property to your component. A value of zero indicates that this element can be tabbed to.

Generally buttons, links and form elements should be reachable via tab key presses. An element that can be tabbed to is said to be in the {\itshape tab ring}.\hypertarget{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2interactive-supports-focus_autotoc_md13828}{}\doxysubsection{\texorpdfstring{Case\+: I got the error "{}\+Elements with the \textquotesingle{}\$\{role\}\textquotesingle{} interactive role must be focusable"{}. This element is part of a group of buttons, links, menu items, etc. Or this element is part of a composite widget. \doxylink{struct_composite}{Composite} widgets prescribe standard \href{https://www.w3.org/TR/wai-aria-practices-1.1/\#kbd_generalnav}{\texttt{ keyboard interaction patterns}}. Within a group of similar elements -- like a button bar -- or within a composite widget, elements that can be focused are given a tabindex of -\/1. This makes the element {\itshape focusable} but not {\itshape tabbable}. Generally one item in a group should have a tabindex of zero so that a user can tab to the component. Once an element in the component has focus, your key management behaviors will control traversal within the component\textquotesingle{}s pieces. As the UI author, you will need to implement the key handling behaviors such as listening for traversal key (up/down/left/right) presses and moving the page focus between the focusable elements in your widget.

In the example above, the first item in the group can be tabbed to. The developer provides the ability to traverse to the subsequent items via the up/down/left/right arrow keys. Traversing via arrow keys is not provided by the browser or the assistive technology. Marking an element with the role {\ttfamily presentation} indicates to assistive technology that this element should be ignored; it exists to support the web application and is not meant for humans to interact with directly.\hypertarget{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2interactive-supports-focus_autotoc_md13830}{}\doxysection{\texorpdfstring{Rule options}{Rule options}}\label{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2interactive-supports-focus_autotoc_md13830}
This rule takes an options object with the key {\ttfamily tabbable}. The value is an array of interactive ARIA roles that should be considered tabbable, not just focusable. Any interactive role not included in this list will be flagged as needing to be focusable (tabindex of -\/1).
