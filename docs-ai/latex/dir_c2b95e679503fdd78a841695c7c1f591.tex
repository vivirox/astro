This package is a hyperscript interface (like {\ttfamily create\+Element} from React and {\ttfamily h} from Vue and such) to help with creating hast trees.\hypertarget{readme.md_autotoc_md16879}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{readme.md_autotoc_md16879}
You can use this utility in your project when you generate hast syntax trees with code. It helps because it replaces most of the repetition otherwise needed in a syntax tree with function calls. It also helps as it improves the attributes you pass by turning them into the form that is required by hast.

You can instead use \href{https://github.com/syntax-tree/unist-builder}{\texttt{ {\ttfamily unist-\/builder}}} when creating any unist nodes and \href{https://github.com/syntax-tree/xastscript}{\texttt{ {\ttfamily xastscript}}} when creating xast (XML) nodes.\hypertarget{readme.md_autotoc_md16880}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md16880}

If the selector is a string but there is no tag name in it, {\ttfamily h} defaults to build a {\ttfamily div} element, and {\ttfamily s} to a {\ttfamily g} element. {\ttfamily selector} is parsed by \href{https://github.com/syntax-tree/hast-util-parse-selector}{\texttt{ {\ttfamily hast-\/util-\/parse-\/selector}}}. This package can be used with JSX. You should use the automatic JSX runtime set to {\ttfamily hastscript} (also available as the more explicit name {\ttfamily hastscript/html}) or {\ttfamily hastscript/svg}.

\begin{quote}
👉 {\bfseries{Note}}\+: while {\ttfamily h} supports dots ({\ttfamily .}) for classes or number signs ({\ttfamily \#}) for IDs in {\ttfamily selector}, those are not supported in JSX. \end{quote}


\begin{quote}
🪦 {\bfseries{Legacy}}\+: you can also use the classic JSX runtime, but this is not recommended. To do so, import {\ttfamily h} (or {\ttfamily s}) yourself and define it as the pragma (plus set the fragment to {\ttfamily null}). \end{quote}
