\doxysubsection{Detailed Description}
Given some data, {\itshape jsesc} returns a stringified representation of that data. jsesc is similar to {\ttfamily JSON.\+stringify()} except\+:


\begin{DoxyEnumerate}
\item it outputs Java\+Script instead of JSON \doxylink{calcwit_8hpp_ab701e3ac61a85b337ec5c1abaad6742d}{by default}, enabling support for data structures like ES6 maps and sets;
\item it offers \doxylink{README.md_api}{many options} to customize the output;
\item its output is ASCII-\/safe by default, thanks to its use of \href{https://mathiasbynens.be/notes/javascript-escapes}{\texttt{ escape sequences}} where needed.
\end{DoxyEnumerate}

For any input, jsesc generates the shortest possible valid printable-\/\+ASCII-\/only output. \href{https://mothereff.in/js-escapes}{\texttt{ Here‚Äôs an online demo.}}

jsesc‚Äôs output can be used instead of {\ttfamily JSON.\+stringify}‚Äôs to avoid \href{https://en.wikipedia.org/wiki/Mojibake}{\texttt{ mojibake}} and other encoding issues, or even to \href{https://twitter.com/annevk/status/380000829643571200}{\texttt{ avoid errors}} when passing JSON-\/formatted data (which may contain U+2028 LINE SEPARATOR, U+2029 PARAGRAPH SEPARATOR, or \href{https://esdiscuss.org/topic/code-points-vs-unicode-scalar-values\#content-14}{\texttt{ lone surrogates}}) to a Java\+Script parser or an UTF-\/8 encoder.\hypertarget{README.md_autotoc_md28670}{}\doxysubsection{\texorpdfstring{Installation}{Installation}}\label{README.md_autotoc_md28670}
