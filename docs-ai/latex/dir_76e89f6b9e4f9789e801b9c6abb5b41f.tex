The \href{https://atom.io/packages/linter-eslint}{\texttt{ {\ttfamily linter-\/eslint}}} package allows for linting within the \href{https://atom.io/}{\texttt{ Atom IDE}}.

In order to see {\ttfamily @eslint/markdown} work its magic within Markdown code blocks in your Atom editor, you can go to {\ttfamily linter-\/eslint}\textquotesingle{}s settings and within "{}\+List of scopes to run ESLint on..."{}, add the cursor scope "{}source.\+gfm"{}.

However, this reports a problem when viewing Markdown which does not have configuration, so you may wish to use the cursor scope "{}source.\+embedded.\+js"{}, but note that {\ttfamily @eslint/markdown} configuration comments and skip directives won\textquotesingle{}t work in this context.\hypertarget{README.md_autotoc_md3267}{}\doxysubsection{\texorpdfstring{File Name Details}{File Name Details}}\label{README.md_autotoc_md3267}
This processor will use file names from blocks if a {\ttfamily filename} meta is present.

For example, the following block will result in a parsed file name of {\ttfamily src/index.\+js}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\`{}\`{}\`{}js\ filename="{}src/index.js"{}}
\DoxyCodeLine{export\ const\ value\ =\ "{}Hello,\ world!"{};}
\DoxyCodeLine{\`{}\`{}\`{}}

\end{DoxyCode}


This can be useful for user configurations that include linting overrides for specific file paths.
