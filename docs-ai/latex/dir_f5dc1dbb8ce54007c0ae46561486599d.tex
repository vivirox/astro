{\bfseries{Opinionated bare simple logger for Node\+JS (with Type\+Script typings)}}.

BSLogger has been created after being disapointed not finding a matching logger on the internet. Not that others aren\textquotesingle{}t good, they just did not fit what I was looking for.

Here is what I was looking for (and tried to implemented in BSLogger)\+:
\begin{DoxyItemize}
\item light memory usage
\item easily extendable (see {\ttfamily child} method)
\item as few dependencies as possible
\item ability to define all targets in a string (so that {\ttfamily ENV} vars can be used)
\item when using file targets, not re-\/opening them
\item reasonable defautls\+:
\begin{DoxyItemize}
\item logs warnings and above to {\ttfamily stderr}
\item logs JSON to files
\end{DoxyItemize}
\item no overhead if it\textquotesingle{}s not going to log anywhere
\end{DoxyItemize}\hypertarget{README.md_autotoc_md9611}{}\doxysubsubsection{\texorpdfstring{TL,DR\+:}{TL,DR\+:}}\label{README.md_autotoc_md9611}

\hypertarget{README.md_autotoc_md9620}{}\doxysubsubsubsection{\texorpdfstring{Defining target(s)}{Defining target(s)}}\label{README.md_autotoc_md9620}
Each root logger (created using {\ttfamily create\+Logger} helper) is attached to 0 or more "{}target"{}. A target is responsible of writing a log entry somewhere. It is an object with the following properties\+:


\begin{DoxyItemize}
\item {\bfseries{min\+Level}} {\ttfamily string}\+: The minimum log level this target\textquotesingle{}s strem writer will be called for
\item {\bfseries{stream}} {\ttfamily \{ write\+: (str\+: string) =\texorpdfstring{$>$}{>} void \}}\+: An object with a write function (like node\textquotesingle{}s {\ttfamily stream.\+Writable}) which will be used to write log entries
\item {\bfseries{format}} {\ttfamily (msg\+: Log\+Message) =\texorpdfstring{$>$}{>} string}\+: A formatter which will be used to transform a log entry (message object) into a string
\end{DoxyItemize}\hypertarget{README.md_autotoc_md9621}{}\doxysubsubsubsubsection{\texorpdfstring{Using targets}{Using targets}}\label{README.md_autotoc_md9621}
When using the global logger, or if no {\ttfamily targets} specified when creating a logger, calling log methods will output to STDERR anything which has log level higher or equal to {\ttfamily warn}. This can be modified as follow by defineing the {\ttfamily LOG\+\_\+\+TARGETS} environment variable or passing the {\ttfamily targets} option to {\ttfamily create\+Logger}. The {\ttfamily targets} can be an array of {\ttfamily Log\+Target} (see above) or a {\ttfamily string} defining a list of one or more targets separated by comma ({\ttfamily ,}). A {\ttfamily string} target is composed as follow\+:
\begin{DoxyItemize}
\item The file path, absolute or relative to CWD. It can also be the specials {\ttfamily stdout} or {\ttfamily stderr} strings (case insensitive). When giving a path to a file, if it ends with the plus sign ({\ttfamily +}) the log data will be appended to the file instead of re-\/creating the file for each run.
\item An optional minimum log level after a colon ({\ttfamily \+:}). It should be a {\ttfamily number} or the log level name (ie {\ttfamily trace}, {\ttfamily error}, ...).
\item An optional formatter name after a percent sign ({\ttfamily \%}). There are 2 included formatter\+: {\ttfamily json} (used for files by default) and {\ttfamily simple} (used for {\ttfamily stdout} and {\ttfamily stderr} by default). See below to define your own.
\end{DoxyItemize}

Examples\+:
\begin{DoxyItemize}
\item {\ttfamily debug.\+logsimple,stdout\+:fatal}
\begin{DoxyItemize}
\item Log everything to {\ttfamily debug.\+log} file in CWD dir (re-\/creates the file for each run). Uses the {\ttfamily simple} formatter.
\item Log only messages with level \texorpdfstring{$>$}{>}= {\ttfamily fatal} to the standard out.
\end{DoxyItemize}
\item {\ttfamily errors.\+log+\+:error,debug.\+log\+:15}
\begin{DoxyItemize}
\item Log only messages with level \texorpdfstring{$>$}{>}= {\ttfamily error} to {\ttfamily errors.\+log} file (without re-\/creating the file at each run).
\item Log only messages with level \texorpdfstring{$>$}{>}= 15 to {\ttfamily debug.\+log} file (re-\/creates the file for each run).
\end{DoxyItemize}
\end{DoxyItemize}\hypertarget{README.md_autotoc_md9622}{}\doxysubsubsubsubsection{\texorpdfstring{Custom formatters}{Custom formatters}}\label{README.md_autotoc_md9622}

The whole {\ttfamily testing} namespace has useful helpers for using BSLogger while unit testing your product.

In your tests you would usually prefer not having any logging to happen, or you would like to check what has been logged but without actually logging it to any target.

The {\ttfamily testing} namespace holds all testing utilities\+: 
\begin{DoxyCode}{0}

\end{DoxyCode}



\begin{DoxyItemize}

\begin{DoxyCode}{0}
