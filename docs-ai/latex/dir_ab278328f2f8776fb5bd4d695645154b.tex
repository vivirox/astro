\doxysubsection{Detailed Description}
\begin{quote}
It\textquotesingle{}s a very fast and efficient \href{https://en.wikipedia.org/wiki/Glob_(programming)}{\texttt{ glob}} library for \href{https://nodejs.org/en}{\texttt{ Node.\+js}}. \end{quote}


This package provides methods for traversing the file system and returning pathnames that matched a defined set of a specified pattern according to the rules used by the Unix Bash shell with some simplifications, meanwhile results are returned in {\bfseries{arbitrary order}}. \begin{DoxyItemize}
\item Type\+: {\ttfamily number}
\item Default\+: {\ttfamily os.\+cpus().length}
\end{DoxyItemize}

Specifies the maximum number of concurrent requests from a reader to read directories.

\begin{quote}
\+:book\+: The higher the number, the higher the performance and load on the file system. If you want to read in quiet mode, set the value to a comfortable number or {\ttfamily 1}. \end{quote}




{\bfseries{More details}}\begin{adjustwidth}{1em}{0em}




In Node, there are \href{https://nodejs.org/en/docs/guides/dont-block-the-event-loop}{\texttt{ two types of threads}}\+: Event Loop (code) and a Thread Pool (fs, dns, …). The thread pool size controlled by the {\ttfamily UV\+\_\+\+THREADPOOL\+\_\+\+SIZE} environment variable. Its default size is 4 (\href{http://docs.libuv.org/en/v1.x/threadpool.html}{\texttt{ documentation}}). The pool is one for all tasks within a single Node process.

Any code can make 4 real concurrent accesses to the file system. The rest of the FS requests will wait in the queue.

\begin{quote}
\+:book\+: Each new instance of FG in the same Node process will use the same Thread pool. \end{quote}


But this package also has the {\ttfamily concurrency} option. This option allows you to control the number of concurrent accesses to the FS at the package level. By default, this package has a value equal to the number of cores available for the current Node process. This allows you to set a value smaller than the pool size ({\ttfamily concurrency\+: 1}) or, conversely, to prepare tasks for the pool queue more quickly ({\ttfamily concurrency\+: Number.\+POSITIVE\+\_\+\+INFINITY}).

So, in fact, this package can {\bfseries{only make 4 concurrent requests to the FS}}. You can increase this value by using an environment variable ({\ttfamily UV\+\_\+\+THREADPOOL\+\_\+\+SIZE}), but in practice this does not give a multiple advantage.

\end{adjustwidth}

\hypertarget{README.md_autotoc_md15172}{}\doxysubsection{\texorpdfstring{What is a static or dynamic pattern?}{What is a static or dynamic pattern?}}\label{README.md_autotoc_md15172}
All patterns can be divided into two types\+:


\begin{DoxyItemize}
\item {\bfseries{static}}. A pattern is considered static if it can be used to get an entry on the file system without using matching mechanisms. For example, the {\ttfamily file.\+js} pattern is a static pattern because we can just verify that it exists on the file system.
\item {\bfseries{dynamic}}. A pattern is considered dynamic if it cannot be used directly to find occurrences without using a matching mechanisms. For example, the {\ttfamily \texorpdfstring{$\ast$}{*}} pattern is a dynamic pattern because we cannot use this pattern directly.
\end{DoxyItemize}

A pattern is considered dynamic if it contains the following characters ({\ttfamily …} — any characters or their absence) or options\+:


\begin{DoxyItemize}
\item The \`{}case\+Sensitive\+Match\`{} option is disabled
\item {\ttfamily \textbackslash{}\textbackslash{}} (the escape character)
\item {\ttfamily \texorpdfstring{$\ast$}{*}}, {\ttfamily ?}, {\ttfamily !} (at the beginning of line)
\item {\ttfamily \mbox{[}…\mbox{]}}
\item {\ttfamily (…\texorpdfstring{$\vert$}{|}…)}
\item {\ttfamily @(…)}, {\ttfamily !(…)}, {\ttfamily \texorpdfstring{$\ast$}{*}(…)}, {\ttfamily ?(…)}, {\ttfamily +(…)} (respects the \`{}extglob\`{} option)
\item {\ttfamily \{…,…\}}, {\ttfamily \{…..…\}} (respects the \`{}brace\+Expansion\`{} option)
\end{DoxyItemize}\hypertarget{README.md_autotoc_md15173}{}\doxysubsection{\texorpdfstring{How to write patterns on Windows?}{How to write patterns on Windows?}}\label{README.md_autotoc_md15173}
