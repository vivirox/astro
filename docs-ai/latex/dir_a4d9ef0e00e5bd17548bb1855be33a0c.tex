This package contains two extensions that add support for directive syntax in markdown to \href{https://github.com/syntax-tree/mdast}{\texttt{ mdast}}. These extensions plug into \href{https://github.com/syntax-tree/mdast-util-from-markdown}{\texttt{ {\ttfamily mdast-\/util-\/from-\/markdown}}} (to support parsing directives in markdown into a syntax tree) and \href{https://github.com/syntax-tree/mdast-util-to-markdown}{\texttt{ {\ttfamily mdast-\/util-\/to-\/markdown}}} (to support serializing directives in syntax trees to markdown).\hypertarget{readme.md_autotoc_md22264}{}\doxysubsection{\texorpdfstring{When to use this}{When to use this}}\label{readme.md_autotoc_md22264}
Directives are one of the four ways to extend markdown\+: an arbitrary extension syntax (see \href{https://github.com/micromark/micromark\#extending-markdown}{\texttt{ Extending markdown}} in micromark’s docs for the alternatives and more info). This mechanism works well when you control the content\+: who authors it, what tools handle it, and where it’s displayed. When authors can read a guide on how to embed a tweet but are not expected to know the ins and outs of HTML or Java\+Script. Directives don’t work well if you don’t know who authors content, what tools handle it, and where it ends up. Example use cases are a docs website for a project or product, or blogging tools and static site generators.

You can use these extensions when you are working with {\ttfamily mdast-\/util-\/from-\/markdown} and {\ttfamily mdast-\/util-\/to-\/markdown} already.

When working with {\ttfamily mdast-\/util-\/from-\/markdown}, you must combine this package with \href{https://github.com/micromark/micromark-extension-directive}{\texttt{ {\ttfamily micromark-\/extension-\/directive}}}.

When you don’t need a syntax tree, you can use \href{https://github.com/micromark/micromark}{\texttt{ {\ttfamily micromark}}} directly with {\ttfamily micromark-\/extension-\/directive}.

All these packages are used \href{https://github.com/remarkjs/remark-directive}{\texttt{ {\ttfamily remark-\/directive}}}, which focusses on making it easier to transform content by abstracting these internals away.

This package only handles the syntax tree. For example, it does not handle how markdown is turned to HTML. You can use this with some more code to match your specific needs, to allow for anything from callouts, citations, styled blocks, forms, embeds, spoilers, etc. \href{https://unifiedjs.com/learn/recipe/tree-traversal/}{\texttt{ Traverse the tree}} to change directives to whatever you please.\hypertarget{readme.md_autotoc_md22265}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md22265}

{\bfseries{Directive}} represents something defined by an extension.

The {\ttfamily name} field must be present and represents an identifier of an extension.

The {\ttfamily attributes} field represents information associated with the node. The value of the {\ttfamily attributes} field implements the {\bfseries{Attributes}} interface.

In the {\bfseries{Attributes}} interface, every field must be an {\ttfamily Attribute\+Name} and every value an {\ttfamily Attribute\+Value}. The fields and values can be anything\+: there are no semantics (such as by HTML or hast).

\begin{quote}
In JSON, the value {\ttfamily null} must be treated as if the attribute was not included. In Java\+Script, both {\ttfamily null} and {\ttfamily undefined} must be similarly ignored. \end{quote}

Projects maintained by the unified collective are compatible with maintained versions of Node.\+js.

When we cut a new major release, we drop support for unmaintained versions of Node. This means we try to keep the current release line, {\ttfamily mdast-\/util-\/directive@\texorpdfstring{$^\wedge$}{\string^}3}, compatible with Node.\+js 16.
