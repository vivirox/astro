\item {\ttfamily Config\+Comment\+Parser} -\/ used to parse ESLint configuration comments (i.\+e., {\ttfamily /\texorpdfstring{$\ast$}{*} eslint-\/disable rule \texorpdfstring{$\ast$}{*}/})

\hypertarget{README.md_autotoc_md3287}{}\doxysubsubsection{\texorpdfstring{{\ttfamily Visit\+Node\+Step} and {\ttfamily Call\+Method\+Step}}{{\ttfamily Visit\+Node\+Step} and {\ttfamily Call\+Method\+Step}}}\label{README.md_autotoc_md3287}
The {\ttfamily Visit\+Node\+Step} and {\ttfamily Call\+Method\+Step} classes represent steps in the traversal of source code. They implement the correct interfaces to return from the {\ttfamily Source\+Code\#traverse()} method.

The {\ttfamily Visit\+Node\+Step} class is the more common of the two, where you are describing a visit to a particular node during the traversal. The constructor accepts three arguments\+:


\begin{DoxyItemize}
\item {\ttfamily target} -\/ the node being visited. This is used to determine the method to call inside of a rule. For instance, if the node\textquotesingle{}s type is {\ttfamily Literal} then ESLint will call a method named {\ttfamily Literal()} on the rule (if present).
\item {\ttfamily phase} -\/ either 1 for enter or 2 for exit.
\item {\ttfamily args} -\/ an array of arguments to pass into the visitor method of a rule.
\end{DoxyItemize}

For example\+:

\hypertarget{README.md_autotoc_md3289}{}\doxysubsubsection{\texorpdfstring{{\ttfamily Text\+Source\+Code\+Base}}{{\ttfamily Text\+Source\+Code\+Base}}}\label{README.md_autotoc_md3289}
The {\ttfamily Text\+Source\+Code\+Base} class is intended to be a base class that has several of the common members found in {\ttfamily Source\+Code} objects already implemented. Those members are\+:


\begin{DoxyItemize}
\item {\ttfamily lines} -\/ an array of text lines that is created automatically when the constructor is called.
\item {\ttfamily get\+Loc(node)} -\/ gets the location of a node. Works for nodes that have the ESLint-\/style {\ttfamily loc} property and nodes that have the Unist-\/style \href{https://github.com/syntax-tree/unist?tab=readme-ov-file\#position}{\texttt{ {\ttfamily position} property}}. If you\textquotesingle{}re using an AST with a different location format, you\textquotesingle{}ll still need to implement this method yourself.
\item {\ttfamily get\+Range(node)} -\/ gets the range of a node within the source text. Works for nodes that have the ESLint-\/style {\ttfamily range} property and nodes that have the Unist-\/style \href{https://github.com/syntax-tree/unist?tab=readme-ov-file\#position}{\texttt{ {\ttfamily position} property}}. If you\textquotesingle{}re using an AST with a different range format, you\textquotesingle{}ll still need to implement this method yourself.
\item {\ttfamily get\+Text(node\+Or\+Token, chars\+Before, chars\+After)} -\/ gets the source text for the given node or token that has range information attached. Optionally, can return additional characters before and after the given node or token. As long as {\ttfamily get\+Range()} is properly implemented, this method will just work.
\item {\ttfamily get\+Ancestors(node)} -\/ returns the ancestry of the node. In order for this to work, you must implement the {\ttfamily get\+Parent()} method yourself.
\end{DoxyItemize}
