This package lets you find nodes in a tree, similar to how {\ttfamily matches}, {\ttfamily query\+Selector}, and {\ttfamily query\+Selector\+All} work with the DOM.

One notable difference between DOM and hast is that DOM nodes have references to their parents, meaning that `document.\+body.\+matches('\+:last-\/child\textquotesingle{})\`{} can be evaluated to check whether the body is the last child of its parent. This information is not stored in hast, so selectors like that don’t work.\hypertarget{readme.md_autotoc_md17232}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{readme.md_autotoc_md17232}
This is a small utility that is quite useful, but is rather slow if you use it a lot. For each call, it has to walk the entire tree. In some cases, walking the tree once with \href{https://github.com/syntax-tree/unist-util-visit}{\texttt{ {\ttfamily unist-\/util-\/visit}}} is smarter, such as when you want to change certain nodes. On the other hand, this is quite powerful and fast enough for many other cases.

This utility is similar to \href{https://github.com/syntax-tree/unist-util-select}{\texttt{ {\ttfamily unist-\/util-\/select}}}, which can find and match any unist node.\hypertarget{readme.md_autotoc_md17233}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md17233}
This package is \href{https://gist.github.com/sindresorhus/a39789f98801d908bbc7ff3ecc99d99c}{\texttt{ ESM only}}. In Node.\+js (version 16+), install with \href{https://docs.npmjs.com/cli/install}{\texttt{ npm}}\+:

Check that the given {\ttfamily node} matches {\ttfamily selector}.

This only checks the element itself, not the surrounding tree. Thus, nesting in selectors is not supported ({\ttfamily p b}, {\ttfamily p \texorpdfstring{$>$}{>} b}), neither are selectors like {\ttfamily \+:first-\/child}, etc. This only checks that the given element matches the selector.\hypertarget{readme.md_autotoc_md17237}{}\doxysubsubsubsubsubsection{\texorpdfstring{Parameters}{Parameters}}\label{readme.md_autotoc_md17237}
