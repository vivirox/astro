\hypertarget{README.md_autotoc_md8334}{}\doxysubsubsection{\texorpdfstring{Why astro-\/robots-\/txt?}{Why astro-\/robots-\/txt?}}\label{README.md_autotoc_md8334}
The {\itshape robots.\+txt} file informs search engines which pages on your website should be crawled. \href{https://developers.google.com/search/docs/advanced/robots/intro}{\texttt{ See Google\textquotesingle{}s own advice on robots.\+txt}} to learn more.

For Astro project you usually create the {\itshape robots.\+txt} in a text editor and place it to the {\ttfamily public/} directory. In that case you must manually synchronize {\ttfamily site} option in {\itshape astro.\+config.\texorpdfstring{$\ast$}{*}} with {\ttfamily Sitemap\+:} record in {\itshape robots.\+txt}. ~\newline
 It brakes DRY principle.

Sometimes, especially during development, it\textquotesingle{}s necessary to prevent your site from being indexed. To achieve this you need to place the meta tag {\ttfamily \texorpdfstring{$<$}{<}meta name="{}robots"{} content="{}noindex"{}\texorpdfstring{$>$}{>}} into the {\ttfamily \texorpdfstring{$<$}{<}head\texorpdfstring{$>$}{>}} section of your pages or add {\ttfamily X-\/\+Robots-\/\+Tag\+: noindex} to the HTTP response header, then add the lines {\ttfamily User-\/agent\+: \texorpdfstring{$\ast$}{*}} and {\ttfamily Disallow\+: \textbackslash{}} to {\itshape robots.\+txt}. ~\newline
 Again you have to do it manually in two different places.

{\bfseries{astro-\/robots-\/txt}} can help in both cases on the {\itshape robots.\+txt} side. See details in this demo \href{https://github.com/alextim/astro-lib/tree/main/examples/robots-txt/advanced}{\texttt{ repo}}.

{\bfseries{Quick Install}}\begin{adjustwidth}{1em}{0em}


The experimental {\ttfamily astro add} command-\/line tool automates the installation for you. Run one of the following commands in a new terminal window. (If you aren\textquotesingle{}t sure which package manager you\textquotesingle{}re using, run the first command.) Then, follow the prompts, and type "{}y"{} in the terminal (meaning "{}yes"{}) for each one.


\begin{DoxyCode}{0}
\DoxyCodeLine{\#\ Using\ NPM}
\DoxyCodeLine{npx\ astro\ add\ astro-\/robots-\/txt}
\DoxyCodeLine{}
\DoxyCodeLine{\#\ Using\ Yarn}
\DoxyCodeLine{yarn\ astro\ add\ astro-\/robots-\/txt}
\DoxyCodeLine{}
\DoxyCodeLine{\#\ Using\ PNPM}
\DoxyCodeLine{pnpx\ astro\ add\ astro-\/robots-\/txt}

\end{DoxyCode}


Then, restart the dev server by typing {\ttfamily CTRL-\/C} and then {\ttfamily npm run astro dev} in the terminal window that was running Astro.

Because this command is new, it might not properly set things up. If that happens, \href{https://github.com/withastro/astro/issues}{\texttt{ log an issue on Astro Git\+Hub}} and try the manual installation steps below.

\hypertarget{README.md_autotoc_md8337}{}\doxysubsubsection{\texorpdfstring{Usage}{Usage}}\label{README.md_autotoc_md8337}
The {\ttfamily astro-\/robots-\/txt} integration requires a deployment / site URL for generation. Add your site\textquotesingle{}s URL under your {\itshape astro.\+config.\texorpdfstring{$\ast$}{*}} using the {\ttfamily site} property.

Then, apply this integration to your {\itshape astro.\+config.\texorpdfstring{$\ast$}{*}} file using the {\ttfamily integrations} property.

{\bfseries{{\ttfamily astro.\+config.\+mjs}}}


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ \{\ defineConfig\ \}\ from\ 'astro/config';}
\DoxyCodeLine{import\ robotsTxt\ from\ 'astro-\/robots-\/txt';}
\DoxyCodeLine{}
\DoxyCodeLine{export\ default\ defineConfig(\{}
\DoxyCodeLine{\ \ site:\ 'https://example.com',}
\DoxyCodeLine{}
\DoxyCodeLine{\ \ integrations:\ [robotsTxt()],}
\DoxyCodeLine{\});}

\end{DoxyCode}


Note that unlike other configuration options, {\ttfamily site} is set in the root {\ttfamily define\+Config} object, rather than inside the {\ttfamily robots\+Txt()} call. ~\newline


Now, \href{https://docs.astro.build/en/reference/cli-reference/\#astro-build}{\texttt{ build your site for production}} via the {\ttfamily astro build} command. You should find your {\itshape robots.\+txt} under {\ttfamily dist/robots.\+txt}!

\begin{quote}
{\bfseries{Warning}} If you forget to add a {\ttfamily site}, you\textquotesingle{}ll get a friendly warning when you build, and the {\ttfamily robots.\+txt} file won\textquotesingle{}t be generated. \end{quote}
