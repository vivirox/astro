Advanced converter that parses HTML and returns beautiful text.\hypertarget{README.md_autotoc_md17831}{}\doxysubsection{\texorpdfstring{Features}{Features}}\label{README.md_autotoc_md17831}

\begin{DoxyItemize}
\item Inline and block-\/level tags.
\item Tables with colspans and rowspans.
\item Links with both text and href.
\item Word wrapping.
\item Unicode support.
\item Plenty of customization options.
\end{DoxyItemize}\hypertarget{README.md_autotoc_md17832}{}\doxysubsection{\texorpdfstring{Changelog}{Changelog}}\label{README.md_autotoc_md17832}

Version 7 contains an important change for custom formatters.

Version 8 brings the selectors support to greatly increase the flexibility but that also changes some things introduced in version 6. Base element(s) selection also got important changes.

Version 9 drops a lot of previously deprecated options, introduces some new formatters and new capabilities for custom formatters. Now a dual-\/mode package (cjs and esm). Selectors array is our loose approximation of a stylesheet.


\begin{DoxyItemize}
\item highest \href{https://www.w3.org/TR/selectors/\#specificity}{\texttt{ specificity}} selector is used when there are multiple matches;
\item the last selector is used when there are multiple matches of equal specificity;
\item all entries with the same selector value are merged (recursively) at the compile stage, in such way so the last defined properties a kept and the relative order of unique selectors is kept;
\item user-\/defined entries are appended after predefined entries;
\item Every unique selector must have {\ttfamily format} value specified (at least once);
\item unlike in CSS, values from different matched selectors are NOT merged at the convert stage. Single best match is used instead (that is the last one of those with highest specificity).
\end{DoxyItemize}

To achieve the best performance when checking each DOM element against provided selectors, they are compiled into a decision tree. But it is also important how you choose selectors. For example, {\ttfamily div\#id} is much better than {\ttfamily \#id} -\/ the former will only check divs for the id while the latter has to check every element in the DOM.\hypertarget{README.md_autotoc_md17839}{}\doxysubsubsubsubsection{\texorpdfstring{Supported selectors}{Supported selectors}}\label{README.md_autotoc_md17839}
{\ttfamily html-\/to-\/text} relies on \href{https://github.com/mxxii/parseley}{\texttt{ parseley}} and \href{https://github.com/mxxii/selderee}{\texttt{ selderee}} packages for selectors support.

Following selectors can be used in any combinations\+:


\begin{DoxyItemize}
\item {\ttfamily \texorpdfstring{$\ast$}{*}} -\/ universal selector;
\item {\ttfamily div} -\/ tag name;
\item {\ttfamily .foo} -\/ class name;
\item {\ttfamily \#bar} -\/ id;
\item {\ttfamily \mbox{[}baz\mbox{]}} -\/ attribute presence;
\item {\ttfamily \mbox{[}baz=buzz\mbox{]}} -\/ attribute value (with any operators and also quotes and case sensitivity modifiers -\/ \href{https://developer.mozilla.org/en-US/docs/Web/CSS/Attribute_selectors\#syntax}{\texttt{ syntax}});
\item {\ttfamily +} and {\ttfamily \texorpdfstring{$>$}{>}} combinators (other combinators are not supported).
\end{DoxyItemize}

You can match {\ttfamily \texorpdfstring{$<$}{<}p style="{}...; display\+:\+INLINE; ..."{}\texorpdfstring{$>$}{>}...\texorpdfstring{$<$}{<}/p\texorpdfstring{$>$}{>}} with {\ttfamily p\mbox{[}style\texorpdfstring{$\ast$}{*}="{}display\+:inline"{}i\mbox{]}} for example.\hypertarget{README.md_autotoc_md17840}{}\doxysubsubsubsubsection{\texorpdfstring{Predefined formatters}{Predefined formatters}}\label{README.md_autotoc_md17840}
