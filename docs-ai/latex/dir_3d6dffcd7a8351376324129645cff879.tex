This package is a \href{https://github.com/unifiedjs/unified}{\texttt{ unified}} processor with support for parsing markdown as input and serializing markdown as output by using unified with \href{../remark-parse}{\texttt{ {\ttfamily remark-\/parse}}} and \href{../remark-stringify}{\texttt{ {\ttfamily remark-\/stringify}}}.

See \href{https://github.com/remarkjs/remark}{\texttt{ the monorepo readme}} for info on what the remark ecosystem is.\hypertarget{readme.md_autotoc_md29305}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{readme.md_autotoc_md29305}
You can use this package when you want to use unified, have markdown as input, and want markdown as output. This package is a shortcut for {\ttfamily unified().use(remark\+Parse).use(remark\+Stringify)}. When the input isn’t markdown (meaning you don’t need {\ttfamily remark-\/parse}) or the output is not markdown (you don’t need {\ttfamily remark-\/stringify}), it’s recommended to use {\ttfamily unified} directly.

When you want to inspect and format markdown files in a project on the command line, you can use \href{../remark-cli}{\texttt{ {\ttfamily remark-\/cli}}}.\hypertarget{readme.md_autotoc_md29306}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md29306}

\hypertarget{readme.md_autotoc_md29316}{}\doxysubsection{\texorpdfstring{Compatibility}{Compatibility}}\label{readme.md_autotoc_md29316}
Projects maintained by the unified collective are compatible with maintained versions of Node.\+js.

When we cut a new major release, we drop support for unmaintained versions of Node. This means we try to keep the current release line, {\ttfamily remark@\texorpdfstring{$^\wedge$}{\string^}15}, compatible with Node.\+js 16.\hypertarget{readme.md_autotoc_md29317}{}\doxysubsection{\texorpdfstring{Security}{Security}}\label{readme.md_autotoc_md29317}
As markdown can be turned into HTML and improper use of HTML can open you up to \href{https://en.wikipedia.org/wiki/Cross-site_scripting}{\texttt{ cross-\/site scripting (XSS)}} attacks, use of remark can be unsafe. When going to HTML, you will combine remark with {\bfseries{\href{https://github.com/rehypejs/rehype}{\texttt{ rehype}}}}, in which case you should use \href{https://github.com/rehypejs/rehype-sanitize}{\texttt{ {\ttfamily rehype-\/sanitize}}}.

Use of remark plugins could also open you up to other attacks. Carefully assess each plugin and the risks involved in using them.
