\doxysubsection*{  jest-\/circus The next-\/gen test runner for Jest }\hypertarget{README.md_autotoc_md19627}{}\doxysubsubsection{\texorpdfstring{Overview}{Overview}}\label{README.md_autotoc_md19627}
Circus is a flux-\/based test runner for Jest that is fast, maintainable, and simple to extend.

Circus allows you to bind to events via an optional event handler on any \href{https://jestjs.io/docs/configuration\#testenvironment-string}{\texttt{ custom environment}}. See the \href{https://github.com/jestjs/jest/blob/main/packages/jest-types/src/Circus.ts}{\texttt{ type definitions}} for more information on the events and state data currently available.

Mutating event or state data is currently unsupported and may cause unexpected behavior or break in a future release without warning. New events, event data, and/or state data will not be considered a breaking change and may be added in any minor release.

Note, that {\ttfamily jest-\/circus} test runner would pause until a promise returned from {\ttfamily handle\+Test\+Event} gets fulfilled. {\bfseries{However, there are a few events that do not conform to this rule, namely}}\+: {\ttfamily start\+\_\+describe\+\_\+definition}, {\ttfamily finish\+\_\+describe\+\_\+definition}, {\ttfamily add\+\_\+hook}, {\ttfamily add\+\_\+test} or {\ttfamily error} (for the up-\/to-\/date list you can look at \href{https://github.com/jestjs/jest/blob/main/packages/jest-types/src/Circus.ts}{\texttt{ Sync\+Event type in the types definitions}}). That is caused by backward compatibility reasons and `process.\+on(\textquotesingle{}unhandled\+Rejection', callback)\`{} signature, but that usually should not be a problem for most of the use cases.\hypertarget{README.md_autotoc_md19628}{}\doxysubsubsection{\texorpdfstring{Installation}{Installation}}\label{README.md_autotoc_md19628}
