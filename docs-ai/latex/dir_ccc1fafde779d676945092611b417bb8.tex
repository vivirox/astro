{\ttfamily npm install react-\/syntax-\/highlighter -\/-\/save}\hypertarget{README.md_autotoc_md27307}{}\doxysubsubsection{\texorpdfstring{Why This One?}{Why This One?}}\label{README.md_autotoc_md27307}
There are other syntax highlighters for {\ttfamily React} out there so why use this one? The biggest reason is that all the others rely on triggering calls in {\ttfamily component\+Did\+Mount} and {\ttfamily component\+Did\+Update} to highlight the code block and then insert it in the render function using {\ttfamily dangerously\+Set\+Inner\+HTML} or just manually altering the DOM with native javascript. This utilizes a syntax tree to dynamically build the virtual dom which allows for updating only the changing DOM instead of completely overwriting it on any change, and because of this it is also using more idiomatic {\ttfamily React} and allows the use of pure function components brought into {\ttfamily React} as of {\ttfamily 0.\+14}.\hypertarget{README.md_autotoc_md27308}{}\doxysubsubsection{\texorpdfstring{Javascript Styles!}{Javascript Styles!}}\label{README.md_autotoc_md27308}
One of the biggest pain points for me trying to find a syntax highlighter for my own projects was the need to put a stylesheet tag on my page. I wanted to provide out of the box code styling with my modules without requiring awkward inclusion of another libs stylesheets. The styles in this module are all javascript based, and all styles supported by {\ttfamily highlight.\+js} have been ported!

I do realize that javascript styles are not for everyone, so you can optionally choose to use css based styles with class\+Names added to elements by setting the prop {\ttfamily use\+Inline\+Styles} to {\ttfamily false} (it defaults to {\ttfamily true}).\hypertarget{README.md_autotoc_md27309}{}\doxysubsubsection{\texorpdfstring{Use}{Use}}\label{README.md_autotoc_md27309}
