\hypertarget{README.md_autotoc_md31739}{}\doxysubsection{\texorpdfstring{CSSTree}{CSSTree}}\label{README.md_autotoc_md31739}
\href{https://www.npmjs.com/package/css-tree}{\texttt{ }} \href{https://github.com/csstree/csstree/actions/workflows/build.yml}{\texttt{ }} \href{https://coveralls.io/github/csstree/csstree?branch=master}{\texttt{ }} \href{https://www.npmjs.com/package/css-tree}{\texttt{ }} \href{https://twitter.com/csstree}{\texttt{ }}

CSSTree is a tool set for CSS\+: \href{https://github.com/postcss/benchmark}{\texttt{ fast}} detailed parser (CSS → AST), walker (AST traversal), generator (AST → CSS) and lexer (validation and matching) based on specs and browser implementations. The main goal is to be efficient and W3C spec compliant, with focus on CSS analyzing and source-\/to-\/source transforming tasks.\hypertarget{README.md_autotoc_md31740}{}\doxysubsubsection{\texorpdfstring{Features}{Features}}\label{README.md_autotoc_md31740}

\begin{DoxyItemize}
\item {\bfseries{Detailed parsing with an adjustable level of detail}}

By default CSSTree parses CSS as detailed as possible, i.\+e. each single logical part is representing with its own AST node (see AST format for all possible node types). The parsing detail level can be changed through \href{docs/parsing.md\#parsesource-options}{\texttt{ parser options}}, for example, you can disable parsing of selectors or declaration values for component parts.
\item {\bfseries{Tolerant to errors by design}}

Parser behaves as \href{https://www.w3.org/TR/css-syntax-3/\#error-handling}{\texttt{ spec says}}\+: "{}\+When errors occur in CSS, the parser attempts to recover gracefully, throwing away only the minimum amount of content before returning to parsing as normal"{}. The only thing the parser departs from the specification is that it doesn\textquotesingle{}t throw away bad content, but wraps it in a special node type ({\ttfamily Raw}) that allows processing it later.
\item {\bfseries{Fast and efficient}}

CSSTree is created with focus on performance and effective memory consumption. Therefore it\textquotesingle{}s \href{https://github.com/postcss/benchmark}{\texttt{ one of the fastest CSS parsers}} at the moment.
\item {\bfseries{Syntax validation}}

The build-\/in lexer can test CSS against syntaxes defined by W3C. CSSTree uses \href{https://github.com/mdn/data/}{\texttt{ mdn/data}} as a basis for lexer\textquotesingle{}s dictionaries and extends it with vendor specific and legacy syntaxes. Lexer can only check the declaration values currently, but this feature will be extended to other parts of the CSS in the future.
