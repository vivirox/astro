\doxysubsection{Detailed Description}
\href{https://npmjs.com/package/@antfu/eslint-config}{\texttt{ }} \href{https://github.com/antfu/eslint-config}{\texttt{ }}


\begin{DoxyItemize}
\item Auto fix for formatting (aimed to be used standalone {\bfseries{without}} Prettier)
\item Reasonable defaults, best practices, only one line of config
\item Designed to work with Type\+Script, JSX, Vue, JSON, YAML, Toml, Markdown, etc. Out-\/of-\/box.
\item Opinionated, but very customizable
\item \href{https://eslint.org/docs/latest/use/configure/configuration-files-new}{\texttt{ ESLint Flat config}}, compose easily!
\item Optional React, Svelte, Uno\+CSS, Astro, Solid support
\item Optional formatters support for formatting CSS, HTML, XML, etc.
\item {\bfseries{Style principle}}\+: Minimal for reading, stable for diff, consistent
\begin{DoxyItemize}
\item Sorted imports, dangling commas
\item Single quotes, no semi
\item Using \href{https://github.com/eslint-stylistic/eslint-stylistic}{\texttt{ ESLint Stylistic}}
\end{DoxyItemize}
\item Respects {\ttfamily .gitignore} by default
\item Requires ESLint v9.\+5.\+0+
\end{DoxyItemize}

\begin{DoxyNote}{Note}
Since v1.\+0.\+0, this config is rewritten to the new \href{https://eslint.org/docs/latest/use/configure/configuration-files-new}{\texttt{ ESLint Flat config}}, check the \href{https://github.com/antfu/eslint-config/releases/tag/v1.0.0}{\texttt{ release note}} for more details. ~\newline

 Since v3.\+0.\+0, ESLint v9.\+5.\+0+ is now required.

\begin{DoxyNote}{Note}
About plugin renaming -\/ it is actually rather a dangrous move that might leading to potential naming collisions, pointed out \href{https://github.com/eslint/eslint/discussions/17766}{\texttt{ here}} and \href{https://github.com/prettier/eslint-config-prettier\#eslintconfigjs-flat-config-plugin-caveat}{\texttt{ here}}. As this config also very {\bfseries{personal}} and {\bfseries{opinionated}}, I ambitiously position this config as the only {\bfseries{"{}top-\/level"{}}} config per project, that might pivots the taste of how rules are named. ~\newline
~\newline
 This config cares more about the user-\/facings DX, and try to ease out the implementation details. For example, users could keep using the semantic {\ttfamily import/order} without ever knowing the underlying plugin has migrated twice to {\ttfamily eslint-\/plugin-\/i} and then to {\ttfamily eslint-\/plugin-\/import-\/x}. User are also not forced to migrate to the implicit {\ttfamily i/order} halfway only because we swapped the implementation to a fork. ~\newline
~\newline
 That said, it\textquotesingle{}s probably still not a good idea. You might not want to doing this if you are maintaining your own eslint config. ~\newline
~\newline
 Feel free to open issues if you want to combine this config with some other config presets but faced naming collisions. I am happy to figure out a way to make them work. But at this moment I have no plan to revert the renaming.
\end{DoxyNote}

\href{https://dprint.dev/}{\texttt{ dprint}} is also a great formatter that with more abilities to customize. However, it\textquotesingle{}s in the same model as Prettier which reads the AST and reprints the code from scratch. So in general, we prefer to use ESLint to format and lint Java\+Script/\+Type\+Script code.
