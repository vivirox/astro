This package is a \href{https://github.com/unifiedjs/unified}{\texttt{ unified}} (\href{https://github.com/remarkjs/remark}{\texttt{ remark}}) plugin to generate a table of contents of the document such as the one above.\hypertarget{readme.md_autotoc_md29282}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{readme.md_autotoc_md29282}
This project is useful when authors are writing docs in markdown that are sometimes quite long and so would benefit from automated overviews inside them. It is assumed that headings define the structure of documents and that they can be linked to. When this plugin is used, authors can add a certain heading (say, {\ttfamily \#\# Contents}) to documents and this plugin will populate those sections with lists that link to all following sections.

Git\+Hub and similar services automatically add IDs (and anchors that link-\/to-\/self) to headings. You can add similar features when combining remark with \href{https://github.com/rehypejs/rehype}{\texttt{ rehype}} through \href{https://github.com/remarkjs/remark-rehype}{\texttt{ {\ttfamily remark-\/rehype}}} after this plugin. Then it’s possible to use the rehype plugins \href{https://github.com/rehypejs/rehype-slug}{\texttt{ {\ttfamily rehype-\/slug}}} (for IDs on headings) and \href{https://github.com/rehypejs/rehype-autolink-headings}{\texttt{ {\ttfamily rehype-\/autolink-\/headings}}} (for anchors that link-\/to-\/self).

This plugin does not generate a table of contents for the {\itshape whole} document or expose it to other plugins. You can use the underlying mdast utility \href{https://github.com/syntax-tree/mdast-util-toc}{\texttt{ {\ttfamily mdast-\/util-\/toc}}} and \href{https://unifiedjs.com/learn/guide/create-a-plugin/}{\texttt{ create a plugin}} yourself to do that and more.\hypertarget{readme.md_autotoc_md29283}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md29283}

It exports the additional type \doxylink{README.md_options}{\`{}\+Options\`{}}.\hypertarget{readme.md_autotoc_md29297}{}\doxysubsection{\texorpdfstring{Compatibility}{Compatibility}}\label{readme.md_autotoc_md29297}
Projects maintained by the unified collective are compatible with maintained versions of Node.\+js.

When we cut a new major release, we drop support for unmaintained versions of Node. This means we try to keep the current release line, {\ttfamily remark-\/toc@\texorpdfstring{$^\wedge$}{\string^}9}, compatible with Node.\+js 16.

This plugin works with {\ttfamily unified} version 3+ and {\ttfamily remark} version 4+.\hypertarget{readme.md_autotoc_md29298}{}\doxysubsection{\texorpdfstring{Security}{Security}}\label{readme.md_autotoc_md29298}
Use of {\ttfamily remark-\/toc} involves user content and changes the tree, so it can open you up for a \href{https://en.wikipedia.org/wiki/Cross-site_scripting}{\texttt{ cross-\/site scripting (XSS)}} attack.

Existing nodes are copied into the table of contents. The following example shows how an existing script is copied into the table of contents.

The following markdown\+:
