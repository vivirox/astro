\hypertarget{README.md_autotoc_md5171}{}\doxysubsection{\texorpdfstring{How it works}{How it works}}\label{README.md_autotoc_md5171}
Git\+Hub\textquotesingle{}s OAuth Device flow is different from the web flow in two ways


\begin{DoxyEnumerate}
\item It does not require a URL redirect, which makes it great for devices and CLI apps
\item It does not require the OAuth client secret, which means there is no user-\/owned server component required.
\end{DoxyEnumerate}

The flow has 3 parts (see \href{https://docs.github.com/en/developers/apps/authorizing-oauth-apps\#device-flow}{\texttt{ Git\+Hub documentation}})


\begin{DoxyEnumerate}
\item {\ttfamily @octokit/auth-\/oauth-\/device} requests a device and user code
\item Then the user has to open \href{https://github.com/login/device}{\texttt{ https\+://github.\+com/login/device}} (or it\textquotesingle{}s Git\+Hub Enterprise Server equivalent) and enter the user code
\item While the user enters the code, {\ttfamily @octokit/auth-\/oauth-\/device} is sending requests in the background to retrieve the OAuth access token. Once the user completed step 2, the request will succeed and the token will be returned
\end{DoxyEnumerate}\hypertarget{README.md_autotoc_md5172}{}\doxysubsection{\texorpdfstring{Contributing}{Contributing}}\label{README.md_autotoc_md5172}
