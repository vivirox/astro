!\mbox{[}Satori\mbox{]}(.github/card.\+png)

{\bfseries{Satori}}\+: Enlightened library to convert HTML and CSS to SVG.

\begin{quote}
{\bfseries{Note}}

To use Satori in your project to generate PNG images like Open Graph images and social cards, check out our \href{https://vercel.com/blog/introducing-vercel-og-image-generation-fast-dynamic-social-card-images}{\texttt{ announcement}} and \href{https://vercel.com/docs/concepts/functions/edge-functions/og-image-generation}{\texttt{ Vercel’s Open Graph Image Generation →}}

To use it in Next.\+js, take a look at the \href{https://vercel.com/templates/next.js/og-image-generation}{\texttt{ Next.\+js Open Graph Image Generation template →}} \end{quote}
\hypertarget{README.md_autotoc_md30251}{}\doxysubsubsection{\texorpdfstring{Overview}{Overview}}\label{README.md_autotoc_md30251}
Satori supports the JSX syntax, which makes it very straightforward to use. Here’s an overview of the basic usage\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{//\ api.jsx}
\DoxyCodeLine{import\ satori\ from\ 'satori'}
\DoxyCodeLine{}
\DoxyCodeLine{const\ svg\ =\ await\ satori(}
\DoxyCodeLine{\ \ <div\ style=\{\{\ color:\ 'black'\ \}\}>hello,\ world</div>,}
\DoxyCodeLine{\ \ \{}
\DoxyCodeLine{\ \ \ \ width:\ 600,}
\DoxyCodeLine{\ \ \ \ height:\ 400,}
\DoxyCodeLine{\ \ \ \ fonts:\ [}
\DoxyCodeLine{\ \ \ \ \ \ \{}
\DoxyCodeLine{\ \ \ \ \ \ \ \ name:\ 'Roboto',}
\DoxyCodeLine{\ \ \ \ \ \ \ \ //\ Use\ \`{}fs`\ (Node.js\ only)\ or\ \`{}fetch`\ to\ read\ the\ font\ as\ Buffer/ArrayBuffer\ and\ provide\ \`{}data`\ here.}
\DoxyCodeLine{\ \ \ \ \ \ \ \ data:\ robotoArrayBuffer,}
\DoxyCodeLine{\ \ \ \ \ \ \ \ weight:\ 400,}
\DoxyCodeLine{\ \ \ \ \ \ \ \ style:\ 'normal',}
\DoxyCodeLine{\ \ \ \ \ \ \},}
\DoxyCodeLine{\ \ \ \ ],}
\DoxyCodeLine{\ \ \},}
\DoxyCodeLine{)}

\end{DoxyCode}


Satori will render the element into a 600×400 SVG, and return the SVG string\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{'<svg\ ...><path\ d="{}..."{}\ fill="{}black"{}></path></svg>'}

\end{DoxyCode}


Under the hood, it handles layout calculation, font, typography and more, to generate a SVG that matches the exact same HTML and CSS in a browser.

\hypertarget{README.md_autotoc_md30253}{}\doxysubsubsubsection{\texorpdfstring{JSX}{JSX}}\label{README.md_autotoc_md30253}
Satori only accepts JSX elements that are pure and stateless. You can use a subset of HTML elements (see section below), or custom React components, but React APIs such as {\ttfamily use\+State}, {\ttfamily use\+Effect}, {\ttfamily dangerously\+Set\+Inner\+HTML} are not supported.\hypertarget{README.md_autotoc_md30254}{}\doxysubsubsubsubsection{\texorpdfstring{Use without JSX}{Use without JSX}}\label{README.md_autotoc_md30254}
If you don\textquotesingle{}t have JSX transpiler enabled, you can simply pass \href{https://reactjs.org/docs/introducing-jsx.html}{\texttt{ React-\/elements-\/like objects}} that have {\ttfamily type}, {\ttfamily props.\+children} and {\ttfamily props.\+style} (and other properties too) directly\+:

\hypertarget{README.md_autotoc_md30255}{}\doxysubsubsubsection{\texorpdfstring{HTML Elements}{HTML Elements}}\label{README.md_autotoc_md30255}
Satori supports a limited subset of HTML and CSS features, due to its special use cases. In general, only these static and visible elements and properties that are implemented.

For example, the {\ttfamily \texorpdfstring{$<$}{<}input\texorpdfstring{$>$}{>}} HTML element, the {\ttfamily cursor} CSS property are not in consideration. And you can\textquotesingle{}t use {\ttfamily \texorpdfstring{$<$}{<}style\texorpdfstring{$>$}{>}} tags or external resources via {\ttfamily \texorpdfstring{$<$}{<}link\texorpdfstring{$>$}{>}} or {\ttfamily \texorpdfstring{$<$}{<}script\texorpdfstring{$>$}{>}}.

Also, Satori does not guarantee that the SVG will 100\% match the browser-\/rendered HTML output since Satori implements its own layout engine based on the \href{https://www.w3.org/TR/SVG11}{\texttt{ SVG 1.\+1 spec}}.

You can find the list of supported HTML elements and their preset styles \href{https://github.com/vercel/satori/blob/main/src/handler/presets.ts}{\texttt{ here}}.\hypertarget{README.md_autotoc_md30256}{}\doxysubsubsubsubsection{\texorpdfstring{Images}{Images}}\label{README.md_autotoc_md30256}
You can use {\ttfamily \texorpdfstring{$<$}{<}img\texorpdfstring{$>$}{>}} to embed images. However, {\ttfamily width}, and {\ttfamily height} attributes are recommended to set\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{await\ satori(}
\DoxyCodeLine{\ \ <img\ src="{}https://picsum.photos/200/300"{}\ width=\{200\}\ height=\{300\}\ />,}
\DoxyCodeLine{\ \ options}
\DoxyCodeLine{)}

\end{DoxyCode}


When using {\ttfamily background-\/image}, the image will be stretched to fit the element by default if you don\textquotesingle{}t specify the size.

If you want to render the generated SVG to another image format such as PNG, it would be better to use base64 encoded image data (or buffer) directly as {\ttfamily props.\+src} so no extra I/O is needed in Satori\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{await\ satori(}
\DoxyCodeLine{\ \ <img\ src="{}data:image/png;base64,..."{}\ width=\{200\}\ height=\{300\}\ />,}
\DoxyCodeLine{\ \ //\ Or\ src=\{arrayBuffer\},\ src=\{buffer\}}
\DoxyCodeLine{\ \ options}
\DoxyCodeLine{)}

\end{DoxyCode}

\hypertarget{README.md_autotoc_md30263}{}\doxysubsubsubsection{\texorpdfstring{Runtime and WASM}{Runtime and WASM}}\label{README.md_autotoc_md30263}
Satori can be used in browser, Node.\+js (\texorpdfstring{$>$}{>}= 16), and Web Workers.

By default, Satori depends on asm.\+js for the browser runtime, and native module in Node.\+js. However, you can optionally load WASM instead by importing {\ttfamily satori/wasm} and provide the initialized WASM module instance of Yoga to Satori\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ satori,\ \{\ init\ \}\ from\ 'satori/wasm'}
\DoxyCodeLine{import\ initYoga\ from\ 'yoga-\/wasm-\/web'}
\DoxyCodeLine{}
\DoxyCodeLine{const\ yoga\ =\ initYoga(await\ fetch('/yoga.wasm').then(res\ =>\ res.arrayBuffer()))}
\DoxyCodeLine{init(yoga)}
\DoxyCodeLine{}
\DoxyCodeLine{await\ satori(...)}

\end{DoxyCode}


When running in the browser or in the Node.\+js environment, WASM files need to be hosted and fetched before initializing. asm.\+js can be bundled together with the lib. In this case WASM should be faster.

When running on the Node.\+js server, native modules should be faster. However there are Node.\+js environments where native modules are not supported (e.\+g. Stack\+Blitz\textquotesingle{}s Web\+Containers), or other JS runtimes that support WASM (e.\+g. Vercel\textquotesingle{}s Edge Runtime, Cloudflare Workers, or Deno).

Additionally, there are other difference between asm.\+js, native and WASM, such as security and compatibility.

Overall there are many trade-\/offs between each choice, and it\textquotesingle{}s better to pick the one that works the best for your use case.\hypertarget{README.md_autotoc_md30264}{}\doxysubsubsubsection{\texorpdfstring{Font Embedding}{Font Embedding}}\label{README.md_autotoc_md30264}
By default, Satori renders the text as {\ttfamily \texorpdfstring{$<$}{<}path\texorpdfstring{$>$}{>}} in SVG, instead of {\ttfamily \texorpdfstring{$<$}{<}text\texorpdfstring{$>$}{>}}.
