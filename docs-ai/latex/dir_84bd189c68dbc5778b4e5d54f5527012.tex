\doxysubsection{Detailed Description}
\href{https://www.npmjs.com/package/eslint-plugin-jsonc}{\texttt{ eslint-\/plugin-\/jsonc}} is ESLint plugin for \href{https://json.org/}{\texttt{ JSON}}, \href{https://github.com/microsoft/node-jsonc-parser}{\texttt{ JSONC}} and \href{https://json5.org/}{\texttt{ JSON5}} files.

\href{https://www.npmjs.com/package/eslint-plugin-jsonc}{\texttt{ }} \href{https://www.npmjs.com/package/eslint-plugin-jsonc}{\texttt{ }} \href{http://www.npmtrends.com/eslint-plugin-jsonc}{\texttt{ }} \href{http://www.npmtrends.com/eslint-plugin-jsonc}{\texttt{ }} \href{http://www.npmtrends.com/eslint-plugin-jsonc}{\texttt{ }} \href{http://www.npmtrends.com/eslint-plugin-jsonc}{\texttt{ }} \href{http://www.npmtrends.com/eslint-plugin-jsonc}{\texttt{ }} \href{https://github.com/ota-meshi/eslint-plugin-jsonc/actions?query=workflow\%3ACI}{\texttt{ }} \href{https://coveralls.io/github/ota-meshi/eslint-plugin-jsonc?branch=master}{\texttt{ }}\hypertarget{README.md_autotoc_md13592}{}\doxysubsection{\texorpdfstring{\+:name\+\_\+badge\+: Features}{\+:name\+\_\+badge\+: Features}}\label{README.md_autotoc_md13592}
This ESLint plugin provides linting rules relate to better ways to help you avoid problems when using \href{https://json.org/}{\texttt{ JSON}}, \href{https://github.com/microsoft/node-jsonc-parser}{\texttt{ JSONC}} and \href{https://json5.org/}{\texttt{ JSON5}}.


\begin{DoxyItemize}
\item You can use ESLint to lint \href{https://json.org/}{\texttt{ JSON}}.
\item You can apply rules similar to the rules you use for Java\+Script to JSON using the \href{https://ota-meshi.github.io/eslint-plugin-jsonc/rules/auto.html}{\texttt{ {\ttfamily "{}jsonc/auto"{}}}} rule provided by this plugin.
\item You can choose the appropriate config provided by this plugin depending on whether you are using \href{https://json.org/}{\texttt{ JSON}}, \href{https://github.com/microsoft/node-jsonc-parser}{\texttt{ JSONC}} or \href{https://json5.org/}{\texttt{ JSON5}}.
\item Supports \href{https://vue-loader.vuejs.org/spec.html}{\texttt{ Vue SFC}} custom blocks such as {\ttfamily \texorpdfstring{$<$}{<}i18n\texorpdfstring{$>$}{>}}. ~\newline

\item Supports ESLint directives. e.\+g. {\ttfamily // eslint-\/disable-\/next-\/line}
\item You can check your code in real-\/time using the ESLint editor integrations.

You can check on the \href{https://ota-meshi.github.io/eslint-plugin-jsonc/playground/}{\texttt{ Online DEMO}}.\hypertarget{README.md_autotoc_md13593}{}\doxysubsection{\texorpdfstring{\+:question\+: Why is it ESLint plugin?}{\+:question\+: Why is it ESLint plugin?}}\label{README.md_autotoc_md13593}
ESLint is a great linter for Java\+Script. ~\newline
 Since \href{https://json.org/}{\texttt{ JSON}} is a subset of Java\+Script, the same parser and rules can be applied to \href{https://json.org/}{\texttt{ JSON}}. ~\newline
 Also, \href{https://github.com/microsoft/node-jsonc-parser}{\texttt{ JSONC}} and \href{https://json5.org/}{\texttt{ JSON5}}, which are variants of \href{https://json.org/}{\texttt{ JSON}}, are more similar to Java\+Script than \href{https://json.org/}{\texttt{ JSON}}. Applying a Java\+Script linter to \href{https://json.org/}{\texttt{ JSON}} is more rational than creating a JSON-\/specific linter.\hypertarget{README.md_autotoc_md13594}{}\doxysubsubsection{\texorpdfstring{How does {\ttfamily eslint-\/plugin-\/jsonc} work?}{How does {\ttfamily eslint-\/plugin-\/jsonc} work?}}\label{README.md_autotoc_md13594}
This plugin parses {\ttfamily .json} with its own parser, but this parser just converts AST parsed by {\ttfamily acorn} (It is used internally by the ESLint standard parser) into AST with another name. However, ASTs that do not exist in \href{https://json.org/}{\texttt{ JSON}} and the superset of JSON syntaxes are reported as parsing errors. By converting the AST to another name, we prevent false positives from ESLint core rules. ~\newline
 Moreover, You can do the same linting using the extended rules of the ESLint core rules provided by this plugin.

The parser package used by this plugin is \href{https://github.com/ota-meshi/jsonc-eslint-parser}{\texttt{ jsonc-\/eslint-\/parser}}.\hypertarget{README.md_autotoc_md13595}{}\doxysubsection{\texorpdfstring{\+:question\+: How is it different from other JSON plugins?}{\+:question\+: How is it different from other JSON plugins?}}\label{README.md_autotoc_md13595}
\hypertarget{README.md_autotoc_md13596}{}\doxysubsubsection{\texorpdfstring{\href{https://github.com/eslint/json}{\texttt{ {\ttfamily @eslint/json}}}}{\href{https://github.com/eslint/json}{\texttt{ {\ttfamily @eslint/json}}}}}\label{README.md_autotoc_md13596}
They work similarly, but \href{https://github.com/eslint/json}{\texttt{ {\ttfamily @eslint/json}}} is an ESLint JSON language plugin, but {\ttfamily eslint-\/plugin-\/jsonc} is an ESLint plugin.


\begin{DoxyItemize}
\item {\ttfamily eslint-\/plugin-\/jsonc} was created 4 years before \href{https://github.com/eslint/json}{\texttt{ {\ttfamily @eslint/json}}} and it has more rules than \href{https://github.com/eslint/json}{\texttt{ {\ttfamily @eslint/json}}}.
\item The parser used by {\ttfamily eslint-\/plugin-\/jsonc} accepts more non-\/standard JSON syntax than \href{https://github.com/eslint/json}{\texttt{ {\ttfamily @eslint/json}}}. {\ttfamily eslint-\/plugin-\/jsonc} has rules that can auto-\/fixed these non-\/standard syntax to standard syntax.
\item {\ttfamily eslint-\/plugin-\/jsonc} can also be used together with \href{https://github.com/eslint/json}{\texttt{ {\ttfamily @eslint/json}}}.
\end{DoxyItemize}\hypertarget{README.md_autotoc_md13597}{}\doxysubsubsection{\texorpdfstring{Plugins that do not use AST}{Plugins that do not use AST}}\label{README.md_autotoc_md13597}
e.\+g. \href{https://www.npmjs.com/package/eslint-plugin-json}{\texttt{ eslint-\/plugin-\/json}}

These plugins use the processor to parse and return the results independently, without providing the ESLint engine with AST and source code text.

Plugins don\textquotesingle{}t provide AST, so you can\textquotesingle{}t use directive comments (e.\+g. {\ttfamily /\texorpdfstring{$\ast$}{*} eslint-\/disable \texorpdfstring{$\ast$}{*}/}). ~\newline
 Plugins don\textquotesingle{}t provide source code text, so you can\textquotesingle{}t use it with plugins and rules that use text (e.\+g. \href{https://github.com/prettier/eslint-plugin-prettier}{\texttt{ eslint-\/plugin-\/prettier}}, \href{https://eslint.org/docs/rules/eol-last}{\texttt{ eol-\/last}}). ~\newline
 Also, most plugins don\textquotesingle{}t support JSON5.

{\bfseries{eslint-\/plugin-\/jsonc}} works by providing AST and source code text to ESLint.\hypertarget{README.md_autotoc_md13598}{}\doxysubsubsection{\texorpdfstring{Plugins that use the same AST as Java\+Script}{Plugins that use the same AST as Java\+Script}}\label{README.md_autotoc_md13598}
e.\+g. \href{https://www.npmjs.com/package/eslint-plugin-json-files}{\texttt{ eslint-\/plugin-\/json-\/files}}, \href{https://www.npmjs.com/package/eslint-plugin-json-es}{\texttt{ eslint-\/plugin-\/json-\/es}}

These plugins use the same AST as Java\+Script for linting.

Since the plugin uses the same AST as Java\+Script, it may not report syntax that is not available in JSON (e.\+g. {\ttfamily 1 + 1}, {\ttfamily (42)}). Also, ESLint core rules and other plugin rules can false positives (e.\+g. \href{https://eslint.org/docs/rules/quote-props}{\texttt{ quote-\/props}} rule reports quote on keys), which can complicate the your configuration.

The AST used by {\bfseries{eslint-\/plugin-\/jsonc}} is similar to Java\+Script AST, but with a different node name. This will prevent false positives. This means that it can be easily used in combination with other plugins.\hypertarget{README.md_autotoc_md13599}{}\doxysubsection{\texorpdfstring{\+:book\+: Documentation}{\+:book\+: Documentation}}\label{README.md_autotoc_md13599}
