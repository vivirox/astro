\hypertarget{readme.md_autotoc_md21449}{}\doxysubsubsection{\texorpdfstring{API}{API}}\label{readme.md_autotoc_md21449}
Any {\ttfamily kleur} method returns a {\ttfamily String} when invoked with input; otherwise chaining is expected.

\begin{quote}
It\textquotesingle{}s up to the developer to pass the output to destinations like {\ttfamily console.\+log}, {\ttfamily process.\+stdout.\+write}, etc. \end{quote}


The methods below are grouped by type for legibility purposes only. They each can be chained or nested with one another.

{\itshape {\bfseries{Colors\+:}}} \begin{quote}
black --- red --- green --- yellow --- blue --- magenta --- cyan --- white --- gray --- grey \end{quote}


{\itshape {\bfseries{Backgrounds\+:}}} \begin{quote}
bg\+Black --- bg\+Red --- bg\+Green --- bg\+Yellow --- bg\+Blue --- bg\+Magenta --- bg\+Cyan --- bg\+White \end{quote}


{\itshape {\bfseries{Modifiers\+:}}} \begin{quote}
reset --- bold --- dim --- italic\texorpdfstring{$\ast$}{*} --- underline --- inverse --- hidden --- strikethrough\texorpdfstring{$\ast$}{*} \end{quote}


\textsuperscript{\texorpdfstring{$\ast$}{*} {\itshape Not widely supported}}\hypertarget{readme.md_autotoc_md21450}{}\doxysubsubsection{\texorpdfstring{Individual Colors}{Individual Colors}}\label{readme.md_autotoc_md21450}
When you only need a few colors, it doesn\textquotesingle{}t make sense to import {\itshape all} of {\ttfamily kleur} because, as small as it is, {\ttfamily kleur} is not treeshakeable, and so most of its code will be doing nothing. In order to fix this, you can import from the {\ttfamily kleur/colors} submodule which {\itshape fully} supports tree-\/shaking.

The caveat with this approach is that color functions {\bfseries{are not}} chainable\texorpdfstring{$\sim$}{\string~}!~\newline
Each function receives and colorizes its input. You may combine colors, backgrounds, and modifiers by nesting function calls within other functions.


\begin{DoxyCode}{0}
\DoxyCodeLine{//\ or:\ import\ *\ as\ kleur\ from\ 'kleur/colors';}
\DoxyCodeLine{import\ \{\ red,\ underline,\ bgWhite\ \}\ from\ 'kleur/colors';}
\DoxyCodeLine{}
\DoxyCodeLine{red('red\ text');}
\DoxyCodeLine{//\string~>\ kleur.red('red\ text');}
\DoxyCodeLine{}
\DoxyCodeLine{underline(red('red\ underlined\ text'));}
\DoxyCodeLine{//\string~>\ kleur.underline().red('red\ underlined\ text');}
\DoxyCodeLine{}
\DoxyCodeLine{bgWhite(underline(red('red\ underlined\ text\ w/\ white\ background')));}
\DoxyCodeLine{//\string~>\ kleur.bgWhite().underline().red('red\ underlined\ text\ w/\ white\ background');}

\end{DoxyCode}


\begin{quote}
{\bfseries{Note\+:}} All the same \doxylink{README.md_api}{colors, backgrounds, and modifiers} are available. \end{quote}


{\itshape {\bfseries{Conditional Support}}}

The {\ttfamily kleur/colors} submodule also allows you to toggle color support, as needed.~\newline
 It includes the same initial assumptions as {\ttfamily kleur}, in an attempt to have colors enabled by default.

Unlike {\ttfamily kleur}, this setting exists as {\ttfamily kleur.\$.enabled} instead of {\ttfamily kleur.\+enabled}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ *\ as\ kleur\ from\ 'kleur/colors';}
\DoxyCodeLine{//\ or:\ import\ \{\ \$,\ red\ \}\ from\ 'kleur/colors';}
\DoxyCodeLine{}
\DoxyCodeLine{//\ manually\ disabled}
\DoxyCodeLine{kleur.\$.enabled\ =\ false;}
\DoxyCodeLine{}
\DoxyCodeLine{//\ or\ use\ another\ library\ to\ detect\ support}
\DoxyCodeLine{kleur.\$.enabled\ =\ require('color-\/support').level\ >\ 0;}
\DoxyCodeLine{}
\DoxyCodeLine{console.log(red('I\ will\ only\ be\ colored\ red\ if\ the\ terminal\ supports\ colors'));}

\end{DoxyCode}

\hypertarget{readme.md_autotoc_md21454}{}\doxysubsubsection{\texorpdfstring{History}{History}}\label{readme.md_autotoc_md21454}
This project originally forked \href{https://github.com/doowb/ansi-colors}{\texttt{ {\ttfamily ansi-\/colors}}}.

Beginning with {\ttfamily kleur@3.\+0}, the Chalk-\/style syntax (magical getter) has been replaced with function calls per key\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{//\ Old:}
\DoxyCodeLine{c.red.bold.underline('old');}
\DoxyCodeLine{}
\DoxyCodeLine{//\ New:}
\DoxyCodeLine{c.red().bold().underline('new');}

\end{DoxyCode}
 \begin{quote}
\textsuperscript{{\itshape As I work more with Rust, the newer syntax feels so much better \& more natural!}} \end{quote}


If you prefer the old syntax, you may migrate to {\ttfamily ansi-\/colors} or newer {\ttfamily chalk} releases.~\newline
