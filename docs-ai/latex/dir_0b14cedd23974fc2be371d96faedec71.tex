Trunk is a blazingly fast meta code checker and formatter with extraordinary features like caching, preexisting issue detection, a daemon, and a language server. It\textquotesingle{}s managed completely via config-\/as-\/code, so you can easily pin your repo to specific versions of your linters, formatters, and static analyzers.

After you {\ttfamily trunk init}, {\ttfamily trunk.\+yaml} will contain a pinned version of Trunk to use for your repo. When you run trunk, it will automatically detect which version you should be running for a particular repo and, if needed, download it, then run it. The trunk VSCode extension works the same way\+: regardless of what repo you\textquotesingle{}re in, the underlying version of trunk it runs is based on the pinned version in your {\ttfamily trunk.\+yaml} file. Config-\/as-\/code üëç.

This means that {\bfseries{everyone working in a repo, using the {\ttfamily trunk} cli, the VSCode extension, or when running on CI, all get the same results}} because they\textquotesingle{}re running the same version of trunk and the same versions of all the linters/formatters. No more "{}doesn\textquotesingle{}t happen on my machine"{}. When you want to upgrade to a newer version, just run {\ttfamily trunk upgrade} and commit the updated {\ttfamily trunk.\+yaml}.\hypertarget{README.md_autotoc_md6572}{}\doxysubsubsection{\texorpdfstring{Development Status}{Development Status}}\label{README.md_autotoc_md6572}
