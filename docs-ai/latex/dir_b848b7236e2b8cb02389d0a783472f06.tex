This library comes in multiple variants\+:
\begin{DoxyItemize}
\item {\ttfamily web-\/streams-\/polyfill}\+: a polyfill that replaces the native stream implementations. Recommended for use in web apps supporting older browsers through a {\ttfamily \texorpdfstring{$<$}{<}script\texorpdfstring{$>$}{>}} tag.
\item {\ttfamily web-\/streams-\/polyfill/es6}\+: a polyfill targeting ES2015+ environments. Recommended for use in web apps supporting modern browsers through a {\ttfamily \texorpdfstring{$<$}{<}script\texorpdfstring{$>$}{>}} tag.
\item {\ttfamily web-\/streams-\/polyfill/es2018}\+: a polyfill targeting ES2018+ environments.
\item {\ttfamily web-\/streams-\/polyfill/ponyfill}\+: a \href{https://github.com/sindresorhus/ponyfill}{\texttt{ ponyfill}} that provides the stream implementations without replacing any globals. Recommended for use in legacy Node applications, or in web libraries supporting older browsers.
\item {\ttfamily web-\/streams-\/polyfill/ponyfill/es6}\+: a ponyfill targeting ES2015+ environments. Recommended for use in Node 6+ applications, or in web libraries supporting modern browsers.
\item {\ttfamily web-\/streams-\/polyfill/ponyfill/es2018}\+: a ponyfill targeting ES2018+ environments. Recommended for use in Node 10+ applications.
\end{DoxyItemize}

Each variant also includes Type\+Script type definitions, compatible with the DOM type definitions for streams included in Type\+Script.

\hypertarget{README.md_autotoc_md35495}{}\doxysubsection{\texorpdfstring{Compatibility}{Compatibility}}\label{README.md_autotoc_md35495}
The {\ttfamily polyfill} and {\ttfamily ponyfill} variants work in any ES5-\/compatible environment that has a global {\ttfamily Promise}. If you need to support older browsers or Node versions that do not have a native {\ttfamily Promise} implementation (check the \href{https://kangax.github.io/compat-table/es6/\#test-Promise}{\texttt{ support table}}), you must first include a {\ttfamily Promise} polyfill (e.\+g. \href{https://www.npmjs.com/package/promise-polyfill}{\texttt{ promise-\/polyfill}}).

The {\ttfamily polyfill/es6} and {\ttfamily ponyfill/es6} variants work in any ES2015-\/compatible environment.

The {\ttfamily polyfill/es2018} and {\ttfamily ponyfill/es2018} variants work in any ES2018-\/compatible environment.

\href{https://streams.spec.whatwg.org/\#rs-asynciterator}{\texttt{ Async iterable support for {\ttfamily Readable\+Stream}}} is available in all variants, but requires an ES2018-\/compatible environment or a polyfill for {\ttfamily Symbol.\+async\+Iterator}.

\href{https://streams.spec.whatwg.org/\#ws-default-controller-signal}{\texttt{ {\ttfamily Writable\+Stream\+Default\+Controller.\+signal}}} is available in all variants, but requires a global {\ttfamily Abort\+Controller} constructor. If necessary, consider using a polyfill such as \href{https://www.npmjs.com/package/abortcontroller-polyfill}{\texttt{ abortcontroller-\/polyfill}}.

\href{https://developer.mozilla.org/en-US/docs/Web/API/ReadableStreamBYOBReader/read}{\texttt{ Reading with a BYOB reader}} is available in all variants, but requires {\ttfamily Array\+Buffer.\+prototype.\+transfer()} or {\ttfamily structured\+Clone()} to exist in order to correctly transfer the given view\textquotesingle{}s buffer. If not available, then the buffer won\textquotesingle{}t be transferred during the read.\hypertarget{README.md_autotoc_md35496}{}\doxysubsection{\texorpdfstring{Compliance}{Compliance}}\label{README.md_autotoc_md35496}
The polyfill implements \href{https://streams.spec.whatwg.org/commit-snapshots/4dc123a6e7f7ba89a8c6a7975b021156f39cab52/}{\texttt{ version {\ttfamily 4dc123a} (13 Nov 2023)}} of the streams specification.

The polyfill is tested against the same \href{https://github.com/web-platform-tests/wpt/tree/2a298b616b7c865917d7198a287310881cbfdd8d/streams}{\texttt{ web platform tests}} that are used by browsers to test their native implementations. The polyfill aims to pass all tests, although it allows some exceptions for practical reasons\+:
\begin{DoxyItemize}
\item The {\ttfamily es2018} variant passes all of the tests.
\item The {\ttfamily es6} variant passes the same tests as the {\ttfamily es2018} variant, except for the \href{https://github.com/web-platform-tests/wpt/blob/2a298b616b7c865917d7198a287310881cbfdd8d/streams/readable-streams/async-iterator.any.js\#L24}{\texttt{ test for the prototype of {\ttfamily Readable\+Stream}\textquotesingle{}s async iterator}}. Retrieving the correct {\ttfamily Async\+Iterator\+Prototype\%} requires using an async generator ({\ttfamily async function\texorpdfstring{$\ast$}{*} () \{\}}), which is invalid syntax before ES2018. Instead, the polyfill \href{https://github.com/MattiasBuelens/web-streams-polyfill/blob/v2.0.0/src/target/es5/stub/async-iterator-prototype.ts}{\texttt{ creates its own version}} which is functionally equivalent to the real prototype.
\item The {\ttfamily es5} variant passes the same tests as the {\ttfamily es6} variant, except for various tests about specific characteristics of the constructors, properties and methods. These test failures do not affect the run-\/time behavior of the polyfill. For example\+:
\begin{DoxyItemize}
\item The {\ttfamily name} property of down-\/leveled constructors is incorrect.
\item The {\ttfamily length} property of down-\/leveled constructors and methods with optional arguments is incorrect.
\item Not all properties and methods are correctly marked as non-\/enumerable.
\item Down-\/leveled class methods are not correctly marked as non-\/constructable.
\end{DoxyItemize}
\end{DoxyItemize}
