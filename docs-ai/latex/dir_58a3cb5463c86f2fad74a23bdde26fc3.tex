\hypertarget{README.md_autotoc_md32698}{}\doxysubsection{\texorpdfstring{Command line usage}{Command line usage}}\label{README.md_autotoc_md32698}

\begin{DoxyCode}{0}
\DoxyCodeLine{terser\ [input\ files]\ [options]}

\end{DoxyCode}


Terser can take multiple input files. It\textquotesingle{}s recommended that you pass the input files first, then pass the options. Terser will parse input files in sequence and apply any compression options. The files are parsed in the same global scope, that is, a reference from a file to some variable/function declared in another file will be matched properly.

Command line arguments that take options (like --parse, --compress, --mangle and --format) can take in a comma-\/separated list of default option overrides. For instance\+: \begin{DoxyVerb}terser input.js --compress ecma=2015,computed_props=false
\end{DoxyVerb}


If no input file is specified, Terser will read from STDIN.

If you wish to pass your options before the input files, separate the two with a double dash to prevent input files being used as option arguments\+: \begin{DoxyVerb}terser --compress --mangle -- input.js
\end{DoxyVerb}

Specify {\ttfamily -\/-\/output} ({\ttfamily -\/o}) to declare the output file. Otherwise the output goes to STDOUT.\hypertarget{README.md_autotoc_md32700}{}\doxysubsubsection{\texorpdfstring{CLI source map options}{CLI source map options}}\label{README.md_autotoc_md32700}
Terser can generate a source map file, which is highly useful for debugging your compressed Java\+Script. To get a source map, pass {\ttfamily -\/-\/source-\/map -\/-\/output output.\+js} (source map will be written out to {\ttfamily output.\+js.\+map}).

Additional options\+:


\begin{DoxyItemize}
\item `-\/-\/source-\/map "{}filename='$<$\+NAME$>$\textquotesingle{}"{}\`{} to specify the name of the source map.
\item `-\/-\/source-\/map "{}root='$<$\+URL$>$\textquotesingle{}"{}\`{} to pass the URL where the original files can be found.
\item `-\/-\/source-\/map "{}url='$<$\+URL$>$\textquotesingle{}"{}{\ttfamily to specify the URL where the source map can be found. Otherwise Terser assumes HTTP}X-\/\+Source\+Map{\ttfamily is being used and will omit the }//\# source\+Mapping\+URL=\`{} directive.
\end{DoxyItemize}

For example\+: \begin{DoxyVerb}terser js/file1.js js/file2.js \
         -o foo.min.js -c -m \
         --source-map "root='http://foo.com/src',url='foo.min.js.map'"
\end{DoxyVerb}


The above will compress and mangle {\ttfamily file1.\+js} and {\ttfamily file2.\+js}, will drop the output in {\ttfamily foo.\+min.\+js} and the source map in {\ttfamily foo.\+min.\+js.\+map}. The source mapping will refer to {\ttfamily \href{http://foo.com/src/js/file1.js}{\texttt{ http\+://foo.\+com/src/js/file1.\+js}}} and {\ttfamily \href{http://foo.com/src/js/file2.js}{\texttt{ http\+://foo.\+com/src/js/file2.\+js}}} (in fact it will list {\ttfamily \href{http://foo.com/src}{\texttt{ http\+://foo.\+com/src}}} as the source map root, and the original files as {\ttfamily js/file1.\+js} and {\ttfamily js/file2.\+js}).\hypertarget{README.md_autotoc_md32701}{}\doxysubsubsubsection{\texorpdfstring{Composed source map}{Composed source map}}\label{README.md_autotoc_md32701}
When you\textquotesingle{}re compressing JS code that was output by a compiler such as Coffee\+Script, mapping to the JS code won\textquotesingle{}t be too helpful. Instead, you\textquotesingle{}d like to map back to the original code (i.\+e. Coffee\+Script). Terser has an option to take an input source map. Assuming you have a mapping from Coffee\+Script → compiled JS, Terser can generate a map from Coffee\+Script → compressed JS by mapping every token in the compiled JS to its original location.

To use this feature pass `-\/-\/source-\/map "{}content='/path/to/input/source.\+map\textquotesingle{}"{}\`{} or \`{}-\/-\/source-\/map "{}content=inline"{}\`{} if the source map is included inline with the sources.\hypertarget{README.md_autotoc_md32702}{}\doxysubsubsection{\texorpdfstring{CLI compress options}{CLI compress options}}\label{README.md_autotoc_md32702}
You need to pass {\ttfamily -\/-\/compress} ({\ttfamily -\/c}) to enable the compressor. Optionally you can pass a comma-\/separated list of compress options.

Options are in the form {\ttfamily foo=bar}, or just {\ttfamily foo} (the latter implies a boolean option that you want to set {\ttfamily true}; it\textquotesingle{}s effectively a shortcut for {\ttfamily foo=true}).

Example\+: \begin{DoxyVerb}terser file.js -c toplevel,sequences=false
\end{DoxyVerb}
\hypertarget{README.md_autotoc_md32703}{}\doxysubsubsection{\texorpdfstring{CLI mangle options}{CLI mangle options}}\label{README.md_autotoc_md32703}
To enable the mangler you need to pass {\ttfamily -\/-\/mangle} ({\ttfamily -\/m}). The following (comma-\/separated) options are supported\+:


\begin{DoxyItemize}
\item {\ttfamily toplevel} (default {\ttfamily false}) -- mangle names declared in the top level scope.
\item {\ttfamily eval} (default {\ttfamily false}) -- mangle names visible in scopes where {\ttfamily eval} or {\ttfamily with} are used.
\end{DoxyItemize}

When mangling is enabled but you want to prevent certain names from being mangled, you can declare those names with {\ttfamily -\/-\/mangle reserved} — pass a comma-\/separated list of names. For example\+: \begin{DoxyVerb}terser ... -m reserved=['$','require','exports']
\end{DoxyVerb}

{\bfseries{Note\+:}} THIS {\bfseries{WILL}} BREAK YOUR CODE. A good rule of thumb is not to use this unless you know exactly what you\textquotesingle{}re doing and how this works and read this section until the end.

Mangling property names is a separate step, different from variable name mangling. Pass {\ttfamily -\/-\/mangle-\/props} to enable it. The least dangerous way to use this is to use the {\ttfamily regex} option like so\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{terser\ example.js\ -\/c\ -\/m\ -\/-\/mangle-\/props\ regex=/\_\$/}

\end{DoxyCode}


This will mangle all properties that end with an underscore. So you can use it to mangle internal methods.

By default, it will mangle all properties in the input code with the exception of built in DOM properties and properties in core Java\+Script classes, which is what will break your code if you don\textquotesingle{}t\+:


\begin{DoxyEnumerate}
\item Control all the code you\textquotesingle{}re mangling
\item Avoid using a module bundler, as they usually will call Terser on each file individually, making it impossible to pass mangled objects between modules.
\item Avoid calling functions like {\ttfamily define\+Property} or {\ttfamily has\+Own\+Property}, because they refer to object properties using strings and will break your code if you don\textquotesingle{}t know what you are doing.
\end{DoxyEnumerate}

\hypertarget{README.md_autotoc_md32721}{}\doxysubsubsubsection{\texorpdfstring{Annotations}{Annotations}}\label{README.md_autotoc_md32721}
Annotations in Terser are a way to tell it to treat a certain function call differently. The following annotations are available\+:


\begin{DoxyItemize}
\item {\ttfamily /\texorpdfstring{$\ast$}{*}@\+\_\+\+\_\+\+INLINE\+\_\+\+\_\+\texorpdfstring{$\ast$}{*}/} -\/ forces a function to be inlined somewhere.
\item {\ttfamily /\texorpdfstring{$\ast$}{*}@\+\_\+\+\_\+\+NOINLINE\+\_\+\+\_\+\texorpdfstring{$\ast$}{*}/} -\/ Makes sure the called function is not inlined into the call site.
\item {\ttfamily /\texorpdfstring{$\ast$}{*}@\+\_\+\+\_\+\+PURE\+\_\+\+\_\+\texorpdfstring{$\ast$}{*}/} -\/ Marks a function call as pure. That means, it can safely be dropped.
\item {\ttfamily /\texorpdfstring{$\ast$}{*}@\+\_\+\+\_\+\+KEY\+\_\+\+\_\+\texorpdfstring{$\ast$}{*}/} -\/ Marks a string literal as a property to also mangle it when mangling properties.
\item {\ttfamily /\texorpdfstring{$\ast$}{*}@\+\_\+\+\_\+\+MANGLE\+\_\+\+PROP\+\_\+\+\_\+\texorpdfstring{$\ast$}{*}/} -\/ Opts-\/in an object property (or class field) for mangling, when the property mangler is enabled.
\end{DoxyItemize}

You can use either a {\ttfamily @} sign at the start, or a {\ttfamily \#}.

Here are some examples on how to use them\+:

\end{DoxyCode}
\hypertarget{README.md_autotoc_md32722}{}\doxysubsubsubsection{\texorpdfstring{ESTree / Spider\+Monkey AST}{ESTree / Spider\+Monkey AST}}\label{README.md_autotoc_md32722}
Terser has its own abstract syntax tree format; for \href{http://lisperator.net/blog/uglifyjs-why-not-switching-to-spidermonkey-ast/}{\texttt{ practical reasons}} we can\textquotesingle{}t easily change to using the Spider\+Monkey AST internally. However, Terser now has a converter which can import a Spider\+Monkey AST.

For example \href{https://github.com/acornjs/acorn}{\texttt{ Acorn}} is a super-\/fast parser that produces a Spider\+Monkey AST. It has a small CLI utility that parses one file and dumps the AST in JSON on the standard output. To use Terser to mangle and compress that\+: \begin{DoxyVerb}acorn file.js | terser -p spidermonkey -m -c
\end{DoxyVerb}


The {\ttfamily -\/p spidermonkey} option tells Terser that all input files are not Java\+Script, but JS code described in Spider\+Monkey AST in JSON. Therefore we don\textquotesingle{}t use our own parser in this case, but just transform that AST into our internal AST.

{\ttfamily spidermonkey} is also available in {\ttfamily minify} as {\ttfamily parse} and {\ttfamily format} options to accept and/or produce a spidermonkey AST.\hypertarget{README.md_autotoc_md32723}{}\doxysubsubsubsection{\texorpdfstring{Use Acorn for parsing}{Use Acorn for parsing}}\label{README.md_autotoc_md32723}
More for fun, I added the {\ttfamily -\/p acorn} option which will use Acorn to do all the parsing. If you pass this option, Terser will {\ttfamily require("{}acorn"{})}.

Acorn is really fast (e.\+g. 250ms instead of 380ms on some 650K code), but converting the Spider\+Monkey tree that Acorn produces takes another 150ms so in total it\textquotesingle{}s a bit more than just using Terser\textquotesingle{}s own parser.\hypertarget{README.md_autotoc_md32724}{}\doxysubsubsubsection{\texorpdfstring{Terser Fast Minify Mode}{Terser Fast Minify Mode}}\label{README.md_autotoc_md32724}

\hypertarget{README.md_autotoc_md32725}{}\doxysubsubsubsubsection{\texorpdfstring{Source maps and debugging}{Source maps and debugging}}\label{README.md_autotoc_md32725}
Various {\ttfamily compress} transforms that simplify, rearrange, inline and remove code are known to have an adverse effect on debugging with source maps. This is expected as code is optimized and mappings are often simply not possible as some code no longer exists. For highest fidelity in source map debugging disable the {\ttfamily compress} option and just use {\ttfamily mangle}.

When debugging, make sure you enable the {\bfseries{"{}map scopes"{}}} feature to map mangled variable names back to their original names. ~\newline
 Without this, all variable values will be {\ttfamily undefined}. See \href{https://github.com/terser/terser/issues/1367}{\texttt{ https\+://github.\+com/terser/terser/issues/1367}} for more details. ~\newline

\hypertarget{README.md_autotoc_md32726}{}\doxysubsubsubsection{\texorpdfstring{Compiler assumptions}{Compiler assumptions}}\label{README.md_autotoc_md32726}
To allow for better optimizations, the compiler makes various assumptions\+:


\begin{DoxyItemize}
\item {\ttfamily .to\+String()} and {\ttfamily .value\+Of()} don\textquotesingle{}t have side effects, and for built-\/in objects they have not been overridden.
\item {\ttfamily undefined}, {\ttfamily NaN} and {\ttfamily Infinity} have not been externally redefined.
\item {\ttfamily arguments.\+callee}, {\ttfamily arguments.\+caller} and {\ttfamily Function.\+prototype.\+caller} are not used.
\item The code doesn\textquotesingle{}t expect the contents of {\ttfamily Function.\+prototype.\+to\+String()} or {\ttfamily Error.\+prototype.\+stack} to be anything in particular.
\item Getting and setting properties on a plain object does not cause other side effects (using {\ttfamily .watch()} or {\ttfamily Proxy}).
\item Object properties can be added, removed and modified (not prevented with {\ttfamily Object.\+define\+Property()}, {\ttfamily Object.\+define\+Properties()}, {\ttfamily Object.\+freeze()}, {\ttfamily Object.\+prevent\+Extensions()} or {\ttfamily Object.\+seal()}).
\item {\ttfamily document.\+all} is not {\ttfamily == null}
\item Assigning properties to a class doesn\textquotesingle{}t have side effects and does not throw.
\end{DoxyItemize}\hypertarget{README.md_autotoc_md32727}{}\doxysubsubsubsection{\texorpdfstring{Build Tools and Adaptors using Terser}{Build Tools and Adaptors using Terser}}\label{README.md_autotoc_md32727}
