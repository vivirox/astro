For the creation of \href{https://www.ietf.org/rfc/rfc4122.txt}{\texttt{ RFC4122}} UUIDs


\begin{DoxyItemize}
\item {\bfseries{Complete}} -\/ Support for RFC4122 version 1, 3, 4, and 5 UUIDs
\item {\bfseries{Cross-\/platform}} -\/ Support for ...
\begin{DoxyItemize}
\item Common\+JS, ECMAScript Modules and CDN builds
\item Node\+JS 12+ (\href{https://github.com/nodejs/Release}{\texttt{ LTS releases}})
\item Chrome, Safari, Firefox, Edge browsers
\item Webpack and rollup.\+js module bundlers
\item React Native / Expo
\end{DoxyItemize}
\item {\bfseries{Secure}} -\/ Cryptographically-\/strong random values
\item {\bfseries{Small}} -\/ Zero-\/dependency, small footprint, plays nice with "{}tree shaking"{} packagers
\item {\bfseries{CLI}} -\/ Includes the \`{}uuid\`{} command line utility
\end{DoxyItemize}

\begin{quote}
{\bfseries{Note}} Upgrading from {\ttfamily uuid@3}? Your code is probably okay, but check out Upgrading From \`{}uuid@3\`{} for details. \end{quote}


\begin{quote}
{\bfseries{Note}} Only interested in creating a version 4 UUID? You might be able to use \href{https://developer.mozilla.org/en-US/docs/Web/API/Crypto/randomUUID}{\texttt{ {\ttfamily crypto.\+random\+UUID()}}}, eliminating the need to install this library. \end{quote}
