This is a simple package that takes table data and generates a \href{https://docs.github.com/en/github/writing-on-github/working-with-advanced-formatting/organizing-information-with-tables}{\texttt{ Git\+Hub flavored markdown (GFM)}} table.\hypertarget{readme.md_autotoc_md22216}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{readme.md_autotoc_md22216}
You can use this package when you want to generate the source code of a GFM table from some data.

This is a simple solution in that it doesn’t handle escapes or HTML or any of that. For a complete but heavier solution, build an AST and serialize it with \href{https://github.com/syntax-tree/mdast-util-to-markdown}{\texttt{ {\ttfamily mdast-\/util-\/to-\/markdown}}} (with \href{https://github.com/syntax-tree/mdast-util-gfm}{\texttt{ {\ttfamily mdast-\/util-\/gfm}}}).\hypertarget{readme.md_autotoc_md22217}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md22217}

Function to detect the length of table cell content ({\ttfamily Function}, default\+: {\ttfamily s =\texorpdfstring{$>$}{>} s.\+length}). This is used when aligning the delimiters ({\ttfamily \texorpdfstring{$\vert$}{|}}) between table cells. Full-\/width characters and emoji mess up delimiter alignment when viewing the markdown source. To fix this, you can pass this function, which receives the cell content and returns its “visible” size. Note that what is and isn’t visible depends on where the text is displayed.
