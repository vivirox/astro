\doxysubsection{Detailed Description}
\href{https://www.npmjs.com/package/eslint-plugin-no-only-tests}{\texttt{ }} \href{https://npmcharts.com/compare/eslint-plugin-no-only-tests?minimal=true}{\texttt{ }} \href{https://github.com/levibuzolic/eslint-plugin-no-only-tests/actions?query=workflow\%3ATests}{\texttt{ }}

ESLint rule for {\ttfamily .only} tests in \href{https://mochajs.org/}{\texttt{ Mocha}}, \href{https://jestjs.io/}{\texttt{ Jest}}, \href{https://jasmine.github.io/}{\texttt{ Jasmine}}, \href{https://github.com/iensu/mocha-cakes-2}{\texttt{ Mocha Cakes 2}} and other JS testing libraries.

The following test blocks are matched by default\+: {\ttfamily describe}, {\ttfamily it}, {\ttfamily context}, {\ttfamily tape}, {\ttfamily test}, {\ttfamily fixture}, {\ttfamily serial}, {\ttfamily Feature}, {\ttfamily Scenario}, {\ttfamily Given}, {\ttfamily And}, {\ttfamily When} and {\ttfamily Then}.

Designed to prevent you from committing focused ({\ttfamily .only}) tests to CI, which may prevent your entire test suite from running.

If the testing framework you use doesn\textquotesingle{}t use {\ttfamily .only} to focus tests, you can override the matchers with \doxylink{README.md_options}{options}.\hypertarget{README.md_autotoc_md14086}{}\doxysubsection{\texorpdfstring{Installation}{Installation}}\label{README.md_autotoc_md14086}
