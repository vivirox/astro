{\bfseries{ESLint Plugin Command}} is a special kind of ESLint plugin, that by default, {\bfseries{does nothing}}. Instead of checking for code quality, it serves as a micro-\/codemod tool triggers by special comments on-\/demand, resuse the infrastructure of ESLint.

For example, one of the built-\/in commands, {\ttfamily /// to-\/function} allows you to convert a single arrow function expression to a function declaration.


\begin{DoxyCode}{0}
\DoxyCodeLine{///\ to-\/function}
\DoxyCodeLine{const\ foo\ =\ async\ <T>(msg:\ T):\ void\ =>\ \{}
\DoxyCodeLine{\ \ console.log(msg)}
\DoxyCodeLine{\}}

\end{DoxyCode}


Will be transformed to this when you hit save with your editor or run {\ttfamily eslint . -\/-\/fix}. After executing the command, the comment will also be removed along with the transformation\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{async\ function\ foo<T>(msg:\ T):\ void\ \{}
\DoxyCodeLine{\ \ console.log(msg)}
\DoxyCodeLine{\}}

\end{DoxyCode}


One more example that {\ttfamily /// to-\/promise-\/all} converts a sequence of {\ttfamily await} expressions to {\ttfamily await Promise.\+all()}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{///\ to-\/promise-\/all}
\DoxyCodeLine{const\ foo\ =\ await\ bar().then}
\DoxyCodeLine{const\ \{\ get\ \}\ =\ await\ import('lodash-\/es')}

\end{DoxyCode}


Will be transformed to\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{const\ [}
\DoxyCodeLine{\ \ foo,}
\DoxyCodeLine{\ \ \{\ get\ \},}
\DoxyCodeLine{]\ =\ await\ Promise.all([}
\DoxyCodeLine{\ \ bar(),}
\DoxyCodeLine{\ \ import('lodash-\/es'),}
\DoxyCodeLine{]\ as\ const)}

\end{DoxyCode}
