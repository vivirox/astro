This package is a \href{https://github.com/unifiedjs/unified}{\texttt{ unified}} (\href{https://github.com/rehypejs/rehype}{\texttt{ rehype}}) plugin that defines how to take HTML as input and turn it into a syntax tree. When it‚Äôs used, HTML can be parsed and other rehype plugins can be used after it.

See \href{https://github.com/rehypejs/rehype}{\texttt{ the monorepo readme}} for info on what the rehype ecosystem is.\hypertarget{readme.md_autotoc_md28830}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{readme.md_autotoc_md28830}
This plugin adds support to unified for parsing HTML. If you also need to serialize HTML, you can alternatively use \href{../rehype/}{\texttt{ {\ttfamily rehype}}}, which combines unified, this plugin, and \href{../rehype-stringify/}{\texttt{ {\ttfamily rehype-\/stringify}}}.

When you are in a browser, trust your content, don‚Äôt need positional info, and value a smaller bundle size, you can use \href{https://github.com/rehypejs/rehype-dom/tree/main/packages/rehype-dom-parse}{\texttt{ {\ttfamily rehype-\/dom-\/parse}}} instead.

If you don‚Äôt use plugins and want to access the syntax tree, you can directly use \href{https://github.com/syntax-tree/hast-util-from-html}{\texttt{ {\ttfamily hast-\/util-\/from-\/html}}}, which is used inside this plugin. rehype focusses on making it easier to transform content by abstracting such internals away.\hypertarget{readme.md_autotoc_md28831}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md28831}
