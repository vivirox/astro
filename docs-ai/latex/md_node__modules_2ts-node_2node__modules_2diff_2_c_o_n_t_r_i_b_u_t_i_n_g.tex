We also accept \href{https://github.com/kpdecker/jsdiff/pull/new/master}{\texttt{ pull requests}}!

Generally we like to see pull requests that


\begin{DoxyItemize}
\item Maintain the existing code style
\item Are focused on a single change (i.\+e. avoid large refactoring or style adjustments in untouched code if not the primary goal of the pull request)
\item Have \href{http://tbaggery.com/2008/04/19/a-note-about-git-commit-messages.html}{\texttt{ good commit messages}}
\item Have tests
\item Don\textquotesingle{}t decrease the current code coverage (see coverage/lcov-\/report/index.\+html)
\end{DoxyItemize}\hypertarget{md_node__modules_2ts-node_2node__modules_2diff_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md33609}{}\doxysection{\texorpdfstring{Building}{Building}}\label{md_node__modules_2ts-node_2node__modules_2diff_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md33609}

\begin{DoxyCode}{0}
\DoxyCodeLine{npm\ install}
\DoxyCodeLine{npm\ test}

\end{DoxyCode}


The {\ttfamily npm test -\/-\/ dev} implements watching for tests within Node and {\ttfamily karma start} may be used for manual testing in browsers.
