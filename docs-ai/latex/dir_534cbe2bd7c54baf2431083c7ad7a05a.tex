This package is a utility that takes an \href{https://github.com/syntax-tree/mdast}{\texttt{ mdast}} syntax tree as input and turns it into serialized markdown.

This utility is a low level project. It‚Äôs used in \href{https://github.com/remarkjs/remark/tree/main/packages/remark-stringify}{\texttt{ {\ttfamily remark-\/stringify}}}, which focusses on making it easier to transform content by abstracting these internals away.\hypertarget{readme.md_autotoc_md22764}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{readme.md_autotoc_md22764}
If you want to handle syntax trees manually, use this. For an easier time processing content, use the {\bfseries{\href{https://github.com/remarkjs/remark}{\texttt{ remark}}}} ecosystem instead.

You can combine this utility with other utilities to add syntax extensions. Notable examples that deeply integrate with it are \href{https://github.com/syntax-tree/mdast-util-gfm}{\texttt{ {\ttfamily mdast-\/util-\/gfm}}}, \href{https://github.com/syntax-tree/mdast-util-mdx}{\texttt{ {\ttfamily mdast-\/util-\/mdx}}}, \href{https://github.com/syntax-tree/mdast-util-frontmatter}{\texttt{ {\ttfamily mdast-\/util-\/frontmatter}}}, \href{https://github.com/syntax-tree/mdast-util-math}{\texttt{ {\ttfamily mdast-\/util-\/math}}}, and \href{https://github.com/syntax-tree/mdast-util-directive}{\texttt{ {\ttfamily mdast-\/util-\/directive}}}.\hypertarget{readme.md_autotoc_md22765}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md22765}

{\ttfamily }\hypertarget{readme.md_autotoc_md22825}{}\doxysubsection{\texorpdfstring{{\ttfamily Security}}{{\ttfamily Security}}}\label{readme.md_autotoc_md22825}
{\ttfamily }

{\ttfamily {\ttfamily mdast-\/util-\/to-\/markdown} will do its best to serialize markdown to match the syntax tree, but there are several cases where that is impossible. It‚Äôll do its best, but complete roundtripping is impossible given that any value could be injected into the tree.}

{\ttfamily As markdown is sometimes used for HTML, and improper use of HTML can open you up to a \href{https://en.wikipedia.org/wiki/Cross-site_scripting}{\texttt{ cross-\/site scripting (XSS)}} attack, use of {\ttfamily mdast-\/util-\/to-\/markdown} and parsing it again later could potentially be unsafe. When parsing markdown afterwards and then going to HTML, use something like \href{https://github.com/syntax-tree/hast-util-sanitize}{\texttt{ {\ttfamily hast-\/util-\/sanitize}}} to make the tree safe.}
