Shorthands will only accept values that are supported in React, so {\ttfamily background} will only accept a colour, {\ttfamily background\+Color}

There is also support for the {\ttfamily box-\/shadow} shorthand, and this converts into {\ttfamily shadow-\/} properties. Note that these only work on i\+OS.\hypertarget{README.md_autotoc_md11147}{}\doxysubsubsubsection{\texorpdfstring{Shorthand Notes}{Shorthand Notes}}\label{README.md_autotoc_md11147}
{\ttfamily border\{Top,Right,Bottom,Left\}} shorthands are not supported, because {\ttfamily border\+Style} cannot be applied to individual border sides.\hypertarget{README.md_autotoc_md11148}{}\doxysubsection{\texorpdfstring{API}{API}}\label{README.md_autotoc_md11148}
The API is mostly for implementors. However, the main API may be useful for non-\/implementors. The main API is an array of {\ttfamily \mbox{[}property, value\mbox{]}} tuples.


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ transform\ from\ 'css-\/to-\/react-\/native';}
\DoxyCodeLine{//\ or\ const\ transform\ =\ require('css-\/to-\/react-\/native').default;}
\DoxyCodeLine{}
\DoxyCodeLine{transform([}
\DoxyCodeLine{\ \ ['font',\ 'bold\ 14px/16px\ "{}Helvetica"{}'],}
\DoxyCodeLine{\ \ ['margin',\ '5px\ 7px\ 2px'],}
\DoxyCodeLine{\ \ ['border-\/left-\/width',\ '5px'],}
\DoxyCodeLine{]);\ //\ =>\ \{\ fontFamily:\ 'Helvetica',\ ...\ \}}

\end{DoxyCode}


We don\textquotesingle{}t provide a way to get these style tuples in this library, so you\textquotesingle{}ll need to do that yourself. I expect most people will use post\+CSS or another CSS parser. You should try avoid getting these with {\ttfamily string.\+split}, as that has a lot of edge cases (colons and semi-\/colons appearing in comments etc.)

For implementors, there is also a few extra APIs available.

These are for specific use-\/cases, and most people should just be using the API above.


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ \{\ getPropertyName,\ getStylesForProperty\ \}\ from\ 'css-\/to-\/react-\/native';}
\DoxyCodeLine{}
\DoxyCodeLine{getPropertyName('border-\/width');\ //\ =>\ 'borderWidth'}
\DoxyCodeLine{getStylesForProperty('borderWidth',\ '1px\ 0px\ 2px\ 0px');\ //\ =>\ \{\ borderTopWidth:\ 1,\ ...\ \}}

\end{DoxyCode}


Should you wish to opt-\/out of transforming certain shorthands, an array of property names in camel\+Case can be passed as a second argument to {\ttfamily transform}.


\begin{DoxyCode}{0}
\DoxyCodeLine{transform([['border-\/radius',\ '50px']],\ ['borderRadius']);}
\DoxyCodeLine{//\ \{\ borderRadius:\ 50\ \}\ rather\ than\ \{\ borderTopLeft:\ ...\ \}}

\end{DoxyCode}
