\doxysubsection{Detailed Description}
This module provides a variety of interpolation methods for blending between two values. Values may be numbers, colors, strings, arrays, or even deeply-\/nested objects. For example\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{const\ i\ =\ d3.interpolateNumber(10,\ 20);}
\DoxyCodeLine{i(0.0);\ //\ 10}
\DoxyCodeLine{i(0.2);\ //\ 12}
\DoxyCodeLine{i(0.5);\ //\ 15}
\DoxyCodeLine{i(1.0);\ //\ 20}

\end{DoxyCode}


The returned function {\ttfamily i} is called an {\itshape interpolator}. Given a starting value {\itshape a} and an ending value {\itshape b}, it takes a parameter {\itshape t} in the domain \mbox{[}0, 1\mbox{]} and returns the corresponding interpolated value between {\itshape a} and {\itshape b}. An interpolator typically returns a value equivalent to {\itshape a} at {\itshape t} = 0 and a value equivalent to {\itshape b} at {\itshape t} = 1.

You can interpolate more than just numbers. To find the perceptual midpoint between steelblue and brown\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{d3.interpolateLab("{}steelblue"{},\ "{}brown"{})(0.5);\ //\ "{}rgb(142,\ 92,\ 109)"{}}

\end{DoxyCode}


Here’s a more elaborate example demonstrating type inference used by \doxylink{README.md_interpolate}{interpolate}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{const\ i\ =\ d3.interpolate(\{colors:\ ["{}red"{},\ "{}blue"{}]\},\ \{colors:\ ["{}white"{},\ "{}black"{}]\});}
\DoxyCodeLine{i(0.0);\ //\ \{colors:\ ["{}rgb(255,\ 0,\ 0)"{},\ "{}rgb(0,\ 0,\ 255)"{}]\}}
\DoxyCodeLine{i(0.5);\ //\ \{colors:\ ["{}rgb(255,\ 128,\ 128)"{},\ "{}rgb(0,\ 0,\ 128)"{}]\}}
\DoxyCodeLine{i(1.0);\ //\ \{colors:\ ["{}rgb(255,\ 255,\ 255)"{},\ "{}rgb(0,\ 0,\ 0)"{}]\}}

\end{DoxyCode}

\# d3.{\bfseries{interpolate}}({\itshape a}, {\itshape b}) · \href{https://github.com/d3/d3-interpolate/blob/master/src/value.js}{\texttt{ Source}}, \href{https://observablehq.com/@d3/d3-interpolate}{\texttt{ Examples}}

Returns an interpolator between the two arbitrary values {\itshape a} and {\itshape b}. The interpolator implementation is based on the type of the end value {\itshape b}, using the following algorithm\+:


\begin{DoxyEnumerate}
\item If {\itshape b} is null, undefined or a boolean, use the constant {\itshape b}.
\item If {\itshape b} is a number, use \doxylink{README.md_interpolateNumber}{interpolate\+Number}.
\item If {\itshape b} is a \href{https://github.com/d3/d3-color/blob/master/README.md\#color}{\texttt{ color}} or a string coercible to a color, use \doxylink{README.md_interpolateRgb}{interpolate\+Rgb}.
\item If {\itshape b} is a \href{https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Date}{\texttt{ date}}, use \doxylink{README.md_interpolateDate}{interpolate\+Date}.
\item If {\itshape b} is a string, use \doxylink{README.md_interpolateString}{interpolate\+String}.
\item If {\itshape b} is a \href{https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/TypedArray}{\texttt{ typed array}} of numbers, use \doxylink{README.md_interpolateNumberArray}{interpolate\+Number\+Array}.
\item If {\itshape b} is a generic \href{https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array/isArray}{\texttt{ array}}, use \doxylink{README.md_interpolateArray}{interpolate\+Array}.
\item If {\itshape b} is coercible to a number, use \doxylink{README.md_interpolateNumber}{interpolate\+Number}.
\item Use \doxylink{README.md_interpolateObject}{interpolate\+Object}.
\end{DoxyEnumerate}

Based on the chosen interpolator, {\itshape a} is coerced to the suitable corresponding type.

Caution\+: avoid interpolating to or from the number zero when the interpolator is used to generate a string. When very small values are stringified, they may be converted to scientific notation, which is an invalid attribute or style property value in older browsers. For example, the number {\ttfamily 0.\+0000001} is converted to the string {\ttfamily "{}1e-\/7"{}}. This is particularly noticeable with interpolating opacity. To avoid scientific notation, start or end the transition at 1e-\/6\+: the smallest value that is not stringified in scientific notation.

\label{README.md_interpolateString}%
\Hypertarget{README.md_interpolateString}%
\# d3.{\bfseries{interpolate\+String}}({\itshape a}, {\itshape b}) · \href{https://github.com/d3/d3-interpolate/blob/master/src/string.js}{\texttt{ Source}}, \href{https://observablehq.com/@d3/d3-interpolatestring}{\texttt{ Examples}}

Returns an interpolator between the two strings {\itshape a} and {\itshape b}. The string interpolator finds numbers embedded in {\itshape a} and {\itshape b}, where each number is of the form understood by Java\+Script. A few examples of numbers that will be detected within a string\+: {\ttfamily -\/1}, {\ttfamily 42}, {\ttfamily 3.\+14159}, and {\ttfamily 6.\+0221413e+23}.

For each number embedded in {\itshape b}, the interpolator will attempt to find a corresponding number in {\itshape a}. If a corresponding number is found, a numeric interpolator is created using \doxylink{README.md_interpolateNumber}{interpolate\+Number}. The remaining parts of the string {\itshape b} are used as a template\+: the static parts of the string {\itshape b} remain constant for the interpolation, with the interpolated numeric values embedded in the template.

For example, if {\itshape a} is {\ttfamily "{}300 12px sans-\/serif"{}}, and {\itshape b} is {\ttfamily "{}500 36px Comic-\/\+Sans"{}}, two embedded numbers are found. The remaining static parts (of string {\itshape b}) are a space between the two numbers ({\ttfamily "{} "{}}), and the suffix ({\ttfamily "{}px Comic-\/\+Sans"{}}). The result of the interpolator at {\itshape t} = 0.\+5 is {\ttfamily "{}400 24px Comic-\/\+Sans"{}}.

\label{README.md_interpolateArray}%
\Hypertarget{README.md_interpolateArray}%
\# d3.{\bfseries{interpolate\+Array}}({\itshape a}, {\itshape b}) · \href{https://github.com/d3/d3-interpolate/blob/master/src/array.js}{\texttt{ Source}}, \href{https://observablehq.com/@d3/d3-interpolateobject}{\texttt{ Examples}}

Returns an interpolator between the two arrays {\itshape a} and {\itshape b}. If {\itshape b} is a typed array (e.\+g., Float64\+Array), \doxylink{README.md_interpolateNumberArray}{interpolate\+Number\+Array} is called instead.

Internally, an array template is created that is the same length as {\itshape b}. For each element in {\itshape b}, if there exists a corresponding element in {\itshape a}, a generic interpolator is created for the two elements using \doxylink{README.md_interpolate}{interpolate}. If there is no such element, the static value from {\itshape b} is used in the template. Then, for the given parameter {\itshape t}, the template’s embedded interpolators are evaluated. The updated array template is then returned.

For example, if {\itshape a} is the array {\ttfamily \mbox{[}0, 1\mbox{]}} and {\itshape b} is the array {\ttfamily \mbox{[}1, 10, 100\mbox{]}}, then the result of the interpolator for {\itshape t} = 0.\+5 is the array {\ttfamily \mbox{[}0.\+5, 5.\+5, 100\mbox{]}}.

Note\+: {\bfseries{no defensive copy}} of the template array is created; modifications of the returned array may adversely affect subsequent evaluation of the interpolator. No copy is made for performance reasons; interpolators are often part of the inner loop of \href{https://github.com/d3/d3-transition}{\texttt{ animated transitions}}.

\label{README.md_interpolateNumberArray}%
\Hypertarget{README.md_interpolateNumberArray}%
\# d3.{\bfseries{interpolate\+Number\+Array}}({\itshape a}, {\itshape b}) · \href{https://github.com/d3/d3-interpolate/blob/master/src/numberArray.js}{\texttt{ Source}}, \href{https://observablehq.com/@d3/d3-interpolatenumberarray}{\texttt{ Examples}}

Returns an interpolator between the two arrays of numbers {\itshape a} and {\itshape b}. Internally, an array template is created that is the same type and length as {\itshape b}. For each element in {\itshape b}, if there exists a corresponding element in {\itshape a}, the values are directly interpolated in the array template. \label{README.md_interpolateObject}%
\Hypertarget{README.md_interpolateObject}%
\# d3.{\bfseries{interpolate\+Object}}({\itshape a}, {\itshape b}) · \href{https://github.com/d3/d3-interpolate/blob/master/src/object.js}{\texttt{ Source}}, \href{https://observablehq.com/@d3/d3-interpolateobject}{\texttt{ Examples}}

Returns an interpolator between the two objects {\itshape a} and {\itshape b}. Internally, an object template is created that has the same properties as {\itshape b}. For each property in {\itshape b}, if there exists a corresponding property in {\itshape a}, a generic interpolator is created for the two elements using \doxylink{README.md_interpolate}{interpolate}. If there is no such property, the static value from {\itshape b} is used in the template. Then, for the given parameter {\itshape t}, the template\textquotesingle{}s embedded interpolators are evaluated and the updated object template is then returned.

For example, if {\itshape a} is the object {\ttfamily \{x\+: 0, y\+: 1\}} and {\itshape b} is the object {\ttfamily \{x\+: 1, y\+: 10, z\+: 100\}}, the result of the interpolator for {\itshape t} = 0.\+5 is the object {\ttfamily \{x\+: 0.\+5, y\+: 5.\+5, z\+: 100\}}.

Object interpolation is particularly useful for {\itshape dataspace interpolation}, where data is interpolated rather than attribute values. For example, you can interpolate an object which describes an arc in a pie chart, and then use \href{https://github.com/d3/d3-shape/blob/master/README.md\#arc}{\texttt{ d3.\+arc}} to compute the new SVG path data.

Note\+: {\bfseries{no defensive copy}} of the template object is created; modifications of the returned object may adversely affect subsequent evaluation of the interpolator. No copy is made for performance reasons; interpolators are often part of the inner loop of \href{https://github.com/d3/d3-transition}{\texttt{ animated transitions}}.

Returns an interpolator between the two hue angles {\itshape a} and {\itshape b}. If either hue is NaN, the opposing value is used. The shortest path between hues is used. The return value of the interpolator is a number in \mbox{[}0, 360).\hypertarget{README.md_autotoc_md11233}{}\doxysubsubsection{\texorpdfstring{Splines}{Splines}}\label{README.md_autotoc_md11233}
Whereas standard interpolators blend from a starting value {\itshape a} at {\itshape t} = 0 to an ending value {\itshape b} at {\itshape t} = 1, spline interpolators smoothly blend multiple input values for {\itshape t} in \mbox{[}0,1\mbox{]} using piecewise polynomial functions. Only cubic uniform nonrational \href{https://en.wikipedia.org/wiki/B-spline}{\texttt{ B-\/splines}} are currently supported, also known as basis splines.

\# d3.{\bfseries{piecewise}}(\mbox{[}{\itshape interpolate}, \mbox{]}{\itshape values}) · \href{https://github.com/d3/d3-interpolate/blob/master/src/piecewise.js}{\texttt{ Source}}, \href{https://observablehq.com/@d3/d3-piecewise}{\texttt{ Examples}}

Returns a piecewise interpolator, composing interpolators for each adjacent pair of {\itshape values}. The returned interpolator maps {\itshape t} in \mbox{[}0, 1 / ({\itshape n} -\/ 1)\mbox{]} to {\itshape interpolate}({\itshape values}\mbox{[}0\mbox{]}, {\itshape values}\mbox{[}1\mbox{]}), {\itshape t} in \mbox{[}1 / ({\itshape n} -\/ 1), 2 / ({\itshape n} -\/ 1)\mbox{]} to {\itshape interpolate}({\itshape values}\mbox{[}1\mbox{]}, {\itshape values}\mbox{[}2\mbox{]}), and so on, where {\itshape n} = {\itshape values}.length. In effect, this is a lightweight \href{https://github.com/d3/d3-scale/blob/master/README.md\#linear-scales}{\texttt{ linear scale}}. For example, to blend through red, green and blue\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{const\ interpolate\ =\ d3.piecewise(d3.interpolateRgb.gamma(2.2),\ ["{}red"{},\ "{}green"{},\ "{}blue"{}]);}

\end{DoxyCode}


If {\itshape interpolate} is not specified, defaults to \doxylink{README.md_interpolate}{d3.interpolate}.
