This is a small utility that helps you remove the {\ttfamily position} field from nodes in a unist tree.\hypertarget{readme.md_autotoc_md34349}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{readme.md_autotoc_md34349}
Often, positional info is the whole reason, or an important reason, for using ASTs. Sometimes, especially when comparing trees, or when inserting one tree into another, the positional info is at best useless and at worst harmful. In those cases, you can use this utility to remove {\ttfamily position} fields from a tree.

You might find the utility \href{https://github.com/syntax-tree/unist-util-position}{\texttt{ {\ttfamily unist-\/util-\/position}}} useful to instead get clean position info from a tree, or \href{https://github.com/syntax-tree/unist-util-generated}{\texttt{ {\ttfamily unist-\/util-\/generated}}} useful to check whether a node is considered to be generated (not in the original input file).

You might also enjoy \href{https://github.com/syntax-tree/unist-util-stringify-position}{\texttt{ {\ttfamily unist-\/util-\/stringify-\/position}}} when you want to display positional info to users.\hypertarget{readme.md_autotoc_md34350}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md34350}
