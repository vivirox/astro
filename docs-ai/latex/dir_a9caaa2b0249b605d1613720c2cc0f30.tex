\doxysubsection{Detailed Description}
Say you have some code that draws to a 2D canvas\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{function\ drawCircle(context,\ radius)\ \{}
\DoxyCodeLine{\ \ context.moveTo(radius,\ 0);}
\DoxyCodeLine{\ \ context.arc(0,\ 0,\ radius,\ 0,\ 2\ *\ Math.PI);}
\DoxyCodeLine{\}}

\end{DoxyCode}


The d3-\/path module lets you take this exact code and additionally render to \href{http://www.w3.org/TR/SVG/paths.html}{\texttt{ SVG}}. It works by \doxylink{README.md_path_toString}{serializing} \href{http://www.w3.org/TR/2dcontext/\#canvaspathmethods}{\texttt{ Canvas\+Path\+Methods}} calls to \href{http://www.w3.org/TR/SVG/paths.html\#PathData}{\texttt{ SVG path data}}. For example\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{const\ context\ =\ d3.path();}
\DoxyCodeLine{drawCircle(context,\ 40);}
\DoxyCodeLine{pathElement.setAttribute("{}d"{},\ context.toString());}

\end{DoxyCode}


Now code you write once can be used with both Canvas (for performance) and SVG (for convenience). For a practical example, see \href{https://github.com/d3/d3-shape}{\texttt{ d3-\/shape}}.\hypertarget{README.md_autotoc_md11236}{}\doxysubsection{\texorpdfstring{Installing}{Installing}}\label{README.md_autotoc_md11236}
