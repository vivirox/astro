This package is a utility that takes HTML input and turns it into a hast syntax tree.\hypertarget{readme.md_autotoc_md16610}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{readme.md_autotoc_md16610}
If you want to handle syntax trees manually, use this.

Use \href{https://github.com/inikulin/parse5}{\texttt{ {\ttfamily parse5}}} instead when you just want to parse HTML and don‚Äôt care about \href{https://github.com/syntax-tree/hast}{\texttt{ hast}}. You can also use \href{https://github.com/syntax-tree/hast-util-from-parse5}{\texttt{ {\ttfamily hast-\/util-\/from-\/parse5}}} and \href{https://github.com/inikulin/parse5}{\texttt{ {\ttfamily parse5}}} yourself, or use the rehype plugin \href{https://github.com/rehypejs/rehype/tree/main/packages/rehype-parse\#readme}{\texttt{ {\ttfamily rehype-\/parse}}}, which wraps this utility to also parse HTML at a higher-\/level (easier) abstraction. \href{https://github.com/syntax-tree/xast-util-from-xml}{\texttt{ {\ttfamily xast-\/util-\/from-\/xml}}} can be used if you are dealing with XML instead of HTML.

If you might run in a browser and prefer a ligher alternative, while not caring about positional info, parse errors, and consistency across browsers, use \href{https://github.com/syntax-tree/hast-util-from-html-isomorphic}{\texttt{ {\ttfamily hast-\/util-\/from-\/html-\/isomorphic}}}, which wraps this in Node and uses browser APIs otherwise.

Finally you can use the utility \href{https://github.com/syntax-tree/hast-util-to-html}{\texttt{ {\ttfamily hast-\/util-\/to-\/html}}} for the inverse of this utility. It turns hast into HTML.\hypertarget{readme.md_autotoc_md16611}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md16611}
