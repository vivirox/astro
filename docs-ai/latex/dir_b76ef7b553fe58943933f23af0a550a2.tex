\hypertarget{README.md_autotoc_md26025}{}\doxysubsubsection{\texorpdfstring{Resilient • No flaky tests}{Resilient • No flaky tests}}\label{README.md_autotoc_md26025}
{\bfseries{Auto-\/wait}}. Playwright waits for elements to be actionable prior to performing actions. It also has a rich set of introspection events. The combination of the two eliminates the need for artificial timeouts -\/ a primary cause of flaky tests.

{\bfseries{Web-\/first assertions}}. Playwright assertions are created specifically for the dynamic web. Checks are automatically retried until the necessary conditions are met.

{\bfseries{Tracing}}. Configure test retry strategy, capture execution trace, videos and screenshots to eliminate flakes.\hypertarget{README.md_autotoc_md26026}{}\doxysubsubsection{\texorpdfstring{No trade-\/offs • No limits}{No trade-\/offs • No limits}}\label{README.md_autotoc_md26026}
Browsers run web content belonging to different origins in different processes. Playwright is aligned with the architecture of the modern browsers and runs tests out-\/of-\/process. This makes Playwright free of the typical in-\/process test runner limitations.

{\bfseries{Multiple everything}}. Test scenarios that span multiple tabs, multiple origins and multiple users. Create scenarios with different contexts for different users and run them against your server, all in one test.

{\bfseries{Trusted events}}. Hover elements, interact with dynamic controls and produce trusted events. Playwright uses real browser input pipeline indistinguishable from the real user.

Test frames, pierce Shadow DOM. Playwright selectors pierce shadow DOM and allow entering frames seamlessly.\hypertarget{README.md_autotoc_md26027}{}\doxysubsubsection{\texorpdfstring{Full isolation • Fast execution}{Full isolation • Fast execution}}\label{README.md_autotoc_md26027}
{\bfseries{Browser contexts}}. Playwright creates a browser context for each test. Browser context is equivalent to a brand new browser profile. This delivers full test isolation with zero overhead. Creating a new browser context only takes a handful of milliseconds.

{\bfseries{Log in once}}. Save the authentication state of the context and reuse it in all the tests. This bypasses repetitive log-\/in operations in each test, yet delivers full isolation of independent tests.\hypertarget{README.md_autotoc_md26028}{}\doxysubsubsection{\texorpdfstring{Powerful Tooling}{Powerful Tooling}}\label{README.md_autotoc_md26028}
{\bfseries{\href{https://playwright.dev/docs/codegen}{\texttt{ Codegen}}}}. Generate tests by recording your actions. Save them into any language.

{\bfseries{\href{https://playwright.dev/docs/inspector}{\texttt{ Playwright inspector}}}}. Inspect page, generate selectors, step through the test execution, see click points and explore execution logs.

{\bfseries{\href{https://playwright.dev/docs/trace-viewer}{\texttt{ Trace Viewer}}}}. Capture all the information to investigate the test failure. Playwright trace contains test execution screencast, live DOM snapshots, action explorer, test source and many more.
