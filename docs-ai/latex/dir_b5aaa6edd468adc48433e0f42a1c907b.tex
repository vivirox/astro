\hypertarget{README.md_autotoc_md3789}{}\doxysubsubsection{\texorpdfstring{Handling {\ttfamily ctrl+c} gracefully}{Handling {\ttfamily ctrl+c} gracefully}}\label{README.md_autotoc_md3789}
When a user press {\ttfamily ctrl+c} to exit a prompt, Inquirer rejects the prompt promise. This is the expected behavior in order to allow your program to teardown/cleanup its environment. When using {\ttfamily async/await}, rejected promises throw their error. When unhandled, those errors print their stack trace in your user\textquotesingle{}s terminal.


\begin{DoxyCode}{0}
\DoxyCodeLine{ExitPromptError:\ User\ force\ closed\ the\ prompt\ with\ 0\ null}
\DoxyCodeLine{\ \ at\ file://example/packages/core/dist/esm/lib/create-\/prompt.js:55:20}
\DoxyCodeLine{\ \ at\ Emitter.emit\ (file://example/node\_modules/signal-\/exit/dist/mjs/index.js:67:19)}
\DoxyCodeLine{\ \ at\ \#processEmit\ (file://example/node\_modules/signal-\/exit/dist/mjs/index.js:236:27)}
\DoxyCodeLine{\ \ at\ \#process.emit\ (file://example/node\_modules/signal-\/exit/dist/mjs/index.js:187:37)}
\DoxyCodeLine{\ \ at\ process.callbackTrampoline\ (node:internal/async\_hooks:130:17)}

\end{DoxyCode}


This isn\textquotesingle{}t a great UX, which is why we highly recommend you to handle those errors gracefully.

First option is to wrap your scripts in {\ttfamily try/catch}; like \href{https://github.com/SBoudrias/Inquirer.js/blob/649e78147cbb6390a162ff842d4b21d53a233472/packages/demo/src/index.ts\#L89-L95}{\texttt{ we do in our demo program}}. Or handle the error in your CLI framework mechanism; for example \href{https://mael.dev/clipanion/docs/errors\#custom-error-handling}{\texttt{ {\ttfamily Clipanion catch} method}}.

Lastly, you could handle the error globally with an event listener and silence it.

\hypertarget{README.md_autotoc_md3793}{}\doxysubsubsection{\texorpdfstring{Using as pre-\/commit/git hooks, or scripts}{Using as pre-\/commit/git hooks, or scripts}}\label{README.md_autotoc_md3793}
By default scripts ran from tools like {\ttfamily husky}/{\ttfamily lint-\/staged} might not run inside an interactive shell. In non-\/interactive shell, Inquirer cannot run, and users cannot send keypress events to the process.

For it to work, you must make sure you start a {\ttfamily tty} (or "{}interactive"{} input stream.)

If those scripts are set within your {\ttfamily package.\+json}, you can define the stream like so\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{"{}precommit"{}:\ "{}my-\/script\ <\ /dev/tty"{}}

\end{DoxyCode}

\begin{DoxyCode}{0}

\DoxyCodeLine{exec\ <\ /dev/tty}
