\hypertarget{readme.md_autotoc_md11671}{}\doxysubsubsection{\texorpdfstring{{\ttfamily merge(x, y, \mbox{[}options\mbox{]})}}{{\ttfamily merge(x, y, \mbox{[}options\mbox{]})}}}\label{readme.md_autotoc_md11671}
Merge two objects {\ttfamily x} and {\ttfamily y} deeply, returning a new merged object with the elements from both {\ttfamily x} and {\ttfamily y}.

If an element at the same key is present for both {\ttfamily x} and {\ttfamily y}, the value from {\ttfamily y} will appear in the result.

Merging creates a new object, so that neither {\ttfamily x} or {\ttfamily y} is modified.

{\bfseries{Note\+:}} By default, arrays are merged by concatenating them.\hypertarget{readme.md_autotoc_md11672}{}\doxysubsubsection{\texorpdfstring{{\ttfamily merge.\+all(array\+Of\+Objects, \mbox{[}options\mbox{]})}}{{\ttfamily merge.\+all(array\+Of\+Objects, \mbox{[}options\mbox{]})}}}\label{readme.md_autotoc_md11672}

\hypertarget{readme.md_autotoc_md11677}{}\doxysubsubsubsection{\texorpdfstring{{\ttfamily is\+Mergeable\+Object}}{{\ttfamily is\+Mergeable\+Object}}}\label{readme.md_autotoc_md11677}
By default, deepmerge clones every property from almost every kind of object.

You may not want this, if your objects are of special types, and you want to copy the whole object instead of just copying its properties.

You can accomplish this by passing in a function for the {\ttfamily is\+Mergeable\+Object} option.

If you only want to clone properties of plain objects, and ignore all "{}special"{} kinds of instantiated objects, you probably want to drop in \href{https://github.com/jonschlinkert/is-plain-object}{\texttt{ {\ttfamily is-\/plain-\/object}}}.

The {\ttfamily custom\+Merge} function will be passed the key for each property, and should return the function which should be used to merge the values for that property.

It may also return undefined, in which case the default merge behaviour will be used.
