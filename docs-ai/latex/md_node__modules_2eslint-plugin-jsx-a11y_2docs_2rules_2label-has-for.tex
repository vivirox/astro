Enforce label tags have associated control.

There are two supported ways to associate a label with a control\+:


\begin{DoxyItemize}
\item nesting\+: by wrapping a control in a label tag
\item id\+: by using the prop {\ttfamily html\+For} (or any configured attribute) as in {\ttfamily html\+For=\mbox{[}ID of control\mbox{]}}
\end{DoxyItemize}

To fully cover 100\% of assistive devices, you\textquotesingle{}re encouraged to validate for both nesting and id.\hypertarget{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2label-has-for_autotoc_md13849}{}\doxysection{\texorpdfstring{Rule options}{Rule options}}\label{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2label-has-for_autotoc_md13849}

The {\ttfamily required} option (defaults to {\ttfamily "{}required"{}\+: \{ "{}every"{}\+: \mbox{[}"{}nesting"{}, "{}id"{}\mbox{]} \}}) determines which checks are activated. You\textquotesingle{}re allowed to pass in one of the following types\+:


\begin{DoxyItemize}
\item string\+: must be one of the acceptable strings ({\ttfamily "{}nesting"{}} or {\ttfamily "{}id"{}})
\item object, must have one of the following properties\+:
\begin{DoxyItemize}
\item some\+: an array of acceptable strings, will pass if ANY of the requested checks passed
\item every\+: an array of acceptable strings, will pass if ALL of the requested checks passed
\end{DoxyItemize}
\end{DoxyItemize}

The {\ttfamily allow\+Children} option (defaults to {\ttfamily false}) determines whether {\ttfamily \{children\}} content is allowed to be passed into a {\ttfamily label} element. For example, the following pattern, by default, is not allowed\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{<label>\{children\}</label>}

\end{DoxyCode}


However, if {\ttfamily allow\+Children} is set to {\ttfamily true}, no error will be raised. If you want to pass in {\ttfamily \{children\}} content without raising an error, because you cannot be sure what {\ttfamily \{children\}} will render, then set {\ttfamily allow\+Children} to {\ttfamily true}.

Note that passing props as spread attribute without {\ttfamily html\+For} explicitly defined will cause this rule to fail. Explicitly pass down {\ttfamily html\+For} prop for rule to pass. The prop must have an actual value to pass.
