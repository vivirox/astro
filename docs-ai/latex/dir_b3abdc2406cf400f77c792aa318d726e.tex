\doxysubsection{Detailed Description}
This is a big collection of open source vector icons, all validated, cleaned up and converted to the same easy to use format.

Even though all icon sets are open source, some icon sets require attribution.

See \doxysectlink{md_node__modules_2_0diconify_2json_2collections}{collections.md}{0} for list of icon sets and their licenses.\hypertarget{readme.md_autotoc_md3637}{}\doxysubsection{\texorpdfstring{Validation and clean up}{Validation and clean up}}\label{readme.md_autotoc_md3637}
All icons have been processed with \href{https://iconify.design/docs/libraries/tools/}{\texttt{ Iconify Tools}} to clean them up.

Icon parsing process includes\+:


\begin{DoxyItemize}
\item Very strict validation and clean up. Icons do not contain scripts, event listeners, fonts, raster images, external resources and unknown elements.
\item Colors for monotone icons have been replaced with {\ttfamily current\+Color}, making it easy to change icon color by changing text color.
\item Icon content has been optimised to reduce its size.
\end{DoxyItemize}\hypertarget{readme.md_autotoc_md3638}{}\doxysubsection{\texorpdfstring{Maintenance}{Maintenance}}\label{readme.md_autotoc_md3638}
This repository is automatically updated several times a week, so it always contains the latest icons for all icon sets.\hypertarget{readme.md_autotoc_md3639}{}\doxysubsection{\texorpdfstring{Format}{Format}}\label{readme.md_autotoc_md3639}
Icon sets are stored in {\ttfamily Iconify\+JSON} format. Type\+Script definition is available in {\ttfamily @iconify/types} package. Documentation is \href{https://iconify.design/docs/types/iconify-json.html}{\texttt{ available on Iconify Documentation website}}.

To work with icon sets, use \href{https://iconify.design/docs/libraries/utils/}{\texttt{ Iconify Utils}}. Utils package works in any Java\+Script environment\+: Node.\+js, Deno, browsers, isolated Java\+Script environments.\hypertarget{readme.md_autotoc_md3640}{}\doxysubsection{\texorpdfstring{Usage}{Usage}}\label{readme.md_autotoc_md3640}
These icons can be used with many tools, plugins and components. They can also be exported as individual SVG files.

See \href{https://iconify.design/docs/usage/}{\texttt{ Iconify documentation}} for more details.\hypertarget{readme.md_autotoc_md3641}{}\doxysubsection{\texorpdfstring{How to get this repository}{How to get this repository}}\label{readme.md_autotoc_md3641}

\hypertarget{readme.md_autotoc_md3644}{}\doxysubsection{\texorpdfstring{Data format}{Data format}}\label{readme.md_autotoc_md3644}
Icons used by Iconify are in directory json, in Iconify JSON format.

Why JSON instead of SVG? There are several reasons for that\+:


\begin{DoxyItemize}
\item Easy to store images in bulk.
\item Contains only content of icon without {\ttfamily \texorpdfstring{$<$}{<}svg\texorpdfstring{$>$}{>}} element, making it easy to manipulate content without doing complex parsing. It also makes it easier to create components, such as React icon component, allowing to use framework native SVG element.
\item Data can contain additional content\+: aliases for icons, icon set information, categories/tags/themes.
\end{DoxyItemize}

Why not XML?


\begin{DoxyItemize}
\item JSON is much easier to parse without additional tools. All languages support it.
\end{DoxyItemize}

Format of json file is very simple\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\{}
\DoxyCodeLine{\ \ "{}prefix"{}:\ "{}mdi-\/light"{},}
\DoxyCodeLine{\ \ "{}icons"{}:\ \{}
\DoxyCodeLine{\ \ \ \ "{}icon-\/name"{}:\ \{}
\DoxyCodeLine{\ \ \ \ \ \ "{}body"{}:\ "{}<g\ />"{},}
\DoxyCodeLine{\ \ \ \ \ \ "{}width"{}:\ 24,}
\DoxyCodeLine{\ \ \ \ \ \ "{}height"{}:\ 24}
\DoxyCodeLine{\ \ \ \ \}}
\DoxyCodeLine{\ \ \},}
\DoxyCodeLine{\ \ "{}aliases"{}:\ \{}
\DoxyCodeLine{\ \ \ \ "{}icon-\/alias"{}:\ \{}
\DoxyCodeLine{\ \ \ \ \ \ "{}parent"{}:\ "{}icon-\/name"{}}
\DoxyCodeLine{\ \ \ \ \}}
\DoxyCodeLine{\ \ \}}
\DoxyCodeLine{\}}

\end{DoxyCode}


"{}icons"{} object contains list of all icons.

Each icon has following properties\+:


\begin{DoxyItemize}
\item body\+: icon body.
\item left, top\+: left and top coordinates of view\+Box, default is 0.
\item width, height\+: dimensions of view\+Box, default is 16.
\item rotate\+: rotation. Default = 0. Values\+: 0 = 0deg, 1 = 90deg, 2 = 180deg, 3 = 270deg.
\item h\+Flip\+: horizontal flip. Boolean value, default = false.
\item v\+Flip\+: vertical flip. Boolean value, default = false.
\item hidden\+: if set to true, icon is hidden. That means icon was removed from collection for some reason, but it is kept in JSON file to prevent applications that rely on old icon from breaking.
\end{DoxyItemize}

Optional "{}aliases"{} object contains list of aliases for icons. Format is similar to "{}icons"{} object, but without "{}body"{} property and with additional property "{}parent"{} that points to parent icon. Transformation properties (rotate, h\+Flip, v\+Flip) are merged with parent icon\textquotesingle{}s properties. Any other properties overwrite properties of parent icon.

When multiple icons have the same value, it is moved to root object to reduce duplication\+:
