Non-\/interactive HTML elements and non-\/interactive ARIA roles indicate {\itshape content} and {\itshape containers} in the user interface. A non-\/interactive element does not support event handlers (mouse and key handlers). Non-\/interactive elements include {\ttfamily \texorpdfstring{$<$}{<}main\texorpdfstring{$>$}{>}}, {\ttfamily \texorpdfstring{$<$}{<}area\texorpdfstring{$>$}{>}}, {\ttfamily \texorpdfstring{$<$}{<}h1\texorpdfstring{$>$}{>}} (,{\ttfamily \texorpdfstring{$<$}{<}h2\texorpdfstring{$>$}{>}}, etc), {\ttfamily \texorpdfstring{$<$}{<}p\texorpdfstring{$>$}{>}}, {\ttfamily \texorpdfstring{$<$}{<}img\texorpdfstring{$>$}{>}}, {\ttfamily \texorpdfstring{$<$}{<}li\texorpdfstring{$>$}{>}}, {\ttfamily \texorpdfstring{$<$}{<}ul\texorpdfstring{$>$}{>}} and {\ttfamily \texorpdfstring{$<$}{<}ol\texorpdfstring{$>$}{>}}. Non-\/interactive \href{https://www.w3.org/TR/wai-aria-1.1/\#usage_intro}{\texttt{ WAI-\/\+ARIA roles}} include {\ttfamily article}, {\ttfamily banner}, {\ttfamily complementary}, {\ttfamily img}, {\ttfamily listitem}, {\ttfamily main}, {\ttfamily region} and {\ttfamily tooltip}.\hypertarget{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2no-noninteractive-element-interactions_autotoc_md13905}{}\doxysection{\texorpdfstring{How do I resolve this error?}{How do I resolve this error?}}\label{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2no-noninteractive-element-interactions_autotoc_md13905}
\hypertarget{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2no-noninteractive-element-interactions_autotoc_md13906}{}\doxysubsection{\texorpdfstring{Case\+: This element acts like a button, link, menuitem, etc}{Case\+: This element acts like a button, link, menuitem, etc}}\label{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2no-noninteractive-element-interactions_autotoc_md13906}

This leaves the {\itshape content} or {\itshape container} semantic value of this element intact.

Note\+: Adding a role to your element does {\bfseries{not}} add behavior. When a semantic HTML element like {\ttfamily \texorpdfstring{$<$}{<}button\texorpdfstring{$>$}{>}} is used, then it will also respond to Enter key presses when it has focus. The developer is responsible for providing the expected behavior of an element that the role suggests it would have\+: focusability and key press support. see \href{https://www.w3.org/TR/wai-aria-practices-1.1/\#aria_ex}{\texttt{ WAI-\/\+ARIA Authoring Practices Guide -\/ Design Patterns and Widgets}}.\hypertarget{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2no-noninteractive-element-interactions_autotoc_md13907}{}\doxysubsection{\texorpdfstring{Case\+: This element is catching bubbled events from elements that it contains}{Case\+: This element is catching bubbled events from elements that it contains}}\label{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2no-noninteractive-element-interactions_autotoc_md13907}

Marking an element with the role {\ttfamily presentation} indicates to assistive technology that this element should be ignored; it exists to support the web application and is not meant for humans to interact with directly.\hypertarget{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2no-noninteractive-element-interactions_autotoc_md13908}{}\doxysubsection{\texorpdfstring{Case\+: This is a heading that expands/collapses content on the package}{Case\+: This is a heading that expands/collapses content on the package}}\label{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2no-noninteractive-element-interactions_autotoc_md13908}
Headers often double as expand/collapse controls for the content they headline. An accordion component is a common example of this pattern. Rather than assign the interaction handling code to the heading itself, put a button inside the heading instead. This pattern retains the role of the heading and the role of the button.

Table cells (and tables in general) are meant to contain data. ARIA provides us with a construct called a \href{https://w3c.github.io/aria-practices/\#grid}{\texttt{ Grid}} that is essentially a 2 dimensional logical container for content and interactive elements.
