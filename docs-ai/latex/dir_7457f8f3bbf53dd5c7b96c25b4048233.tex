\hypertarget{README.md_autotoc_md23917}{}\doxysubsubsection{\texorpdfstring{Why use micromatch?}{Why use micromatch?}}\label{README.md_autotoc_md23917}
\begin{quote}
micromatch is a replacement for minimatch and multimatch \end{quote}



\begin{DoxyItemize}
\item Supports all of the same matching features as \href{https://github.com/isaacs/minimatch}{\texttt{ minimatch}} and \href{https://github.com/sindresorhus/multimatch}{\texttt{ multimatch}}
\item More complete support for the Bash 4.\+3 specification than minimatch and multimatch. Micromatch passes {\itshape all of the spec tests} from bash, including some that bash still fails.
\item {\bfseries{Fast \& Performant}} -\/ Loads in about 5ms and performs fast matches.
\item {\bfseries{Glob matching}} -\/ Using wildcards ({\ttfamily \texorpdfstring{$\ast$}{*}} and {\ttfamily ?}), globstars ({\ttfamily \texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}}) for nested directories
\item {\bfseries{Advanced globbing}} -\/ Supports extglobs, braces, and POSIX brackets, and support for escaping special characters with {\ttfamily \textbackslash{}} or quotes.
\item {\bfseries{Accurate}} -\/ Covers more scenarios \href{https://github.com/yarnpkg/yarn/pull/3339}{\texttt{ than minimatch}}
\item {\bfseries{Well tested}} -\/ More than 5,000 \href{./test}{\texttt{ test assertions}}
\item {\bfseries{Windows support}} -\/ More reliable windows support than minimatch and multimatch.
\item {\bfseries{\href{https://github.com/micromatch/braces\#braces-is-safe}{\texttt{ Safe}}}} -\/ Micromatch is not subject to DoS with brace patterns like minimatch and multimatch.
\end{DoxyItemize}\hypertarget{README.md_autotoc_md23918}{}\doxysubsubsubsection{\texorpdfstring{Matching features}{Matching features}}\label{README.md_autotoc_md23918}

\begin{DoxyItemize}
\item Support for multiple glob patterns (no need for wrappers like multimatch)
\item Wildcards ({\ttfamily \texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}}, {\ttfamily \texorpdfstring{$\ast$}{*}.js})
\item Negation (`'!a/\texorpdfstring{$\ast$}{*}.js\textquotesingle{}{\ttfamily ,}\textquotesingle{}\texorpdfstring{$\ast$}{*}!(b).js\textquotesingle{}{\ttfamily )}
\item {\ttfamily \mbox{[}extglobs\mbox{]}(\#extglobs) (}+(x\texorpdfstring{$\vert$}{|}y){\ttfamily ,}!(a\texorpdfstring{$\vert$}{|}b){\ttfamily )}
\item {\ttfamily \mbox{[}POSIX character classes\mbox{]}(\#posix-\/bracket-\/expressions) (}\mbox{[}\mbox{[}\+:alpha\+:\mbox{]}\mbox{[}\+:digit\+:\mbox{]}\mbox{]}{\ttfamily )}
\item {\ttfamily \mbox{[}brace expansion\mbox{]}(\href{https://github.com/micromatch/braces}{\texttt{ https\+://github.\+com/micromatch/braces}}) (}foo/\{1..5\}.md{\ttfamily ,}bar/\{a,b,c\}.js{\ttfamily )}
\item {\ttfamily regex character classes (}foo-\/\mbox{[}1-\/5\mbox{]}.js{\ttfamily )}
\item {\ttfamily regex logical "{}or"{} (}foo/(abc\texorpdfstring{$\vert$}{|}xyz).js\`{})
\end{DoxyItemize}

You can mix and match these features to create whatever patterns you need!\hypertarget{README.md_autotoc_md23919}{}\doxysubsubsection{\texorpdfstring{Switching to micromatch}{Switching to micromatch}}\label{README.md_autotoc_md23919}

\hypertarget{README.md_autotoc_md23961}{}\doxysubsubsubsection{\texorpdfstring{Bash 4.\+3 parity}{Bash 4.\+3 parity}}\label{README.md_autotoc_md23961}
Whenever possible matching behavior is based on behavior Bash 4.\+3, which is mostly consistent with minimatch.

However, it\textquotesingle{}s suprising how many edge cases and rabbit holes there are with glob matching, and since there is no real glob specification, and micromatch is more accurate than both Bash and minimatch, there are cases where best-\/guesses were made for behavior. In a few cases where Bash had no answers, we used wildmatch (used by git) as a fallback.\hypertarget{README.md_autotoc_md23962}{}\doxysubsubsubsection{\texorpdfstring{Backslashes}{Backslashes}}\label{README.md_autotoc_md23962}
There is an important, notable difference between minimatch and micromatch {\itshape in regards to how backslashes are handled} in glob patterns.


\begin{DoxyItemize}
\item Micromatch exclusively and explicitly reserves backslashes for escaping characters in a glob pattern, even on windows, which is consistent with bash behavior. {\itshape More importantly, unescaping globs can result in unsafe regular expressions}.
\item Minimatch converts all backslashes to forward slashes, which means you can\textquotesingle{}t use backslashes to escape any characters in your glob patterns.
\end{DoxyItemize}

We made this decision for micromatch for a couple of reasons\+:


\begin{DoxyItemize}
\item Consistency with bash conventions.
\item Glob patterns are not filepaths. They are a type of \href{https://en.wikipedia.org/wiki/Regular_language}{\texttt{ regular language}} that is converted to a Java\+Script regular expression. Thus, when forward slashes are defined in a glob pattern, the resulting regular expression will match windows or POSIX path separators just fine.
\end{DoxyItemize}

{\bfseries{A note about joining paths to globs}}

Note that when you pass something like `path.\+join(\textquotesingle{}foo', \textquotesingle{}\texorpdfstring{$\ast$}{*}\textquotesingle{}){\ttfamily to micromatch, you are creating a filepath and expecting it to still work as a glob pattern. This causes problems on windows, since the}path.\+sep{\ttfamily is}\textbackslash{}\`{}.

In other words, since {\ttfamily \textbackslash{}\textbackslash{}} is reserved as an escape character in globs, on windows `path.\+join(\textquotesingle{}foo', \textquotesingle{}{\itshape \textquotesingle{}){\ttfamily would result in}foo\textbackslash{}\texorpdfstring{$\ast$}{*}{\ttfamily , which tells micromatch to match}}\`{} as a literal character. This is the same behavior as bash.

To solve this, you might be inspired to do something like `\textquotesingle{}foo\textbackslash{}\texorpdfstring{$\ast$}{*}'.replace(/\textbackslash{}/g, \textquotesingle{}/\textquotesingle{})\`{}, but this causes another, potentially much more serious, problem.\hypertarget{README.md_autotoc_md23963}{}\doxysubsubsection{\texorpdfstring{Benchmarks}{Benchmarks}}\label{README.md_autotoc_md23963}
