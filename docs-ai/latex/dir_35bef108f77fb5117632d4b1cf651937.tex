About each config\+:


\begin{DoxyItemize}
\item {\ttfamily recommended}\+: Considers both Common\+JS and ES Modules. If \href{https://medium.com/@nodejs/announcing-a-new-experimental-modules-1be8d2d6c2ff\#b023}{\texttt{ {\ttfamily "{}type"{}\+:"{}module"{}} field}} existed in package.\+json then it considers files as ES Modules. Otherwise it considers files as Common\+JS. In addition, it considers {\ttfamily \texorpdfstring{$\ast$}{*}.mjs} files as ES Modules and {\ttfamily \texorpdfstring{$\ast$}{*}.cjs} files as Common\+JS.
\item {\ttfamily recommended-\/module}\+: Considers all files as ES Modules.
\item {\ttfamily recommended-\/script}\+: Considers all files as Common\+JS.
\end{DoxyItemize}

These preset configs\+:


\begin{DoxyItemize}
\item enable \href{http://eslint.org/docs/rules/no-process-exit}{\texttt{ no-\/process-\/exit}} rule because \href{https://nodejs.org/api/process.html\#process_process_exit_code}{\texttt{ the official document}} does not recommend a use of {\ttfamily process.\+exit()}.
\item enable plugin rules indicated by emojis in the rules table.
\item add {\ttfamily \{ecma\+Version\+: 2021\}} and etc into {\ttfamily parser\+Options}.
\item add proper globals into {\ttfamily globals}.
\item add this plugin into {\ttfamily plugins}.

\item A support for old Node version is dropped.
