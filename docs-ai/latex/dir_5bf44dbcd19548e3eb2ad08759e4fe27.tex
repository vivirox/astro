On recent {\ttfamily Node} versions, you may select this runner to execute your worker file (a {\ttfamily .ts} file) in the native runtime.

As of {\ttfamily Node} v23.\+6, this feature is supported out of the box. For {\ttfamily Node} {\ttfamily \texorpdfstring{$>$}{>}=22.\+6 \texorpdfstring{$<$}{<}23.\+6}, this feature is supported via {\ttfamily -\/-\/experimental-\/strip-\/types} flag. Visit the \href{https://nodejs.org/docs/latest/api/typescript.html\#type-stripping}{\texttt{ documentation}} to learn more.

When {\ttfamily synckit} detects the process to be running with supported {\ttfamily Node} versions (\texorpdfstring{$>$}{>}=22.\+6), it will execute the worker file with the {\ttfamily node} runner by default, you can disable this behavior by setting {\ttfamily -\/-\/no-\/experimental-\/strip-\/types} flag via {\ttfamily NODE\+\_\+\+OPTIONS} env or cli arg.\hypertarget{README.md_autotoc_md13578}{}\doxysubsubsubsection{\texorpdfstring{{\ttfamily bun} (Default, Bun)}{{\ttfamily bun} (Default, Bun)}}\label{README.md_autotoc_md13578}
\href{https://bun.sh/docs/typescript}{\texttt{ {\ttfamily Bun}}} supports {\ttfamily Type\+Script} natively.

When {\ttfamily synckit} detects the process to be running with {\ttfamily Bun}, it will execute the worker file with the {\ttfamily bun} runner by default.

In this case, {\ttfamily synckit} doesn\textquotesingle{}t do anything to the worker itself, it just passes through the worker directly.\hypertarget{README.md_autotoc_md13579}{}\doxysubsubsubsection{\texorpdfstring{{\ttfamily ts-\/node} (Default)}{{\ttfamily ts-\/node} (Default)}}\label{README.md_autotoc_md13579}
