\doxysubsection{Detailed Description}
\begin{quote}
The {\ttfamily path} module from Node.\+js for browsers \end{quote}


This implements the Node.\+js \href{https://nodejs.org/docs/v10.3.0/api/path.html}{\texttt{ {\ttfamily path}}} module for environments that do not have it, like browsers.

\begin{quote}
{\ttfamily path-\/browserify} currently matches the {\bfseries{Node.\+js 10.\+3}} API. \end{quote}
\hypertarget{README.md_autotoc_md25662}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{README.md_autotoc_md25662}
You usually do not have to install {\ttfamily path-\/browserify} yourself! If your code runs in Node.\+js, {\ttfamily path} is built in. If your code runs in the browser, bundlers like \href{https://github.com/browserify/browserify}{\texttt{ browserify}} or \href{https://github.com/webpack/webpack}{\texttt{ webpack}} include the {\ttfamily path-\/browserify} module by default.

But if none of those apply, with npm do\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{npm\ install\ path-\/browserify}

\end{DoxyCode}

\hypertarget{README.md_autotoc_md25664}{}\doxysubsection{\texorpdfstring{API}{API}}\label{README.md_autotoc_md25664}
See the \href{https://nodejs.org/docs/v10.3.0/api/path.html}{\texttt{ Node.\+js path docs}}. {\ttfamily path-\/browserify} currently matches the Node.\+js 10.\+3 API. {\ttfamily path-\/browserify} only implements the POSIX functions, not the win32 ones.\hypertarget{README.md_autotoc_md25665}{}\doxysubsection{\texorpdfstring{Contributing}{Contributing}}\label{README.md_autotoc_md25665}
PRs are very welcome! The main way to contribute to {\ttfamily path-\/browserify} is by porting features, bugfixes and tests from Node.\+js. Ideally, code contributions to this module are copy-\/pasted from Node.\+js and transpiled to ES5, rather than reimplemented from scratch. Matching the Node.\+js code as closely as possible makes maintenance simpler when new changes land in Node.\+js. This module intends to provide exactly the same API as Node.\+js, so features that are not available in the core {\ttfamily path} module will not be accepted. Feature requests should instead be directed at \href{https://github.com/nodejs/node}{\texttt{ nodejs/node}} and will be added to this module once they are implemented in Node.\+js.
