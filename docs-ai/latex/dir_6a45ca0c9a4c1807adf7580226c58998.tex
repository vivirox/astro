\doxysubsection{Detailed Description}
Type definitions for using Iconify icon sets with Type\+Script.\hypertarget{README.md_autotoc_md3648}{}\doxysubsection{\texorpdfstring{Files structure}{Files structure}}\label{README.md_autotoc_md3648}
Iconify icon sets are available in several formats\+:


\begin{DoxyItemize}
\item Big JSON files that combine many icons in one file
\item Node.\+js packages split into individual icons
\end{DoxyItemize}\hypertarget{README.md_autotoc_md3649}{}\doxysubsubsection{\texorpdfstring{Icon format}{Icon format}}\label{README.md_autotoc_md3649}
Each icon is represented by the {\ttfamily Iconify\+Icon} type. It is a simple object with multiple string, number or boolean attributes.

The only required attribute is\+:


\begin{DoxyItemize}
\item {\ttfamily body}\+: string. Value contains inner HTML of an icon as a string, for example {\ttfamily \texorpdfstring{$<$}{<}path d="{}..."{}/\texorpdfstring{$>$}{>}}.
\end{DoxyItemize}

Optional attributes are represented by type {\ttfamily Iconify\+Optional}. They are split into several types\+: dimensions ({\ttfamily Iconify\+Dimenisons} type) and transformations ({\ttfamily Iconify\+Transformations} type).

Dimensions attributes\+:


\begin{DoxyItemize}
\item {\ttfamily width}\+: number. view\+Box width, number. If missing, value is set to 16.
\item {\ttfamily height}\+: number. view\+Box height, number. If missing, value is set to 16.
\item {\ttfamily left}\+: number. view\+Box left, number. If missing, the value is set to 0.
\item {\ttfamily top}\+: number. view\+Box top, number. If missing, the value is set to 0.
\end{DoxyItemize}

Transformations\+:


\begin{DoxyItemize}
\item {\ttfamily rotate}\+: number. Icon rotation. Iconify icons can be rotated in 90 degrees increments, allowing to reuse the same source icon for multiple icons, such as arrow-\/up being a copy of arrow-\/left rotated by 90 degrees. Values are 0 for 0 degrees, 1 for 90 degrees, 2 for 180 degrees, 3 for 270 degrees. The default value is 0.
\item {\ttfamily h\+Flip}\+: boolean. Horizontal flip. Similar to the rotation transformation, an icon can be flipped horizontally and vertically. It can be used to quickly create aliases, such as arrow-\/left being an alias of arrow-\/right, but with h\+Flip set to true. The default value is false.
\item {\ttfamily v\+Flip}\+: boolean. Vertical flip. The default value is false.
\end{DoxyItemize}

Iconify icon sets format is available from multiple sources\+:


\begin{DoxyItemize}
\item NPM package {\ttfamily @iconify/json} that includes all icon sets
\item API responses used by SVG framework
\end{DoxyItemize}

Icon set format structure is available as the {\ttfamily Iconify\+JSON} type. It is an object with several fields\+:


\begin{DoxyItemize}
\item {\ttfamily prefix}\+: string. Icon set prefix.
\item {\ttfamily icons}\+: object. Icons data. Value is an object that represents a set of icons, where the key is an icon name and value is {\ttfamily Iconify\+Icon} object (see "{}\+Icon format"{} above).
\item {\ttfamily aliases}\+: object. Icon aliases, similar to the {\ttfamily icons} object (see "{}\+Aliases"{} section below).
\end{DoxyItemize}

All icon properties except for {\ttfamily body} are optional and are represented by type {\ttfamily Iconify\+Optional}. Type {\ttfamily Iconify\+JSON} also extends type {\ttfamily Iconify\+Optional}, allowing all optional properties to be placed in the root object.

If an icon is missing a property, look in the root object for the default value. If the root object does not have the default value, use Iconify default value for that property (see list of properties and default values in the "{}\+Icon format"{} section above).

Default values in the root object make it possible to reduce duplication.

If both icon and alias have same attribute, following rules apply\+:


\begin{DoxyItemize}
\item {\ttfamily rotate}\+: attributes are combined. For example, icon has rotate = 1, alias has rotate = 1. Result will have rotate = 2. To prevent overflow, if rotate \texorpdfstring{$>$}{>} 3, rotate = rotate -\/ 4.
\item {\ttfamily h\+Flip} and {\ttfamily v\+Flip}\+: attributes are combined. For example, icon has h\+Flip = true, alias also has h\+Flip = true (icon.\+h\+Flip !== alias.\+h\+Flip). Result is false. false + false = false, false + true = true, true + true = false.
\item other attributes are overwritten.
\end{DoxyItemize}

\hypertarget{README.md_autotoc_md3655}{}\doxysubsubsubsubsection{\texorpdfstring{Info}{Info}}\label{README.md_autotoc_md3655}
Information block is part of the metadata, it is used for browsing or searching icon sets. It also contains the license for icons in the icon set and information about the author.

Info block is represented by the type {\ttfamily Iconify\+Info}. You can see an example above in "{}info"{} property.

Iconify\+Info type has the following properties, most of them are optional\+:


\begin{DoxyItemize}
\item {\ttfamily name}\+: string. Icon set name. This field is always set.
\item {\ttfamily total}\+: number. The total number of icons, optional.
\item {\ttfamily version}\+: string. The current version, optional.
\item {\ttfamily author}\+: object. Information about the author, always set. Author information has the following properties\+:
\begin{DoxyItemize}
\item {\ttfamily name}\+: string. Author name. This field is always set.
\item {\ttfamily url}\+: string. Link to icon set, optional. Usually links to Git\+Hub repository.
\end{DoxyItemize}
\item {\ttfamily license}\+: object. Information about the license, always set. License information has the following properties\+:
\begin{DoxyItemize}
\item {\ttfamily title}\+: string. License title. This field is always set.
\item {\ttfamily spdx}\+: string. SPDX license identifier, optional.
\item {\ttfamily url}\+: string. Link to the license, optional.
\end{DoxyItemize}
\item {\ttfamily samples}\+: string\mbox{[}\mbox{]}. Value is an array of icon names that should be used as samples when showing the icon set in an icon sets list.
\item {\ttfamily height}\+: number \texorpdfstring{$\vert$}{|} number\mbox{[}\mbox{]}. Value is a number or array of numbers, values are pixel grids used in the icon set. If any icons in an icon set do not match the grid, this attribute should not be set.
\item {\ttfamily display\+Height}\+: number. The height value that should be used for displaying samples. Value is a number between 16 and 30 (inclusive).
\end{DoxyItemize}\hypertarget{README.md_autotoc_md3656}{}\doxysubsubsubsubsection{\texorpdfstring{Characters map}{Characters map}}\label{README.md_autotoc_md3656}
Characters map is part of the metadata, it is used for icon sets that are either imported from icon fonts or intended to be exported to icon font.

Characters map allows storing characters for export as well as searching icons by character used in an icon font.

It is a simple object, where the key is character code in hexadecimal form, value is an icon name.

Important\+: each icon can have multiple characters!
