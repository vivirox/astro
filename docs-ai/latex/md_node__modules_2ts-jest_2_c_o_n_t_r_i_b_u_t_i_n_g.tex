When contributing to this repository, please first discuss the change you wish to make via \href{https://github.com/kulshekhar/ts-jest/discussions}{\texttt{ discussion}} or \href{https://github.com/kulshekhar/ts-jest/issues}{\texttt{ issue}} with the owners of this repository before making a change.

Please note we have a code of conduct, please follow it in all your interactions with the project.\hypertarget{md_node__modules_2ts-jest_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md33573}{}\doxysection{\texorpdfstring{Workflow and Pull Requests}{Workflow and Pull Requests}}\label{md_node__modules_2ts-jest_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md33573}
The team will monitor pull requests. We\textquotesingle{}ll do our best to provide updates and feedback throughout the process.

{\itshape Before} submitting a pull request, please make sure the following is done…


\begin{DoxyEnumerate}
\item Fork the repo and create your branch from {\ttfamily main}. A guide on how to fork a repository\+: \href{https://help.github.com/articles/fork-a-repo/}{\texttt{ https\+://help.\+github.\+com/articles/fork-\/a-\/repo/}}

Open terminal (e.\+g. Terminal, i\+Term, Git Bash or Git Shell) and type\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{-\/session}
\DoxyCodeLine{\ \ \$\ git\ clone\ https://github.com/<your\_username>/ts-\/jest}
\DoxyCodeLine{\ \ \$\ cd\ ts-\/jest}
\DoxyCodeLine{\ \ \$\ git\ checkout\ -\/b\ my\_branch}

\end{DoxyCode}


Note\+: Replace {\ttfamily \texorpdfstring{$<$}{<}your\+\_\+username\texorpdfstring{$>$}{>}} with your Git\+Hub username
\item {\ttfamily ts-\/jest} uses {\ttfamily npm} for running development scripts. If you haven\textquotesingle{}t already done so, please \href{https://docs.npmjs.com/downloading-and-installing-node-js-and-npm}{\texttt{ install npm}}.
\item Make sure you have a compatible version of {\ttfamily node} installed (As of April 14th 2021, {\ttfamily v14.\+x} is recommended).


\begin{DoxyCode}{0}
\DoxyCodeLine{node\ -\/v}

\end{DoxyCode}

\item Run {\ttfamily npm ci}. {\ttfamily ts-\/jest} will automatically build source files into {\ttfamily dist/} after installing dependencies.
\item Ensure the test suite passes via {\ttfamily npm run test}.
\end{DoxyEnumerate}\hypertarget{md_node__modules_2ts-jest_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md33574}{}\doxysubsection{\texorpdfstring{Testing}{Testing}}\label{md_node__modules_2ts-jest_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md33574}
Code that is written needs to be tested to ensure that it achieves the desired behaviour. Tests either fall into a unit test or an integration test.\hypertarget{md_node__modules_2ts-jest_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md33575}{}\doxysubsubsubsection{\texorpdfstring{Unit tests}{Unit tests}}\label{md_node__modules_2ts-jest_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md33575}
The unit test files are associated with source files which are in {\ttfamily src/}. If the scope of your work only requires a unit test, this is where you will write it in. Tests here usually don\textquotesingle{}t require much if any setup.\hypertarget{md_node__modules_2ts-jest_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md33576}{}\doxysubsubsubsection{\texorpdfstring{Integration tests}{Integration tests}}\label{md_node__modules_2ts-jest_2_c_o_n_t_r_i_b_u_t_i_n_g_autotoc_md33576}
There will be situations however where the work you have done cannot be tested alone using unit tests. In situations like this, you should write an integration test for your code. The integration tests reside within the {\ttfamily e2e} directory. Within this directory, there is a {\ttfamily \+\_\+\+\_\+tests\+\_\+\+\_\+} directory. This is where you will write the integration test itself. The tests within this directory execute jest itself using {\ttfamily run-\/jest.\+ts} and assertions are usually made on one if not all the output of the following {\ttfamily status}, {\ttfamily stdout} and {\ttfamily stderr}. The other subdirectories within the {\ttfamily e2e} directory are where you will write the files that jest will run for your integration tests. Feel free to take a look at any of the tests in the {\ttfamily \+\_\+\+\_\+tests\+\_\+\+\_\+} directory within {\ttfamily e2e} to have a better sense of how it is currently being done.

It is possible to run the integration test itself manually to inspect that the new behaviour is indeed correct. Here is a small code snippet of how to do just that. This is useful when debugging a failing test.

\DoxyCodeLine{\$\ cd\ e2e/test-\/utils}

\DoxyCodeLine{PASS\ \ \_\_tests\_\_/test-\/utils.spec.ts}
\DoxyCodeLine{✓\ stub\ (3ms)}
\DoxyCodeLine{}
\DoxyCodeLine{Test\ Suites:\ 1\ passed,\ 1\ total}
\DoxyCodeLine{Tests:\ \ \ \ \ \ \ 1\ passed,\ 1\ total}
\DoxyCodeLine{Snapshots:\ \ \ 0\ total}
\DoxyCodeLine{Time:\ \ \ \ \ \ \ \ 0.232\ s,\ estimated\ 1\ s}
\DoxyCodeLine{Ran\ all\ test\ suites.}

\end{DoxyCode}

If you are making changes to the website or documentation, test the {\ttfamily website} folder and run the server to check if your changes are being displayed accurately.


\begin{DoxyEnumerate}
\item Locate to the {\ttfamily website} directory and install any website specific dependencies by typing in {\ttfamily npm ci}.
\item Following steps are to be followed for this purpose from the root directory. 
\begin{DoxyCode}{0}
\DoxyCodeLine{-\/session}
\DoxyCodeLine{\ \ \$\ cd\ website\ \ \ \ \ \ \ \#\ Only\ needed\ if\ you\ are\ not\ already\ in\ the\ website\ directory}
\DoxyCodeLine{\ \ \$\ npm\ ci}
\DoxyCodeLine{\ \ \$\ npm\ run\ lint-\/prettier\ \#\ Please\ format\ markdown\ files}
\DoxyCodeLine{\ \ \$\ npm\ run\ start}

\end{DoxyCode}

\item You can run a development server to check if the changes you made are being displayed accurately by running {\ttfamily npm run start} in the website directory.
\end{DoxyEnumerate}
