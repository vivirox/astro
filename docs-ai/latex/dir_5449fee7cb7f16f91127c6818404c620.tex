\hypertarget{README.md_autotoc_md5051}{}\doxysubsection{\texorpdfstring{{\ttfamily old} and {\ttfamily modern} mode}{{\ttfamily old} and {\ttfamily modern} mode}}\label{README.md_autotoc_md5051}
This package has two modes that are used depending on the environment and parameters of use.\hypertarget{README.md_autotoc_md5052}{}\doxysubsubsection{\texorpdfstring{old}{old}}\label{README.md_autotoc_md5052}

\begin{DoxyItemize}
\item Node.\+js below {\ttfamily 10.\+10} or when the {\ttfamily stats} option is enabled
\end{DoxyItemize}

When working in the old mode, the directory is read first ({\ttfamily fs.\+readdir}), then the type of entries is determined ({\ttfamily fs.\+lstat} and/or {\ttfamily fs.\+stat} for symbolic links).\hypertarget{README.md_autotoc_md5053}{}\doxysubsubsection{\texorpdfstring{modern}{modern}}\label{README.md_autotoc_md5053}

\begin{DoxyItemize}
\item Node.\+js 10.\+10+ and the {\ttfamily stats} option is disabled
\end{DoxyItemize}

In the modern mode, reading the directory ({\ttfamily fs.\+readdir} with the {\ttfamily with\+File\+Types} option) is combined with obtaining information about its entries. An additional call for symbolic links ({\ttfamily fs.\+stat}) is still present.

This mode makes fewer calls to the file system. It\textquotesingle{}s faster.\hypertarget{README.md_autotoc_md5054}{}\doxysubsection{\texorpdfstring{Changelog}{Changelog}}\label{README.md_autotoc_md5054}
