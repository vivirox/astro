Enforce that a label tag has a text label and an associated control.

There are two supported ways to associate a label with a control\+:


\begin{DoxyItemize}
\item Wrapping a control in a label tag.
\item Adding {\ttfamily html\+For} to a label and assigning it a DOM ID string that indicates an input on the page.
\end{DoxyItemize}

This rule checks that any {\ttfamily label} tag (or an indicated custom component that will output a {\ttfamily label} tag) either (1) wraps an {\ttfamily input} element (or an indicated custom component that will output an {\ttfamily input} tag) or (2) has an {\ttfamily html\+For} attribute and that the {\ttfamily label} tag has text content.\hypertarget{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2label-has-associated-control_autotoc_md13836}{}\doxysection{\texorpdfstring{How do I resolve this error?}{How do I resolve this error?}}\label{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2label-has-associated-control_autotoc_md13836}

\hypertarget{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2label-has-associated-control_autotoc_md13841}{}\doxysection{\texorpdfstring{How to manage IDs of {\ttfamily input}}{How to manage IDs of {\ttfamily input}}}\label{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2label-has-associated-control_autotoc_md13841}
A common way to think of {\ttfamily id} with libraries like React is, {\ttfamily id}s should be avoided since it must be unique on the page, and components need to be reusable. Hence it is tempted to generate {\ttfamily id} during render-\/time if {\ttfamily id} is required. {\itshape However\+:}

IDs shouldn\textquotesingle{}t be generated in the browser, so that server and client rendering are deterministic. Render-\/time uuids aren\textquotesingle{}t just a hack, they\textquotesingle{}re actually broken and should never be used.

To restate, {\bfseries{every ID needs to be deterministic}}, on the server and the client, and guaranteed to be unique on the page. EG\+: For each input, a required ID prop can be passed down from as far up the tree as possible to guarantee uniqueness.\hypertarget{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2label-has-associated-control_autotoc_md13842}{}\doxysection{\texorpdfstring{Rule options}{Rule options}}\label{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2label-has-associated-control_autotoc_md13842}

{\ttfamily label\+Components} is a list of custom React Component names that should be checked for an associated control.

{\ttfamily label\+Attributes} is a list of attributes to check on the label component and its children for a label. Use this if you have a custom component that uses a string passed on a prop to render an HTML {\ttfamily label}, for example.

{\ttfamily control\+Components} is a list of custom React Components names that will output an input element. \href{https://linuxhint.com/bash_globbing_tutorial/}{\texttt{ Glob format}} is also supported for specifying names (e.\+g., {\ttfamily Label\texorpdfstring{$\ast$}{*}} matches {\ttfamily Label\+Component} but not {\ttfamily Custom\+Label}, {\ttfamily ????Label} matches {\ttfamily Link\+Label} but not {\ttfamily Custom\+Label}).

{\ttfamily assert} asserts that the label has html\+For, a nested label, both or either. Available options\+: `\textquotesingle{}html\+For', \textquotesingle{}nesting\textquotesingle{}, \textquotesingle{}both\textquotesingle{}, \textquotesingle{}either\textquotesingle{}\`{}.
