This module exports the identifiers \`{}push\`{} and \`{}splice\`{}. There is no default export.\hypertarget{readme.md_autotoc_md23620}{}\doxysubsubsection{\texorpdfstring{{\ttfamily push(list, items)}}{{\ttfamily push(list, items)}}}\label{readme.md_autotoc_md23620}
Append {\ttfamily items} (an array) at the end of {\ttfamily list} (another array). When {\ttfamily list} was empty, returns {\ttfamily items} instead.

This prevents a potentially expensive operation when {\ttfamily list} is empty, and adds items in batches to prevent V8 from hanging.\hypertarget{readme.md_autotoc_md23621}{}\doxysubsubsubsubsubsection{\texorpdfstring{Parameters}{Parameters}}\label{readme.md_autotoc_md23621}

Either {\ttfamily list} or {\ttfamily items} ({\ttfamily Array\texorpdfstring{$<$}{<}unknown\texorpdfstring{$>$}{>}}).\hypertarget{readme.md_autotoc_md23623}{}\doxysubsubsection{\texorpdfstring{{\ttfamily splice(list, start, remove, items)}}{{\ttfamily splice(list, start, remove, items)}}}\label{readme.md_autotoc_md23623}
Like {\ttfamily Array\#splice}, but smarter for giant arrays.

{\ttfamily Array\#splice} takes all items to be inserted as individual argument which causes a stack overflow in V8 when trying to insert 100k items for instance.

Otherwise, this does not return the removed items, and takes {\ttfamily items} as an array instead of rest parameters.\hypertarget{readme.md_autotoc_md23624}{}\doxysubsubsubsubsubsection{\texorpdfstring{Parameters}{Parameters}}\label{readme.md_autotoc_md23624}

\item {\ttfamily start} ({\ttfamily number}) — index to remove/insert at (can be negative)
\item {\ttfamily remove} ({\ttfamily number}) — number of items to remove
\item {\ttfamily items} ({\ttfamily Array\texorpdfstring{$<$}{<}unknown\texorpdfstring{$>$}{>}}) — items to inject into {\ttfamily list}
