\hypertarget{README.md_autotoc_md8297}{}\doxysubsection{\texorpdfstring{\+:handshake\+: Compatibility With Existing ESLint Rules}{\+:handshake\+: Compatibility With Existing ESLint Rules}}\label{README.md_autotoc_md8297}
Most of the rules in the ESLint core work for the script part, but some rules are incompatible. ~\newline
 This parser will generate a JSX compatible AST for most of the HTML part of the Astro component. Therefore, some rules of \href{https://github.com/jsx-eslint/eslint-plugin-react/}{\texttt{ eslint-\/plugin-\/react}} may work. For example, the \href{https://github.com/jsx-eslint/eslint-plugin-react/blob/master/docs/rules/jsx-no-target-blank.md}{\texttt{ react/jsx-\/no-\/target-\/blank}} rule works fine. ~\newline
\hypertarget{README.md_autotoc_md8298}{}\doxysubsection{\texorpdfstring{\+:ghost\+: Limitations}{\+:ghost\+: Limitations}}\label{README.md_autotoc_md8298}
If this parser is used with {\ttfamily @typescript-\/eslint/parser} and {\ttfamily parser\+Options.\+project} is set, it will temporarily create a {\ttfamily .tsx} file to parse the {\ttfamily .astro} file. ~\newline
 This parser works by converting the {\ttfamily .astro} file to JSX and letting the Java\+Script parser parse it. Since {\ttfamily @typescript-\/eslint/parser} can only parse files with the extension {\ttfamily .tsx} as JSX, it is necessary to temporarily create a {\ttfamily .tsx} file. Temporarily created files will try to be deleted after parses, but if the parsing takes a long time, the files may be visible to you.
