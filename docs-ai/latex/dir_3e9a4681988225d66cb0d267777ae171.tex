This package is a \href{https://github.com/unifiedjs/unified}{\texttt{ unified}} processor with support for parsing HTML as input and serializing HTML as output by using unified with \href{../rehype-parse/}{\texttt{ {\ttfamily rehype-\/parse}}} and \href{../rehype-stringify/}{\texttt{ {\ttfamily rehype-\/stringify}}}.

See \href{https://github.com/rehypejs/rehype}{\texttt{ the monorepo readme}} for info on what the rehype ecosystem is.\hypertarget{readme.md_autotoc_md28973}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{readme.md_autotoc_md28973}
You can use this package when you want to use unified, have HTML as input, and want HTML as output. This package is a shortcut for {\ttfamily unified().use(rehype\+Parse).use(rehype\+Stringify)}. When the input isn’t HTML (meaning you don’t need {\ttfamily rehype-\/parse}) or the output is not HTML (you don’t need {\ttfamily rehype-\/stringify}), it’s recommended to use {\ttfamily unified} directly.

When you’re in a browser, trust your content, don’t need positional info on nodes or formatting options, and value a smaller bundle size, you can use \href{https://github.com/rehypejs/rehype-dom/tree/main/packages/rehype-dom}{\texttt{ {\ttfamily rehype-\/dom}}} instead.

When you want to inspect and format HTML files in a project on the command line, you can use \href{../rehype-cli/}{\texttt{ {\ttfamily rehype-\/cli}}}.\hypertarget{readme.md_autotoc_md28974}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md28974}
