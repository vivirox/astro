Microsoft takes the security of our software products and services seriously, which includes all source code repositories managed through our Git\+Hub organizations, which include \href{https://github.com/microsoft}{\texttt{ Microsoft}}, \href{https://github.com/Azure}{\texttt{ Azure}}, \href{https://github.com/dotnet}{\texttt{ Dot\+Net}}, \href{https://github.com/aspnet}{\texttt{ Asp\+Net}}, \href{https://github.com/xamarin}{\texttt{ Xamarin}}, and \href{https://opensource.microsoft.com/}{\texttt{ our Git\+Hub organizations}}.

If you believe you have found a security vulnerability in any Microsoft-\/owned repository that meets \href{https://aka.ms/opensource/security/definition}{\texttt{ Microsoft\textquotesingle{}s definition of a security vulnerability}}, please report it to us as described below.\hypertarget{md_node__modules_2request-light_2_s_e_c_u_r_i_t_y_autotoc_md29328}{}\doxysubsection{\texorpdfstring{Reporting Security Issues}{Reporting Security Issues}}\label{md_node__modules_2request-light_2_s_e_c_u_r_i_t_y_autotoc_md29328}
{\bfseries{Please do not report security vulnerabilities through public Git\+Hub issues.}}

Instead, please report them to the Microsoft Security Response Center (MSRC) at \href{https://aka.ms/opensource/security/create-report}{\texttt{ https\+://msrc.\+microsoft.\+com/create-\/report}}.

If you prefer to submit without logging in, send email to \href{mailto:secure@microsoft.com}{\texttt{ secure@microsoft.\+com}}. If possible, encrypt your message with our PGP key; please download it from the \href{https://aka.ms/opensource/security/pgpkey}{\texttt{ Microsoft Security Response Center PGP Key page}}.

You should receive a response within 24 hours. If for some reason you do not, please follow up via email to ensure we received your original message. Additional information can be found at \href{https://aka.ms/opensource/security/msrc}{\texttt{ microsoft.\+com/msrc}}.

Please include the requested information listed below (as much as you can provide) to help us better understand the nature and scope of the possible issue\+:


\begin{DoxyItemize}
\item Type of issue (e.\+g. buffer overflow, SQL injection, cross-\/site scripting, etc.)
\item Full paths of source file(s) related to the manifestation of the issue
\item The location of the affected source code (tag/branch/commit or direct URL)
\item Any special configuration required to reproduce the issue
\item Step-\/by-\/step instructions to reproduce the issue
\item Proof-\/of-\/concept or exploit code (if possible)
\item Impact of the issue, including how an attacker might exploit the issue
\end{DoxyItemize}

This information will help us triage your report more quickly.

If you are reporting for a bug bounty, more complete reports can contribute to a higher bounty award.
