\hypertarget{md_node__modules_2r1csfile_2doc_2r1cs__bin__format_autotoc_md26779}{}\doxysection{\texorpdfstring{Motivation}{Motivation}}\label{md_node__modules_2r1csfile_2doc_2r1cs__bin__format_autotoc_md26779}
The zero knowledge primitives, requires the definition of a statment that wants to be proved. This statment can be expressed as a deterministric program or an algebraic circuit. Lots of primitives like zk\+Snarks, bullet\+Proofs or aurora, requires to convert this statment to a rank-\/one constraint system.

This standard specifies a format for a r1cs and allows the to connect a set of tools that compiles a program or a circuit to r1cs that can be used for the zksnarks or bulletproofs primitives.\hypertarget{md_node__modules_2r1csfile_2doc_2r1cs__bin__format_autotoc_md26780}{}\doxysection{\texorpdfstring{Specification}{Specification}}\label{md_node__modules_2r1csfile_2doc_2r1cs__bin__format_autotoc_md26780}

The standard extension is {\ttfamily .r1cs}

A deterministic program (or circuit) is a program that generates a set of deterministic values given an input. All those values are labeled from l\+\_\+\{0\} to l\+\_\+\{n\+\_\+labels\}

This file defines a map beween l\+\_\+\{i\} -\/\texorpdfstring{$>$}{>} w\+\_\+\{j\} and defines a series a R1\+CS of the form

\$\$ \textbackslash{}left\textbackslash{}\{ \textbackslash{}begin\{array\}\{rclclcl\} (a\+\_\+\{0,0\}w\+\_\+0 + a\+\_\+\{0,1\}w\+\_\+1 + ... + a\+\_\+\{0,n\}w\+\_\+\{n\}) \&\textbackslash{}cdot\& (b\+\_\+\{0,0\} w\+\_\+0 + b\+\_\+\{0,1\} w\+\_\+1 + ... + b\+\_\+\{0,n\} w\+\_\+\{n\}) \&-\/\& (c\+\_\+\{0,0\} w\+\_\+0 + c\+\_\+\{0,1\} w\+\_\+1 + ... + c\+\_\+\{0,n\}w\+\_\+\{n\}) \&=\& 0 \textbackslash{} (a\+\_\+\{1,0\}w\+\_\+0 + a\+\_\+\{1,1\}w\+\_\+1 + ... + a\+\_\+\{1,n\}w\+\_\+\{n\}) \&\textbackslash{}cdot\& (b\+\_\+\{1,0\} w\+\_\+0 + b\+\_\+\{1,1\} w\+\_\+1 + ... + b\+\_\+\{1,n\} w\+\_\+\{n\}) \&-\/\& (c\+\_\+\{1,0\} w\+\_\+0 + c\+\_\+\{1,1\}w\+\_\+1 + ... + c\+\_\+\{1,n\}w\+\_\+\{n\}) \&=\& 0 \textbackslash{} ...\textbackslash{} (a\+\_\+\{m-\/1,0\}w\+\_\+0 + a\+\_\+\{m-\/1,1\}w\+\_\+1 + ... + a\+\_\+\{m-\/1,n\}w\+\_\+\{n\}) \&\textbackslash{}cdot\& (b\+\_\+\{m-\/1,0\} w\+\_\+0 + b\+\_\+\{m-\/1,1\} w\+\_\+1 + ... + b\+\_\+\{m-\/1,n\} w\+\_\+\{n\}) \&-\/\& (c\+\_\+\{m-\/1,0\} w\+\_\+0 + c\+\_\+\{m-\/1,1\}w\+\_\+1 + ... + c\+\_\+\{m-\/1,n\}w\+\_\+\{n\}) \&=\& 0 \textbackslash{}end\{array\} \textbackslash{}right. \$\$

Size\+: 4 bytes Format\+: Little-\/\+Endian

Type of the section.

Currently there are 5 types of sections defined\+:


\begin{DoxyItemize}
\item 0x00000001 \+: Header Section
\item 0x00000002 \+: Constraint Section
\item 0x00000003 \+: Wire2\+Label\+Id Map Section
\item 0x00000004 \+: Custom gates List Section \mbox{[}Ultra\+Plonk\mbox{]}
\item 0x00000005 \+: Custom gates Application Section \mbox{[}Ultra\+Plonk\mbox{]}
\end{DoxyItemize}

If the file contain other types, the format is valid, but they MUST be ignored.

Any order of the section must be accepted.

Example\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{0x01\ 0x00\ 0x00\ 0x00}
