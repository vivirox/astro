The light version of \href{https://github.com/MikeMcl/decimal.js/}{\texttt{ decimal.\+js}}, an arbitrary-\/precision Decimal type for Java\+Script.

\href{https://travis-ci.org/MikeMcl/decimal.js-light}{\texttt{ }}

~\newline


This library is the newest of the family of libraries\+: \href{https://github.com/MikeMcl/bignumber.js/}{\texttt{ bignumber.\+js}}, \href{https://github.com/MikeMcl/big.js/}{\texttt{ big.\+js}}, \href{https://github.com/MikeMcl/decimal.js/}{\texttt{ decimal.\+js}} and {\itshape decimal.\+js-\/light}.~\newline
 The API is more or less a subset of the API of {\itshape decimal.\+js}.



{\bfseries{Differences between this library and {\itshape decimal.\+js}}}

Size of {\itshape decimal.\+js} minified\+: 32.\+1 KB.~\newline
 Size of {\itshape decimal.\+js-\/light} minified\+: 12.\+7 KB.

This library does not include {\ttfamily NaN}, {\ttfamily Infinity} or {\ttfamily -\/0} as legitimate values, or work with values in other bases.

Here, the {\ttfamily Decimal.\+round} property is just the default rounding mode for {\ttfamily to\+Decimal\+Places}, {\ttfamily to\+Exponential}, {\ttfamily to\+Fixed}, {\ttfamily to\+Precision} and {\ttfamily to\+Significant\+Digits}. It does not apply to arithmetic operations, which are simply truncated at the required precision.

If rounding is required just apply it explicitly, for example

The {\ttfamily natural\+Exponential}, {\ttfamily natural\+Logarithm}, {\ttfamily logarithm}, and {\ttfamily to\+Power} methods in this library have by default a limited precision of around 100 digits. This limit can be increased at runtime using the {\ttfamily LN10} (the natural logarithm of ten) configuration object property.
