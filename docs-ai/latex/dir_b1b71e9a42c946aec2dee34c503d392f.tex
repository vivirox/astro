\hypertarget{README.md_autotoc_md28036}{}\doxysubsubsubsection{\texorpdfstring{Error classes}{Error classes}}\label{README.md_autotoc_md28036}
All errors returned by Node\+Redis use own Error classes. You can distinguish different errors easily by checking for these classes.

To know what caused the error they might contain properties to know in more detail what happened.

Each error contains a {\ttfamily message}, a {\ttfamily name} and a {\ttfamily stack} property. Please be aware that the stack might not be useful due to the async nature and is in those cases therefore limited to two frames.

There might be more not yet documented properties as well. Please feel free to open a pull request to document those as well.\hypertarget{README.md_autotoc_md28037}{}\doxysubsubsubsubsection{\texorpdfstring{Redis\+Error}{Redis\+Error}}\label{README.md_autotoc_md28037}
