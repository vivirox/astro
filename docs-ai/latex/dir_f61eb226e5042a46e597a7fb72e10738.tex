SWR is a React Hooks library for data fetching.

The name “\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}\+SWR\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}” is derived from {\ttfamily stale-\/while-\/revalidate}, a cache invalidation strategy popularized by \href{https://tools.ietf.org/html/rfc5861}{\texttt{ HTTP RFC 5861}}. {\bfseries{SWR}} first returns the data from cache (stale), then sends the request (revalidate), and finally comes with the up-\/to-\/date data again.

With just one hook, you can significantly simplify the data fetching logic in your project. And it also covered in all aspects of speed, correctness, and stability to help you build better experiences\+:


\begin{DoxyItemize}
\item {\bfseries{Fast}}, {\bfseries{lightweight}} and {\bfseries{reusable}} data fetching
\item Transport and protocol agnostic
\item Built-\/in {\bfseries{cache}} and request deduplication
\item {\bfseries{Real-\/time}} experience
\item Revalidation on focus
\item Revalidation on network recovery
\item Polling
\item Pagination and scroll position recovery
\item SSR and SSG
\item Local mutation (Optimistic UI)
\item Built-\/in smart error retry
\item Type\+Script
\item React Suspense
\item React Native
\end{DoxyItemize}

...and a lot more.

With SWR, components will get {\bfseries{a stream of data updates constantly and automatically}}. Thus, the UI will be always {\bfseries{fast}} and {\bfseries{reactive}}.

In this example, the React Hook {\ttfamily use\+SWR} accepts a {\ttfamily key} and a {\ttfamily fetcher} function. The {\ttfamily key} is a unique identifier of the request, normally the URL of the API. And the {\ttfamily fetcher} accepts {\ttfamily key} as its parameter and returns the data asynchronously.

{\ttfamily use\+SWR} also returns 2 values\+: {\ttfamily data} and {\ttfamily error}. When the request (fetcher) is not yet finished, {\ttfamily data} will be {\ttfamily undefined}. And when we get a response, it sets {\ttfamily data} and {\ttfamily error} based on the result of {\ttfamily fetcher} and rerenders the component.

Note that {\ttfamily fetcher} can be any asynchronous function, you can use your favourite data-\/fetching library to handle that part.
