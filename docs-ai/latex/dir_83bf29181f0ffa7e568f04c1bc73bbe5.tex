This package is a \href{https://github.com/unifiedjs/unified}{\texttt{ unified}} (\href{https://github.com/remarkjs/remark}{\texttt{ remark}}) plugin to add support for directives\+: one syntax for arbitrary extensions in markdown.\hypertarget{readme.md_autotoc_md29102}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{readme.md_autotoc_md29102}
Directives are one of the four ways to extend markdown\+: an arbitrary extension syntax (see \href{https://github.com/micromark/micromark\#extending-markdown}{\texttt{ Extending markdown}} in micromarkâ€™s docs for the alternatives and more info). This mechanism works well when you control the content\+: who authors it, what tools handle it, and where itâ€™s displayed. When authors can read a guide on how to embed a tweet but are not expected to know the ins and outs of HTML or Java\+Script. Directives donâ€™t work well if you donâ€™t know who authors content, what tools handle it, and where it ends up. Example use cases are a docs website for a project or product, or blogging tools and static site generators.
