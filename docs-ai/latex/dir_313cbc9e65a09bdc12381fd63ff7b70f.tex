\hypertarget{README.md_autotoc_md3724}{}\doxysubsubsection{\texorpdfstring{Basic concept}{Basic concept}}\label{README.md_autotoc_md3724}
Visual terminal apps are at their core strings rendered onto the terminal.

The most basic prompt is a function returning a string that\textquotesingle{}ll be rendered in the terminal. This function will run every time the prompt state change, and the new returned string will replace the previously rendered one. The prompt cursor appears after the string.

Wrapping the rendering function with {\ttfamily create\+Prompt()} will setup the rendering layer, inject the state management utilities, and wait until the {\ttfamily done} callback is called.

\hypertarget{README.md_autotoc_md3728}{}\doxysubsubsubsection{\texorpdfstring{Ref hook}{Ref hook}}\label{README.md_autotoc_md3728}
Refs let a prompt hold some information that isn’t used for rendering, like a class instance or a timeout ID. Unlike with state, updating a ref does not re-\/render your prompt. Refs are an “escape hatch” from the rendering paradigm.

{\ttfamily use\+Ref} declares a ref. You can hold any value in it, but most often it’s used to hold a timeout ID.

\hypertarget{README.md_autotoc_md3733}{}\doxysubsubsubsubsection{\texorpdfstring{Pagination}{Pagination}}\label{README.md_autotoc_md3733}
When looping through a long list of options (like in the {\ttfamily select} prompt), paginating the results appearing on the screen at once can be necessary. The {\ttfamily use\+Pagination} hook is the utility used within the {\ttfamily select} and {\ttfamily checkbox} prompts to cycle through the list of options.

Pagination works by taking in the list of options and returning a subset of the rendered items that fit within the page. The hook takes in a few options. It needs a list of options ({\ttfamily items}), and a {\ttfamily page\+Size} which is the number of lines to be rendered. The {\ttfamily active} index is the index of the currently selected/selectable item. The {\ttfamily loop} option is a boolean that indicates if the list should loop around when reaching the end\+: this is the default behavior. The pagination hook renders items only as necessary, so it takes a function that can render an item at an index, including an {\ttfamily active} state, called {\ttfamily render\+Item}.
