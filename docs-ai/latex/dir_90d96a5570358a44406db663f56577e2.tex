\hypertarget{readme.md_autotoc_md21594}{}\doxysubsection{\texorpdfstring{ESM}{ESM}}\label{readme.md_autotoc_md21594}
ESM configs can be loaded with {\bfseries{async API only}}. Specifically {\ttfamily js} files in projects with {\ttfamily "{}type"{}\+: "{}module"{}} in {\ttfamily package.\+json} or {\ttfamily mjs} files.\hypertarget{readme.md_autotoc_md21595}{}\doxysubsection{\texorpdfstring{Difference to {\ttfamily cosmiconfig}}{Difference to {\ttfamily cosmiconfig}}}\label{readme.md_autotoc_md21595}
Lilconfig does not intend to be 100\% compatible with {\ttfamily cosmiconfig} but tries to mimic it where possible. The key difference is {\bfseries{no}} support for yaml files out of the box({\ttfamily lilconfig} attempts to parse files with no extension as JSON instead of YAML). You can still add the support for YAML files by providing a loader, see an example below.\hypertarget{readme.md_autotoc_md21596}{}\doxysubsubsection{\texorpdfstring{Options difference between the two.}{Options difference between the two.}}\label{readme.md_autotoc_md21596}
