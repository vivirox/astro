\hypertarget{README.md_autotoc_md10103}{}\doxysubsubsection{\texorpdfstring{Usage}{Usage}}\label{README.md_autotoc_md10103}
The primary export of {\ttfamily check-\/error} is an object which has the following methods\+:


\begin{DoxyItemize}
\item {\ttfamily compatible\+Instance(err, error\+Like)} -\/ Checks if an error is compatible with another {\ttfamily error\+Like} object. If {\ttfamily error\+Like} is an error instance we do a strict comparison, otherwise we return {\ttfamily false} by default, because instances of objects can only be compatible if they\textquotesingle{}re both error instances.
\item {\ttfamily compatible\+Constructor(err, error\+Like)} -\/ Checks if an error\textquotesingle{}s constructor is compatible with another {\ttfamily error\+Like} object. If {\ttfamily err} has the same constructor as {\ttfamily error\+Like} or if {\ttfamily err} is an instance of {\ttfamily error\+Like}.
\item {\ttfamily compatible\+Message(err, err\+Matcher)} -\/ Checks if an error message is compatible with an {\ttfamily err\+Matcher} Reg\+Exp or String (we check if the message contains the String).
\item {\ttfamily get\+Constructor\+Name(error\+Like)} -\/ Retrieves the name of a constructor, an error\textquotesingle{}s constructor or {\ttfamily error\+Like} itself if it\textquotesingle{}s not an error instance or constructor.
\item {\ttfamily get\+Message(err)} -\/ Retrieves the message of an error or {\ttfamily err} itself if it\textquotesingle{}s a String. If {\ttfamily err} or {\ttfamily err.\+message} is undefined we return an empty String.
\end{DoxyItemize}
