\begin{DoxyCode}{0}
\DoxyCodeLine{toBeDisabled()}

\end{DoxyCode}


This allows you to check whether an element is disabled from the user\textquotesingle{}s perspective. According to the specification, the following elements can be \href{https://html.spec.whatwg.org/multipage/semantics-other.html\#disabled-elements}{\texttt{ disabled}}\+: {\ttfamily button}, {\ttfamily input}, {\ttfamily select}, {\ttfamily textarea}, {\ttfamily optgroup}, {\ttfamily option}, {\ttfamily fieldset}, and custom elements.

This custom matcher considers an element as disabled if the element is among the types of elements that can be disabled (listed above), and the {\ttfamily disabled} attribute is present. It will also consider the element as disabled if it\textquotesingle{}s inside a parent form element that supports being disabled and has the {\ttfamily disabled} attribute present.\hypertarget{README.md_autotoc_md6336}{}\doxysubsubsubsubsection{\texorpdfstring{Examples}{Examples}}\label{README.md_autotoc_md6336}

\DoxyCodeLine{toBeVisible()}

\end{DoxyCode}


This allows you to check if an element is currently visible to the user.

An element is visible if {\bfseries{all}} the following conditions are met\+:


\begin{DoxyItemize}
\item it is present in the document
\item it does not have its css property {\ttfamily display} set to {\ttfamily none}
\item it does not have its css property {\ttfamily visibility} set to either {\ttfamily hidden} or {\ttfamily collapse}
\item it does not have its css property {\ttfamily opacity} set to {\ttfamily 0}
\item its parent element is also visible (and so on up to the top of the DOM tree)

This allows you to check if a form or fieldset contains form controls for each given name, and having the specified value.

\begin{quote}
It is important to stress that this matcher can only be invoked on a \href{https://developer.mozilla.org/en-US/docs/Web/API/HTMLFormElement}{\texttt{ form}} or a \href{https://developer.mozilla.org/en-US/docs/Web/API/HTMLFieldSetElement}{\texttt{ fieldset}} element.

This allows it to take advantage of the \href{https://developer.mozilla.org/en-US/docs/Web/API/HTMLFormElement/elements}{\texttt{ .elements}} property in {\ttfamily form} and {\ttfamily fieldset} to reliably fetch all form controls within them.

This also avoids the possibility that users provide a container that contains more than one {\ttfamily form}, thereby intermixing form controls that are not related, and could even conflict with one another. \end{quote}


This matcher abstracts away the particularities with which a form control value is obtained depending on the type of form control. For instance, {\ttfamily \texorpdfstring{$<$}{<}input\texorpdfstring{$>$}{>}} elements have a {\ttfamily value} attribute, but {\ttfamily \texorpdfstring{$<$}{<}select\texorpdfstring{$>$}{>}} elements do not. Here\textquotesingle{}s a list of all cases covered\+:


\begin{DoxyItemize}
\item {\ttfamily \texorpdfstring{$<$}{<}input type="{}number"{}\texorpdfstring{$>$}{>}} elements return the value as a {\bfseries{number}}, instead of a string.
\item {\ttfamily \texorpdfstring{$<$}{<}input type="{}checkbox"{}\texorpdfstring{$>$}{>}} elements\+:
\begin{DoxyItemize}
\item if there\textquotesingle{}s a single one with the given {\ttfamily name} attribute, it is treated as a {\bfseries{boolean}}, returning {\ttfamily true} if the checkbox is checked, {\ttfamily false} if unchecked.
\item if there\textquotesingle{}s more than one checkbox with the same {\ttfamily name} attribute, they are all treated collectively as a single form control, which returns the value as an {\bfseries{array}} containing all the values of the selected checkboxes in the collection.
\end{DoxyItemize}
\item {\ttfamily \texorpdfstring{$<$}{<}input type="{}radio"{}\texorpdfstring{$>$}{>}} elements are all grouped by the {\ttfamily name} attribute, and such a group treated as a single form control. This form control returns the value as a {\bfseries{string}} corresponding to the {\ttfamily value} attribute of the selected radio button within the group.
\item {\ttfamily \texorpdfstring{$<$}{<}input type="{}text"{}\texorpdfstring{$>$}{>}} elements return the value as a {\bfseries{string}}. This also applies to {\ttfamily \texorpdfstring{$<$}{<}input\texorpdfstring{$>$}{>}} elements having any other possible {\ttfamily type} attribute that\textquotesingle{}s not explicitly covered in different rules above (e.\+g. {\ttfamily search}, {\ttfamily email}, {\ttfamily date}, {\ttfamily password}, {\ttfamily hidden}, etc.)
\item {\ttfamily \texorpdfstring{$<$}{<}select\texorpdfstring{$>$}{>}} elements without the {\ttfamily multiple} attribute return the value as a {\bfseries{string}} corresponding to the {\ttfamily value} attribute of the selected {\ttfamily option}, or {\ttfamily undefined} if there\textquotesingle{}s no selected option.
\item {\ttfamily \texorpdfstring{$<$}{<}select multiple\texorpdfstring{$>$}{>}} elements return the value as an {\bfseries{array}} containing all the values of the \href{https://developer.mozilla.org/en-US/docs/Web/API/HTMLSelectElement/selectedOptions}{\texttt{ selected options}}.
\item {\ttfamily \texorpdfstring{$<$}{<}textarea\texorpdfstring{$>$}{>}} elements return their value as a {\bfseries{string}}. The value corresponds to their node content.
\end{DoxyItemize}

The above rules make it easy, for instance, to switch from using a single select control to using a group of radio buttons. Or to switch from a multi select control, to using a group of checkboxes. \end{DoxyCode}


This allows you to check whether the given element has a description or not.

An element gets its description via the \href{https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/ARIA_Techniques/Using_the_aria-describedby_attribute}{\texttt{ {\ttfamily aria-\/describedby} attribute}}. Set this to the {\ttfamily id} of one or more other elements. These elements may be nested inside, be outside, or a sibling of the passed in element.

Whitespace is normalized. Using multiple ids will \href{https://www.w3.org/TR/accname-1.1/\#mapping_additional_nd_description}{\texttt{ join the referenced elements’ text content separated by a space}}.

When a {\ttfamily string} argument is passed through, it will perform a whole case-\/sensitive match to the description text.

To perform a case-\/insensitive match, you can use a {\ttfamily Reg\+Exp} with the {\ttfamily /i} modifier.

To perform a partial match, you can pass a {\ttfamily Reg\+Exp} or use {\ttfamily expect.\+string\+Containing("{}partial string"{})}.\hypertarget{README.md_autotoc_md6390}{}\doxysubsubsubsubsection{\texorpdfstring{Examples}{Examples}}\label{README.md_autotoc_md6390}
