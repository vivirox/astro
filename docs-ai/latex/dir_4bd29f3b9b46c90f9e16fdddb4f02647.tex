This package exports the identifiers \`{}\+CONTINUE\`{}, \`{}\+EXIT\`{}, \`{}\+SKIP\`{}, and \`{}visit\+Parents\`{}. There is no default export.\hypertarget{readme.md_autotoc_md34406}{}\doxysubsubsection{\texorpdfstring{{\ttfamily visit\+Parents(tree\mbox{[}, test\mbox{]}, visitor\mbox{[}, reverse\mbox{]})}}{{\ttfamily visit\+Parents(tree\mbox{[}, test\mbox{]}, visitor\mbox{[}, reverse\mbox{]})}}}\label{readme.md_autotoc_md34406}
Visit nodes, with ancestral information.

This algorithm performs {\itshape \href{https://github.com/syntax-tree/unist\#depth-first-traversal}{\texttt{ depth-\/first}}} {\itshape \href{https://github.com/syntax-tree/unist\#tree-traversal}{\texttt{ tree traversal}}} in {\itshape \href{https://github.com/syntax-tree/unist\#preorder}{\texttt{ preorder}}} ({\bfseries{NLR}}) or if {\ttfamily reverse} is given, in {\itshape reverse preorder} ({\bfseries{NRL}}).

You can choose for which nodes {\ttfamily visitor} is called by passing a {\ttfamily test}. For complex tests, you should test yourself in {\ttfamily visitor}, as it will be faster and will have improved type information.

Walking the tree is an intensive task. Make use of the return values of the visitor when possible. Instead of walking a tree multiple times, walk it once, use \href{https://github.com/syntax-tree/unist-util-is}{\texttt{ {\ttfamily unist-\/util-\/is}}} to check if a node matches, and then perform different operations.

You can change the tree. See \`{}\+Visitor\`{} for more info.\hypertarget{readme.md_autotoc_md34407}{}\doxysubsubsubsubsubsection{\texorpdfstring{Parameters}{Parameters}}\label{readme.md_autotoc_md34407}

Handle a node (matching {\ttfamily test}, if given) (Type\+Script type).

Visitors are free to transform {\ttfamily node}. They can also transform the parent of node (the last of {\ttfamily ancestors}).

Replacing {\ttfamily node} itself, if {\ttfamily SKIP} is not returned, still causes its descendants to be walked (which is a bug).

When adding or removing previous siblings of {\ttfamily node} (or next siblings, in case of reverse), the {\ttfamily Visitor} should return a new {\ttfamily Index} to specify the sibling to traverse after {\ttfamily node} is traversed. Adding or removing next siblings of {\ttfamily node} (or previous siblings, in case of reverse) is handled as expected without needing to return a new {\ttfamily Index}.

Removing the children property of an ancestor still results in them being traversed.\hypertarget{readme.md_autotoc_md34423}{}\doxysubsubsubsubsubsection{\texorpdfstring{Parameters}{Parameters}}\label{readme.md_autotoc_md34423}
