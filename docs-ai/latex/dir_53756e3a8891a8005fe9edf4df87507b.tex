\doxysubsection{Detailed Description}
Lightweight (\texorpdfstring{$<$}{<} 1.\+6kB gzipped) and performant natural sorting of arrays and collections by differentiating between unicode characters, numbers, dates, etc.

\href{https://www.npmjs.com/package/natural-orderby}{\texttt{ }} \href{https://www.npmjs.com/package/natural-orderby}{\texttt{ }} \href{https://github.com/yobacca/natural-orderby/actions}{\texttt{ }} \href{https://codecov.io/gh/yobacca/natural-orderby}{\texttt{ }}

People sort strings containing numbers differently than most sorting algorithms, which sort values by comparing strings in Unicode code point order. This produces an ordering that is inconsistent with human logic.

{\ttfamily natural-\/orderby} sorts the primitive values of \href{https://developer.mozilla.org/en-US/docs/Glossary/Boolean}{\texttt{ {\ttfamily Boolean}}}, \href{https://developer.mozilla.org/en-US/docs/Glossary/Null}{\texttt{ {\ttfamily Null}}}, \href{https://developer.mozilla.org/en-US/docs/Glossary/Undefined}{\texttt{ {\ttfamily Undefined}}}, \href{https://developer.mozilla.org/en-US/docs/Glossary/Number}{\texttt{ {\ttfamily Number}}} or \href{https://developer.mozilla.org/en-US/docs/Glossary/String}{\texttt{ {\ttfamily String}}} type as well as \href{https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Date}{\texttt{ {\ttfamily Date}}} objects. When comparing strings it differentiates between unicode characters, integer, floating as well as hexadecimal numbers, various date formats, etc. You may sort flat or nested arrays or arrays of objects in a natural sorting order using {\ttfamily natural-\/orderby}.

In addition to the efficient and fast {\ttfamily order\+By()} method {\ttfamily natural-\/orderby} also provides the method {\ttfamily compare()}, which may be passed to \href{https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array/sort}{\texttt{ {\ttfamily Array.\+prototype.\+sort()}}}.\hypertarget{README.md_autotoc_md24470}{}\doxysubsection{\texorpdfstring{Contents}{Contents}}\label{README.md_autotoc_md24470}

Creates an array of elements, natural sorted by specified {\ttfamily identifiers} and the corresponding sort {\ttfamily orders}. This method implements a stable sort algorithm, which means the original sort order of equal elements is preserved. It also avoids the high overhead caused by \href{https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array/sort}{\texttt{ {\ttfamily Array.\+prototype.\+sort()}}} invoking a compare function multiple times per element within the array.\hypertarget{README.md_autotoc_md24476}{}\doxysubsubsubsection{\texorpdfstring{Syntax}{Syntax}}\label{README.md_autotoc_md24476}

\hypertarget{README.md_autotoc_md24477}{}\doxysubsubsubsection{\texorpdfstring{Description}{Description}}\label{README.md_autotoc_md24477}
{\ttfamily order\+By()} sorts the elements of an array by specified identifiers and the corresponding sort orders in a natural order and returns a new array containing the sorted elements.

If {\ttfamily collection} is an array of primitives, {\ttfamily identifiers} may be unspecified. Otherwise, you should specify {\ttfamily identifiers} to sort by or {\ttfamily collection} will be returned unsorted. An identifier can be expressed by\+:


\begin{DoxyItemize}
\item an index position, if {\ttfamily collection} is a nested array,
\item a property name, if {\ttfamily collection} is an array of objects,
\item a function which returns a particular value from an element of a nested array or an array of objects. This function will be invoked by passing one element of {\ttfamily collection}.
\end{DoxyItemize}

If {\ttfamily orders} is unspecified, all values are sorted in ascending order. Otherwise, specify an order of `\textquotesingle{}desc'{\ttfamily for descending or}\textquotesingle{}asc\textquotesingle{}{\ttfamily for ascending sort order of corresponding values. You may also specify a compare function for an order, which will be invoked by two arguments\+:}(valueA, valueB)\`{}. It must return a number representing the sort order.

If you want to sort unicode strings according to a specific locale, please provide a string value for {\ttfamily locale} with a BCP 47 language tag. If the argument is unspecified, the host locale setting will be used.

\begin{quote}
Note\+: {\ttfamily order\+By()} always returns a new array, even if the original was already sorted. \end{quote}
