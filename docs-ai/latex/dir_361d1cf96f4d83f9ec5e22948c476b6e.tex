This package contains two extensions that add support for GFM autolink literals syntax in markdown to \href{https://github.com/syntax-tree/mdast}{\texttt{ mdast}}. These extensions plug into \href{https://github.com/syntax-tree/mdast-util-from-markdown}{\texttt{ {\ttfamily mdast-\/util-\/from-\/markdown}}} (to support parsing GFM autolinks in markdown into a syntax tree) and \href{https://github.com/syntax-tree/mdast-util-to-markdown}{\texttt{ {\ttfamily mdast-\/util-\/to-\/markdown}}} (to support serializing GFM autolinks in syntax trees to markdown).

Git\+Hub employs different algorithms to autolink\+: one at parse time and one at transform time (similar to how {\ttfamily @mentions} are done at transform time). This difference can be observed because character references and escapes are handled differently. But also because issues/\+PRs/comments omit (perhaps by accident?) the second algorithm for {\ttfamily www.}, {\ttfamily \href{http://}{\texttt{ http\+://}}}, and {\ttfamily \href{https://}{\texttt{ https\+://}}} links (but not for email links).

As the corresponding micromark extension \href{https://github.com/micromark/micromark-extension-gfm-autolink-literal}{\texttt{ {\ttfamily micromark-\/extension-\/gfm-\/autolink-\/literal}}} is a syntax extension, it can only perform the first algorithm. The tree extension {\ttfamily gfm\+Autolink\+Literal\+From\+Markdown} from this package can perform the second algorithm, and as they are combined, both are done.\hypertarget{readme.md_autotoc_md22381}{}\doxysubsection{\texorpdfstring{When to use this}{When to use this}}\label{readme.md_autotoc_md22381}
You can use these extensions when you are working with {\ttfamily mdast-\/util-\/from-\/markdown} and {\ttfamily mdast-\/util-\/to-\/markdown} already.

When working with {\ttfamily mdast-\/util-\/from-\/markdown}, you must combine this package with \href{https://github.com/micromark/micromark-extension-gfm-autolink-literal}{\texttt{ {\ttfamily micromark-\/extension-\/gfm-\/autolink-\/literal}}}.

When you don’t need a syntax tree, you can use \href{https://github.com/micromark/micromark}{\texttt{ {\ttfamily micromark}}} directly with {\ttfamily micromark-\/extension-\/gfm-\/autolink-\/literal}.

When you are working with syntax trees and want all of GFM, use \href{https://github.com/syntax-tree/mdast-util-gfm}{\texttt{ {\ttfamily mdast-\/util-\/gfm}}} instead.

All these packages are used \href{https://github.com/remarkjs/remark-gfm}{\texttt{ {\ttfamily remark-\/gfm}}}, which focusses on making it easier to transform content by abstracting these internals away.

This utility does not handle how markdown is turned to HTML. That’s done by \href{https://github.com/syntax-tree/mdast-util-to-hast}{\texttt{ {\ttfamily mdast-\/util-\/to-\/hast}}}.\hypertarget{readme.md_autotoc_md22382}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md22382}
