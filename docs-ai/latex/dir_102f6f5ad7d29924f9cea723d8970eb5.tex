This package contains two extensions that add support for MDX JSX syntax in markdown to \href{https://github.com/syntax-tree/mdast}{\texttt{ mdast}}. These extensions plug into \href{https://github.com/syntax-tree/mdast-util-from-markdown}{\texttt{ {\ttfamily mdast-\/util-\/from-\/markdown}}} (to support parsing JSX in markdown into a syntax tree) and \href{https://github.com/syntax-tree/mdast-util-to-markdown}{\texttt{ {\ttfamily mdast-\/util-\/to-\/markdown}}} (to support serializing JSX in syntax trees to markdown).

\href{https://facebook.github.io/jsx/}{\texttt{ JSX}} is an XML-\/like syntax extension to ECMAScript (Java\+Script), which MDX brings to markdown. For more info on MDX, see \href{https://mdxjs.com/docs/what-is-mdx/}{\texttt{ What is MDX?}}\hypertarget{readme.md_autotoc_md22595}{}\doxysubsection{\texorpdfstring{When to use this}{When to use this}}\label{readme.md_autotoc_md22595}
You can use these extensions when you are working with {\ttfamily mdast-\/util-\/from-\/markdown} and {\ttfamily mdast-\/util-\/to-\/markdown} already.

When working with {\ttfamily mdast-\/util-\/from-\/markdown}, you must combine this package with \href{https://github.com/micromark/micromark-extension-mdx-jsx}{\texttt{ {\ttfamily micromark-\/extension-\/mdx-\/jsx}}}.

When you are working with syntax trees and want all of MDX, use \href{https://github.com/syntax-tree/mdast-util-mdx}{\texttt{ {\ttfamily mdast-\/util-\/mdx}}} instead.

All these packages are used in \href{https://mdxjs.com/packages/remark-mdx/}{\texttt{ {\ttfamily remark-\/mdx}}}, which focusses on making it easier to transform content by abstracting these internals away.\hypertarget{readme.md_autotoc_md22596}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md22596}

{\bfseries{Mdx\+Jsx\+Element}} represents a JSX element.

The {\ttfamily name} field can be present and represents an identifier. Without {\ttfamily name}, the element represents a fragment, in which case no attributes must be present.

The {\ttfamily attributes} field represents information associated with the node. The value of the {\ttfamily attributes} field is a list of {\bfseries{Mdx\+Jsx\+Expression\+Attribute}} and {\bfseries{Mdx\+Jsx\+Attribute}} nodes.

{\bfseries{Mdx\+Jsx\+Expression\+Attribute}} represents an expression (typically in a programming language) that when evaluated results in multiple attributes.

{\bfseries{Mdx\+Jsx\+Attribute}} represents a single attribute. The {\ttfamily name} field must be present. The {\ttfamily value} field can be present, in which case it is either a string (a static value) or an expression (typically in a programming language) that when evaluated results in an attribute value.\hypertarget{readme.md_autotoc_md22628}{}\doxysubsubsection{\texorpdfstring{Content model}{Content model}}\label{readme.md_autotoc_md22628}
