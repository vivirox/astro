This package is a \href{https://github.com/unifiedjs/unified}{\texttt{ unified}} (\href{https://github.com/mdx-js/recma}{\texttt{ recma}}) plugin that defines how to take a syntax tree as input and turn it into Java\+Script. When it’s used, Java\+Script is serialized as the final result.

See \href{https://github.com/mdx-js/recma}{\texttt{ the monorepo readme}} for info on what the recma ecosystem is.\hypertarget{readme.md_autotoc_md27990}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{readme.md_autotoc_md27990}
This plugin adds support to unified for serializing Java\+Script. If you also need to parse Java\+Script, you can alternatively use \href{https://github.com/mdx-js/recma/tree/main/packages/recma}{\texttt{ {\ttfamily recma}}}, which combines unified, this plugin, and \href{https://github.com/mdx-js/recma/tree/main/packages/recma-parse}{\texttt{ {\ttfamily recma-\/parse}}}.

If you don’t use plugins and have access to the syntax tree, you can directly use \href{https://github.com/syntax-tree/estree-util-to-js}{\texttt{ {\ttfamily estree-\/util-\/to-\/js}}}, which is used inside this plugin. recma focusses on making it easier to transform code by abstracting such internals away.\hypertarget{readme.md_autotoc_md27991}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md27991}
