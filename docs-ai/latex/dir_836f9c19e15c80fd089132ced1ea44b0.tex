\doxysubsection{Detailed Description}
Finds matching opening and closing tag pair for given location in HTML/\+XML source\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ match\ from\ '@emmetio/html-\/matcher';}
\DoxyCodeLine{}
\DoxyCodeLine{const\ content\ =\ '<div><a\ href="{}http://emmet.io"{}>Example</a></div>';}
\DoxyCodeLine{}
\DoxyCodeLine{//\ Find\ tag\ pair\ at\ character\ 35}
\DoxyCodeLine{const\ tag\ =\ match(content,\ 35);}
\DoxyCodeLine{}
\DoxyCodeLine{console.log(tag.name);\ //\ Name\ of\ matched\ tag:\ "{}a"{}}
\DoxyCodeLine{console.log(tag.open);\ //\ Range\ of\ opening\ tag:\ [5,\ 31]}
\DoxyCodeLine{console.log(tag.end);\ //\ Range\ of\ closing\ tag:\ [38,\ 42]}
\DoxyCodeLine{}
\DoxyCodeLine{//\ List\ of\ attributes\ found\ in\ opening\ tag}
\DoxyCodeLine{console.log(tag.attributes);}

\end{DoxyCode}


By default, matcher works in HTML, which means if it finds tag name which is known to be empty (for example, {\ttfamily \texorpdfstring{$<$}{<}img\texorpdfstring{$>$}{>}}) it will not search for it’s closing part. However, such behavior might be unexpected for XML syntaxes where all tags should be either self-\/closed or provide closing part. In this case, you should pass {\ttfamily xml\+: true} option to properly handle XML mode\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ match\ from\ '@emmetio/html-\/matcher';}
\DoxyCodeLine{}
\DoxyCodeLine{const\ content\ =\ '<div><img>Caption</img></div>';}
\DoxyCodeLine{const\ html\ =\ match(content,\ 8);}
\DoxyCodeLine{const\ xml\ =\ match(content,\ 8,\ \{\ xml:\ true\ \});}
\DoxyCodeLine{}
\DoxyCodeLine{console.log(html.name);\ //\ "{}img"{}}
\DoxyCodeLine{console.log(html.open);\ //\ [5,\ 10]}
\DoxyCodeLine{console.log(html.close);\ //\ undefined}
\DoxyCodeLine{}
\DoxyCodeLine{console.log(xml.name);\ //\ "{}img"{}}
\DoxyCodeLine{console.log(xml.open);\ //\ [5,\ 10]}
\DoxyCodeLine{console.log(xml.close);\ //\ [17,\ 23]}

\end{DoxyCode}
\hypertarget{README.md_autotoc_md3006}{}\doxysubsection{\texorpdfstring{Special tags}{Special tags}}\label{README.md_autotoc_md3006}
In HTML, some tags has special meaning. For example, a {\ttfamily \texorpdfstring{$<$}{<}script\texorpdfstring{$>$}{>}} tag\+: its contents should be completely ignored until we find closing {\ttfamily \texorpdfstring{$<$}{<}/script\texorpdfstring{$>$}{>}} tag. But, if {\ttfamily \texorpdfstring{$<$}{<}script\texorpdfstring{$>$}{>}} tag contains unknown {\ttfamily type} attribute value, we should consider it as a regular tag. By default, matcher understands {\ttfamily script} and {\ttfamily style} tags as “special” but you can override them with {\ttfamily special} option\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ match\ from\ '@emmetio/html-\/matcher';}
\DoxyCodeLine{}
\DoxyCodeLine{//\ Treat\ \`{}<foo-\/bar>`\ tag\ as\ ”special”:\ skip\ its\ content\ until\ \`{}</foo-\/bar>`.\ Note\ that\ this\ option\ overwrites\ default\ value\ with\ \`{}['script',\ 'style']`\ value}
\DoxyCodeLine{match('...',\ 10,\ \{\ special:\ \{\ 'foo-\/bar':\ null\ \}\ \});}

\end{DoxyCode}


The {\ttfamily special} option is an object where key is a tag name and value is an array of {\ttfamily type} attribute values which, if present in tag, will make it special. If array is not provided, all instances of tag with given name will be considered as special.
