Accepts a CSS string and returns an AST {\ttfamily object}.

{\ttfamily options}\+:


\begin{DoxyItemize}
\item silent\+: silently fail on parse errors.
\item source\+: the path to the file containing {\ttfamily css}. Makes errors and source maps more helpful, by letting them know where code comes from.
\end{DoxyItemize}\hypertarget{Readme.md_autotoc_md9}{}\doxysubsubsection{\texorpdfstring{stringify(object, \mbox{[}options\mbox{]})}{stringify(object, \mbox{[}options\mbox{]})}}\label{Readme.md_autotoc_md9}
Accepts an AST {\ttfamily object} (as {\ttfamily css.\+parse} produces) and returns a CSS string.

{\ttfamily options}\+:


\begin{DoxyItemize}
\item indent\+: the string used to indent the output. Defaults to two spaces.
\item compress\+: omit comments and extraneous whitespace.

Errors thrown during parsing have the following properties\+:


\begin{DoxyItemize}
\item message\+: {\ttfamily String}. The full error message with the source position.
\item reason\+: {\ttfamily String}. The error message without position.
\item filename\+: {\ttfamily String} or {\ttfamily undefined}. The value of {\ttfamily options.\+source} if passed to {\ttfamily css.\+parse}. Otherwise {\ttfamily undefined}.
\item line\+: {\ttfamily Integer}.
\item column\+: {\ttfamily Integer}.
\item source\+: {\ttfamily String}. The portion of code that couldn\textquotesingle{}t be parsed.
\end{DoxyItemize}

When parsing with the {\ttfamily silent} option, errors are listed in the {\ttfamily parsing\+Errors} property of the \`{}stylesheet\`{} node instead of being thrown.

If you create any errors in plugins such as in \href{https://github.com/reworkcss/rework}{\texttt{ rework}}, you {\bfseries{must}} set the same properties for consistency.\hypertarget{Readme.md_autotoc_md12}{}\doxysubsection{\texorpdfstring{AST}{AST}}\label{Readme.md_autotoc_md12}
