From \href{https://github.com/eslint/eslint/releases/tag/v8.21.0}{\texttt{ {\ttfamily v8.\+21.\+0}}}, eslint announced a new config system. In the new system, {\ttfamily .eslintrc\texorpdfstring{$\ast$}{*}} is no longer used. {\ttfamily eslint.\+config.\+js} would be the default config file name. In eslint {\ttfamily v8}, the legacy system ({\ttfamily .eslintrc\texorpdfstring{$\ast$}{*}}) would still be supported, while in eslint {\ttfamily v9}, only the new system would be supported.

And from \href{https://github.com/eslint/eslint/releases/tag/v8.23.0}{\texttt{ {\ttfamily v8.\+23.\+0}}}, eslint CLI starts to look up {\ttfamily eslint.\+config.\+js}. {\bfseries{So, if your eslint is {\ttfamily \texorpdfstring{$>$}{>}=8.\+23.\+0}, you\textquotesingle{}re 100\% ready to use the new config system.}}
