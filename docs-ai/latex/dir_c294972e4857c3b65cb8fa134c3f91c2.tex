\doxysubsection{Detailed Description}
{\ttfamily yaml} is a definitive library for \href{https://yaml.org/}{\texttt{ YAML}}, the human friendly data serialization standard. This library\+:


\begin{DoxyItemize}
\item Supports both YAML 1.\+1 and YAML 1.\+2 and all common data schemas,
\item Passes all of the \href{https://github.com/yaml/yaml-test-suite}{\texttt{ yaml-\/test-\/suite}} tests,
\item Can accept any string as input without throwing, parsing as much YAML out of it as it can, and
\item Supports parsing, modifying, and writing YAML comments and blank lines.
\end{DoxyItemize}

The library is released under the ISC open source license, and the code is \href{https://github.com/eemeli/yaml/}{\texttt{ available on Git\+Hub}}. It has no external dependencies and runs on Node.\+js as well as modern browsers.

For the purposes of versioning, any changes that break any of the documented endpoints or APIs will be considered semver-\/major breaking changes. Undocumented library internals may change between minor versions, and previous APIs may be deprecated (but not removed).

For more information, see the project\textquotesingle{}s documentation site\+: \href{https://eemeli.org/yaml/}{\texttt{ {\bfseries{eemeli.\+org/yaml}}}}

To install\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{npm\ install\ yaml}

\end{DoxyCode}


{\bfseries{Note\+:}} These docs are for {\ttfamily yaml@2}. For v1, see the \href{https://github.com/eemeli/yaml/tree/v1.10.0}{\texttt{ v1.\+10.\+0 tag}} for the source and \href{https://eemeli.org/yaml/v1/}{\texttt{ eemeli.\+org/yaml/v1}} for the documentation.\hypertarget{README.md_autotoc_md36078}{}\doxysubsection{\texorpdfstring{API Overview}{API Overview}}\label{README.md_autotoc_md36078}
The API provided by {\ttfamily yaml} has three layers, depending on how deep you need to go\+: \href{https://eemeli.org/yaml/\#parse-amp-stringify}{\texttt{ Parse \& Stringify}}, \href{https://eemeli.org/yaml/\#documents}{\texttt{ Documents}}, and the underlying \href{https://eemeli.org/yaml/\#parsing-yaml}{\texttt{ Lexer/\+Parser/\+Composer}}. The first has the simplest API and "{}just works"{}, the second gets you all the bells and whistles supported by the library along with a decent \href{https://eemeli.org/yaml/\#content-nodes}{\texttt{ AST}}, and the third lets you get progressively closer to YAML source, if that\textquotesingle{}s your thing.
