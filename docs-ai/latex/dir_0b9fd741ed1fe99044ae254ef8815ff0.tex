\doxysubsection{Detailed Description}
\href{https://travis-ci.org/Rob--W/proxy-from-env}{\texttt{ }} \href{https://coveralls.io/github/Rob--W/proxy-from-env?branch=master}{\texttt{ }}

{\ttfamily proxy-\/from-\/env} is a Node.\+js package that exports a function ({\ttfamily get\+Proxy\+For\+Url}) that takes an input URL (a string or \href{https://nodejs.org/docs/latest/api/url.html\#url_url_parsing}{\texttt{ {\ttfamily url.\+parse}}}\textquotesingle{}s return value) and returns the desired proxy URL (also a string) based on standard proxy environment variables. If no proxy is set, an empty string is returned.

It is your responsibility to actually proxy the request using the given URL.

Installation\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{npm\ install\ proxy-\/from-\/env}

\end{DoxyCode}

\hypertarget{README.md_autotoc_md26680}{}\doxysubsection{\texorpdfstring{Environment variables}{Environment variables}}\label{README.md_autotoc_md26680}
The environment variables can be specified in lowercase or uppercase, with the lowercase name having precedence over the uppercase variant. A variable that is not set has the same meaning as a variable that is set but has no value.\hypertarget{README.md_autotoc_md26681}{}\doxysubsubsection{\texorpdfstring{NO\+\_\+\+PROXY}{NO\+\_\+\+PROXY}}\label{README.md_autotoc_md26681}
{\ttfamily NO\+\_\+\+PROXY} is a list of host names (optionally with a port). If the input URL matches any of the entries in {\ttfamily NO\+\_\+\+PROXY}, then the input URL should be fetched by a direct request (i.\+e. without a proxy).

Matching follows the following rules\+:


\begin{DoxyItemize}
\item {\ttfamily NO\+\_\+\+PROXY=\texorpdfstring{$\ast$}{*}} disables all proxies.
\item Space and commas may be used to separate the entries in the {\ttfamily NO\+\_\+\+PROXY} list.
\item If {\ttfamily NO\+\_\+\+PROXY} does not contain any entries, then proxies are never disabled.
\item If a port is added after the host name, then the ports must match. If the URL does not have an explicit port name, the protocol\textquotesingle{}s default port is used.
\item Generally, the proxy is only disabled if the host name is an exact match for an entry in the {\ttfamily NO\+\_\+\+PROXY} list. The only exceptions are entries that start with a dot or with a wildcard; then the proxy is disabled if the host name ends with the entry.
\end{DoxyItemize}

See {\ttfamily test.\+js} for examples of what should match and what does not.\hypertarget{README.md_autotoc_md26682}{}\doxysubsubsection{\texorpdfstring{\texorpdfstring{$\ast$}{*}\+\_\+\+PROXY}{\texorpdfstring{$\ast$}{*}\+\_\+\+PROXY}}\label{README.md_autotoc_md26682}
The environment variable used for the proxy depends on the protocol of the URL. For example, {\ttfamily \href{https://example.com}{\texttt{ https\+://example.\+com}}} uses the "{}https"{} protocol, and therefore the proxy to be used is {\ttfamily HTTPS\+\_\+\+PROXY} ({\itshape NOT} {\ttfamily HTTP\+\_\+\+PROXY}, which is {\itshape only} used for \href{http:-URLs}{\texttt{ http\+:-\/\+URLs}}).

The library is not limited to http(s), other schemes such as {\ttfamily FTP\+\_\+\+PROXY} (ftp\+:), {\ttfamily WSS\+\_\+\+PROXY} (wss\+:), {\ttfamily WS\+\_\+\+PROXY} (ws\+:) are also supported.

If present, {\ttfamily ALL\+\_\+\+PROXY} is used as fallback if there is no other match.\hypertarget{README.md_autotoc_md26683}{}\doxysubsection{\texorpdfstring{External resources}{External resources}}\label{README.md_autotoc_md26683}
