This package is a utility that takes a DOM tree (from the actual DOM or from things like \href{https://github.com/jsdom/jsdom}{\texttt{ {\ttfamily jsdom}}}) as input and turns it into a \href{https://github.com/syntax-tree/hast}{\texttt{ hast}} (HTML) syntax tree.\hypertarget{readme.md_autotoc_md16566}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{readme.md_autotoc_md16566}
You can use this project when you want to use hast in browsers. This package is very small, but it does so by\+:


\begin{DoxyItemize}
\item …not providing positional information
\item …potentially yielding varying results in different (especially older) browsers
\end{DoxyItemize}

The hast utility \href{https://github.com/syntax-tree/hast-util-to-dom}{\texttt{ {\ttfamily hast-\/util-\/to-\/dom}}} does the inverse of this utility. It turns hast into a DOM tree.

The rehype plugin \href{https://github.com/rehypejs/rehype-dom/tree/main/packages/rehype-dom-parse}{\texttt{ {\ttfamily rehype-\/dom-\/parse}}} wraps this utility to parse HTML with DOM APIs.\hypertarget{readme.md_autotoc_md16567}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md16567}
