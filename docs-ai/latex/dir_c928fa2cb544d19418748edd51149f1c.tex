\begin{quote}
Check if stdout or stderr is \href{https://unix.stackexchange.com/a/43389/7678}{\texttt{ interactive}} \end{quote}


It checks that the stream is \href{https://jameshfisher.com/2017/12/09/what-is-a-tty/}{\texttt{ TTY}}, not a dumb terminal, and not running in a CI.

This can be useful to decide whether to present interactive UI or animations in the terminal.\hypertarget{readme.md_autotoc_md18637}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md18637}

It\textquotesingle{}s silly to have to detect individual CIs. They should identify themselves with the {\ttfamily CI} environment variable, and most do just that. A manually maintained list of detections will easily get out of date. And if a package using {\ttfamily ci-\/info} doesn\textquotesingle{}t update to the latest version all the time, they will not support certain CIs. It also creates unpredictability as you might assume a CI is not supported and then suddenly it gets supported and you didn\textquotesingle{}t account for that. In addition, some of the manual detections are loose and might cause false-\/positives which could create hard-\/to-\/debug bugs.\hypertarget{readme.md_autotoc_md18645}{}\doxysubsubsubsection{\texorpdfstring{Why does this even exist? It\textquotesingle{}s just a few lines.}{Why does this even exist? It\textquotesingle{}s just a few lines.}}\label{readme.md_autotoc_md18645}
It\textquotesingle{}s not about the number of lines, but rather discoverability and documentation. A lot of people wouldn\textquotesingle{}t even know they need this.
