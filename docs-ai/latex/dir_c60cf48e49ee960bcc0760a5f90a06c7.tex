\hypertarget{README.md_autotoc_md4112}{}\doxysubsubsection{\texorpdfstring{safe\+Load (string \mbox{[} , options \mbox{]})}{safe\+Load (string \mbox{[} , options \mbox{]})}}\label{README.md_autotoc_md4112}
{\bfseries{Recommended loading way.}} Parses {\ttfamily string} as single YAML document. Returns either a plain object, a string or {\ttfamily undefined}, or throws {\ttfamily YAMLException} on error. By default, does not support regexps, functions and undefined. This method is safe for untrusted data.

options\+:


\begin{DoxyItemize}
\item {\ttfamily filename} {\itshape (default\+: null)} -\/ string to be used as a file path in error/warning messages.
\item {\ttfamily on\+Warning} {\itshape (default\+: null)} -\/ function to call on warning messages. Loader will call this function with an instance of {\ttfamily YAMLException} for each warning.
\item {\ttfamily schema} {\itshape (default\+: {\ttfamily DEFAULT\+\_\+\+SAFE\+\_\+\+SCHEMA})} -\/ specifies a schema to use.
\begin{DoxyItemize}
\item {\ttfamily FAILSAFE\+\_\+\+SCHEMA} -\/ only strings, arrays and plain objects\+: \href{http://www.yaml.org/spec/1.2/spec.html\#id2802346}{\texttt{ http\+://www.\+yaml.\+org/spec/1.\+2/spec.\+html\#id2802346}}
\item {\ttfamily JSON\+\_\+\+SCHEMA} -\/ all JSON-\/supported types\+: \href{http://www.yaml.org/spec/1.2/spec.html\#id2803231}{\texttt{ http\+://www.\+yaml.\+org/spec/1.\+2/spec.\+html\#id2803231}}
\item {\ttfamily CORE\+\_\+\+SCHEMA} -\/ same as {\ttfamily JSON\+\_\+\+SCHEMA}\+: \href{http://www.yaml.org/spec/1.2/spec.html\#id2804923}{\texttt{ http\+://www.\+yaml.\+org/spec/1.\+2/spec.\+html\#id2804923}}
\item {\ttfamily DEFAULT\+\_\+\+SAFE\+\_\+\+SCHEMA} -\/ all supported YAML types, without unsafe ones ({\ttfamily !!js/undefined}, {\ttfamily !!js/regexp} and {\ttfamily !!js/function})\+: \href{http://yaml.org/type/}{\texttt{ http\+://yaml.\+org/type/}}
\item {\ttfamily DEFAULT\+\_\+\+FULL\+\_\+\+SCHEMA} -\/ all supported YAML types.
\end{DoxyItemize}
\item {\ttfamily json} {\itshape (default\+: false)} -\/ compatibility with JSON.\+parse behaviour. If true, then duplicate keys in a mapping will override values rather than throwing an error.
\end{DoxyItemize}

NOTE\+: This function {\bfseries{does not}} understand multi-\/document sources, it throws exception on those.

NOTE\+: JS-\/\+YAML {\bfseries{does not}} support schema-\/specific tag resolution restrictions. So, the JSON schema is not as strictly defined in the YAML specification. It allows numbers in any notation, use {\ttfamily Null} and {\ttfamily NULL} as {\ttfamily null}, etc. The core schema also has no such restrictions.
