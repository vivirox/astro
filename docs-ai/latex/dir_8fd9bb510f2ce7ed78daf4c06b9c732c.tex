This ESLint plugin provides linting rules for \href{https://yaml.org/}{\texttt{ YAML}}.


\begin{DoxyItemize}
\item You can use ESLint to lint \href{https://yaml.org/}{\texttt{ YAML}}.
\item You can find out the problem with your \href{https://yaml.org/}{\texttt{ YAML}} files.
\item You can apply consistent code styles to your \href{https://yaml.org/}{\texttt{ YAML}} files.
\item Supports \href{https://vue-loader.vuejs.org/spec.html}{\texttt{ Vue SFC}} custom blocks such as {\ttfamily \texorpdfstring{$<$}{<}i18n lang="{}yaml"{}\texorpdfstring{$>$}{>}}. ~\newline
 Requirements {\ttfamily vue-\/eslint-\/parser} v7.\+3.\+0 and above.
\item Supports ESLint directives. e.\+g. {\ttfamily \# eslint-\/disable-\/next-\/line}
\item You can check your code in real-\/time using the ESLint editor integrations.
\end{DoxyItemize}

These plugins use the processor to parse and return the results independently, without providing the ESLint engine with AST and source code text.

Plugins don\textquotesingle{}t provide AST, so you can\textquotesingle{}t use directive comments (e.\+g. {\ttfamily \# eslint-\/disable}). ~\newline
 Plugins don\textquotesingle{}t provide source code text, so you can\textquotesingle{}t use it with plugins and rules that use text (e.\+g. \href{https://github.com/prettier/eslint-plugin-prettier}{\texttt{ eslint-\/plugin-\/prettier}}, \href{https://eslint.org/docs/rules/eol-last}{\texttt{ eol-\/last}}).
