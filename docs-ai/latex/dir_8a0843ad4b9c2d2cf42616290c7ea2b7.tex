\end{DoxyItemize}\hypertarget{README.md_autotoc_md6399}{}\doxysubsubsection{\texorpdfstring{The problem}{The problem}}\label{README.md_autotoc_md6399}
You want to write maintainable tests for your React components. As a part of this goal, you want your tests to avoid including implementation details of your components and rather focus on making your tests give you the confidence for which they are intended. As part of this, you want your testbase to be maintainable in the long run so refactors of your components (changes to implementation but not functionality) don\textquotesingle{}t break your tests and slow you and your team down.\hypertarget{README.md_autotoc_md6400}{}\doxysubsubsection{\texorpdfstring{The solution}{The solution}}\label{README.md_autotoc_md6400}
The {\ttfamily React Testing Library} is a very lightweight solution for testing React components. It provides light utility functions on top of {\ttfamily react-\/dom} and {\ttfamily react-\/dom/test-\/utils}, in a way that encourages better testing practices. Its primary guiding principle is\+:

\begin{quote}
\href{https://twitter.com/kentcdodds/status/977018512689455106}{\texttt{ The more your tests resemble the way your software is used, the more confidence they can give you.}} \end{quote}

\begin{DoxyCode}{0}

\DoxyCodeLine{//\ upgrade\ to\ 16.9.\ See\ also:\ https://github.com/facebook/react/pull/14853}
\DoxyCodeLine{const\ originalError\ =\ console.error}
\DoxyCodeLine{beforeAll(()\ =>\ \{}
\DoxyCodeLine{\ \ console.error\ =\ (...args)\ =>\ \{}
\DoxyCodeLine{\ \ \ \ if\ (/Warning.*not\ wrapped\ in\ act/.test(args[0]))\ \{}
\DoxyCodeLine{\ \ \ \ \ \ return}
\DoxyCodeLine{\ \ \ \ \}}
\DoxyCodeLine{\ \ \ \ originalError.call(console,\ ...args)}
\DoxyCodeLine{\ \ \}}
\DoxyCodeLine{\})}

\DoxyCodeLine{afterAll(()\ =>\ \{}
\DoxyCodeLine{\ \ console.error\ =\ originalError}
\DoxyCodeLine{\})}

\end{DoxyCode}

\hypertarget{README.md_autotoc_md6408}{}\doxysubsubsection{\texorpdfstring{Guiding Principles}{Guiding Principles}}\label{README.md_autotoc_md6408}
\begin{quote}
\href{https://twitter.com/kentcdodds/status/977018512689455106}{\texttt{ The more your tests resemble the way your software is used, the more confidence they can give you.}} \end{quote}


We try to only expose methods and utilities that encourage you to write tests that closely resemble how your React components are used.

Utilities are included in this project based on the following guiding principles\+:


\begin{DoxyEnumerate}
\item If it relates to rendering components, it deals with DOM nodes rather than component instances, nor should it encourage dealing with component instances.
\item It should be generally useful for testing individual React components or full React applications. While this library is focused on {\ttfamily react-\/dom}, utilities could be included even if they don\textquotesingle{}t directly relate to {\ttfamily react-\/dom}.
\item Utility implementations and APIs should be simple and flexible.
\end{DoxyEnumerate}
