Join the conversation in \href{https://github.com/pkgjs/parseargs}{\texttt{ pkgjs/parseargs}} to contribute to the design.\hypertarget{README.md_autotoc_md5448}{}\doxysubsubsubsection{\texorpdfstring{{\ttfamily parse\+Args} {\ttfamily tokens}}{{\ttfamily parse\+Args} {\ttfamily tokens}}}\label{README.md_autotoc_md5448}
Detailed parse information is available for adding custom behaviours by specifying {\ttfamily tokens\+: true} in the configuration. The returned tokens have properties describing\+:


\begin{DoxyItemize}
\item all tokens
\begin{DoxyItemize}
\item {\ttfamily kind} \{string\} One of \textquotesingle{}option\textquotesingle{}, \textquotesingle{}positional\textquotesingle{}, or \textquotesingle{}option-\/terminator\textquotesingle{}.
\item {\ttfamily index} \{number\} Index of element in {\ttfamily args} containing token. So the source argument for a token is {\ttfamily args\mbox{[}token.\+index\mbox{]}}.
\end{DoxyItemize}
\item option tokens
\begin{DoxyItemize}
\item {\ttfamily name} \{string\} Long name of option.
\item {\ttfamily raw\+Name} \{string\} How option used in args, like {\ttfamily -\/f} of {\ttfamily -\/-\/foo}.
\item {\ttfamily value} \{string \texorpdfstring{$\vert$}{|} undefined\} Option value specified in args. Undefined for boolean options.
\item {\ttfamily inline\+Value} \{boolean \texorpdfstring{$\vert$}{|} undefined\} Whether option value specified inline, like {\ttfamily -\/-\/foo=bar}.
\end{DoxyItemize}
\item positional tokens
\begin{DoxyItemize}
\item {\ttfamily value} \{string\} The value of the positional argument in args (i.\+e. {\ttfamily args\mbox{[}index\mbox{]}}).
\end{DoxyItemize}
\item option-\/terminator token
\end{DoxyItemize}

The returned tokens are in the order encountered in the input args. Options that appear more than once in args produce a token for each use. Short option groups like {\ttfamily -\/xy} expand to a token for each option. So {\ttfamily -\/xxx} produces three tokens.

For example to use the returned tokens to add support for a negated option like {\ttfamily -\/-\/no-\/color}, the tokens can be reprocessed to change the value stored for the negated option.

\hypertarget{README.md_autotoc_md5452}{}\doxysubsubsection{\texorpdfstring{Scope}{Scope}}\label{README.md_autotoc_md5452}
It is already possible to build great arg parsing modules on top of what Node.\+js provides; the prickly API is abstracted away by these modules. Thus, process.\+parse\+Args() is not necessarily intended for library authors; it is intended for developers of simple CLI tools, ad-\/hoc scripts, deployed Node.\+js applications, and learning materials.

It is exceedingly difficult to provide an API which would both be friendly to these Node.\+js users while being extensible enough for libraries to build upon. We chose to prioritize these use cases because these are currently not well-\/served by Node.\+js\textquotesingle{} API.
