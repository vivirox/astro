{\ttfamily micromark} is an open source markdown parser written in Java\+Script. It‚Äôs implemented as a state machine that emits concrete tokens, so that every byte is accounted for, with positional info. It then compiles those tokens directly to HTML, but other tools can take the data and for example build an AST which is easier to work with (\href{https://github.com/syntax-tree/mdast-util-to-markdown}{\texttt{ {\ttfamily mdast-\/util-\/to-\/markdown}}}).

While most markdown parsers work towards compliancy with Common\+Mark (or GFM), this project goes further by following how the reference parsers ({\ttfamily cmark}, {\ttfamily cmark-\/gfm}) work, which is confirmed with thousands of extra tests.

Other than Common\+Mark and GFM, micromark also supports common extensions to markdown such as MDX, math, and frontmatter.

These npm packages have a sibling project in Rust\+: \href{https://github.com/wooorm/markdown-rs}{\texttt{ {\ttfamily markdown-\/rs}}}.

Whether to allow dangerous protocols in links and images ({\ttfamily boolean}, default\+: {\ttfamily false}).

The default is {\ttfamily false}, which drops URLs in links and images that use dangerous protocols.
