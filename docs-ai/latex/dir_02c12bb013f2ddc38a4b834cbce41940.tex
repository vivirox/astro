This package is a \href{https://github.com/unifiedjs/unified}{\texttt{ unified}} (\href{https://github.com/rehypejs/rehype}{\texttt{ rehype}}) plugin to add links from headings back to themselves. It looks for headings (so {\ttfamily \texorpdfstring{$<$}{<}h1\texorpdfstring{$>$}{>}} through {\ttfamily \texorpdfstring{$<$}{<}h6\texorpdfstring{$>$}{>}}) that have {\ttfamily id} properties, and injects a link to themselves. Similar functionality is applied by many places that render markdown. For example, when browsing this readme on Git\+Hub or npm, an anchor is added to headings, which you can share to point people to a particular place in a document.

{\bfseries{unified}} is a project that transforms content with abstract syntax trees (ASTs). {\bfseries{rehype}} adds support for HTML to unified. {\bfseries{hast}} is the HTML AST that rehype uses. This is a rehype plugin that adds links to headings in the AST.\hypertarget{readme.md_autotoc_md28697}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{readme.md_autotoc_md28697}
This plugin is useful when you have relatively long documents, where you want users to be able to link to particular sections, and you already have {\ttfamily id} properties set on all (or certain?) headings.

A different plugin, \href{https://github.com/rehypejs/rehype-slug}{\texttt{ {\ttfamily rehype-\/slug}}}, adds {\ttfamily id}s to headings. When a heading doesn’t already have an {\ttfamily id} property, it creates a slug from it, and adds that as the {\ttfamily id} property. When using both plugins together, all headings (whether explicitly with a certain {\ttfamily id} or automatically with a generate one) will get a link back to themselves.\hypertarget{readme.md_autotoc_md28698}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md28698}

This package is fully typed with \href{https://www.typescriptlang.org}{\texttt{ Type\+Script}}. It exports the additional types \`{}\+Behavior\`{}, \`{}\+Build\`{}, \`{}\+Build\+Properties\`{}, and \doxylink{README.md_options}{\`{}\+Options\`{}}.\hypertarget{readme.md_autotoc_md28721}{}\doxysubsection{\texorpdfstring{Compatibility}{Compatibility}}\label{readme.md_autotoc_md28721}
Projects maintained by the unified collective are compatible with maintained versions of Node.\+js.

When we cut a new major release, we drop support for unmaintained versions of Node. This means we try to keep the current release line, {\ttfamily rehype-\/autolink-\/headings@\texorpdfstring{$^\wedge$}{\string^}7}, compatible with Node.\+js 16.

This plugin works with {\ttfamily rehype-\/parse} version 1+, {\ttfamily rehype-\/stringify} version 1+, {\ttfamily rehype} version 1+, and {\ttfamily unified} version 4+.\hypertarget{readme.md_autotoc_md28722}{}\doxysubsection{\texorpdfstring{Security}{Security}}\label{readme.md_autotoc_md28722}
Use of {\ttfamily rehype-\/autolink-\/headings} can open you up to a \href{https://en.wikipedia.org/wiki/Cross-site_scripting}{\texttt{ cross-\/site scripting (XSS)}} attack if you pass user provided content in {\ttfamily content}, {\ttfamily group}, or {\ttfamily properties}.

Always be wary of user input and use \href{https://github.com/rehypejs/rehype-sanitize}{\texttt{ {\ttfamily rehype-\/sanitize}}}.\hypertarget{readme.md_autotoc_md28723}{}\doxysubsection{\texorpdfstring{Related}{Related}}\label{readme.md_autotoc_md28723}
