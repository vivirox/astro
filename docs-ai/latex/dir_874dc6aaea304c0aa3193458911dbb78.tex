This package is a \href{https://github.com/unifiedjs/unified}{\texttt{ unified}} (\href{https://github.com/rehypejs/rehype}{\texttt{ rehype}}) plugin to parse a document again. To understand how it works, requires knowledge of ASTs (specifically, \href{https://github.com/syntax-tree/hast}{\texttt{ hast}}). This plugin passes each node and embedded raw HTML through an HTML parser (\href{https://github.com/inikulin/parse5}{\texttt{ {\ttfamily parse5}}}), to recreate a tree exactly as how a browser would parse it, while keeping the original data and positional info intact.

{\bfseries{unified}} is a project that transforms content with abstract syntax trees (ASTs). {\bfseries{rehype}} adds support for HTML to unified. {\bfseries{hast}} is the HTML AST that rehype uses. This is a rehype plugin that parses the tree again.\hypertarget{readme.md_autotoc_md28896}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{readme.md_autotoc_md28896}
This plugin is particularly useful when coming from markdown and wanting to support HTML embedded inside that markdown (which requires passing {\ttfamily allow\+Dangerous\+Html\+: true} to {\ttfamily remark-\/rehype}). Markdown dictates how, say, a list item or emphasis can be parsed. We can use that to turn the markdown syntax tree into an HTML syntax tree. But markdown also dictates that things that look like HTML, are passed through untouched, even when it just looks like XML but doesn‚Äôt really make sense, so we can‚Äôt normally use these strings of ‚Äú\+HTML‚Äù to create an HTML syntax tree. This plugin can. It can be used to take those strings of HTML and include them into the syntax tree as actual nodes.

If your final result is HTML and you trust content, then ‚Äústrings‚Äù are fine (you can pass {\ttfamily allow\+Dangerous\+Html\+: true} to {\ttfamily rehype-\/stringify}, which passes HTML through untouched). But there are two main cases where a proper syntax tree is preferred\+:


\begin{DoxyItemize}
\item rehype plugins need a proper syntax tree as they operate on actual nodes to inspect or transform things, they can‚Äôt operate on strings of HTML
\item other output formats (React, MDX, etc) need actual nodes and can‚Äôt handle strings of HTML
\end{DoxyItemize}

This plugin is built on \href{https://github.com/syntax-tree/hast-util-raw}{\texttt{ {\ttfamily hast-\/util-\/raw}}}, which does the work on syntax trees. rehype focusses on making it easier to transform content by abstracting such internals away.\hypertarget{readme.md_autotoc_md28897}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md28897}
