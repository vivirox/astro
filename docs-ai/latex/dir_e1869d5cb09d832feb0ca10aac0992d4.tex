This package is a utility that takes HTML input and turns it into a hast syntax tree.

In a browser, this uses \href{https://github.com/syntax-tree/hast-util-from-dom}{\texttt{ {\ttfamily hast-\/util-\/from-\/dom}}}, otherwise it uses \href{https://github.com/syntax-tree/hast-util-from-html}{\texttt{ {\ttfamily hast-\/util-\/from-\/html}}}.\hypertarget{readme.md_autotoc_md16588}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{readme.md_autotoc_md16588}
If you want to get a syntax tree without positional info, and your code should be isomorphic (it could run anywhere), as it results in a smaller bundle size.

If you need positional information, use \href{https://github.com/syntax-tree/hast-util-from-html}{\texttt{ {\ttfamily hast-\/util-\/from-\/html}}}.

If you donâ€™t care about positional info and your code only runs in browsers, use \href{https://github.com/syntax-tree/hast-util-from-dom}{\texttt{ {\ttfamily hast-\/util-\/from-\/dom}}}.

Finally you can use the utility \href{https://github.com/syntax-tree/hast-util-to-html}{\texttt{ {\ttfamily hast-\/util-\/to-\/html}}}, or \href{https://github.com/syntax-tree/hast-util-to-dom}{\texttt{ {\ttfamily hast-\/util-\/to-\/dom}}} with {\ttfamily .outer\+HTML}, to do the inverse of this utility. That turns hast into HTML.\hypertarget{readme.md_autotoc_md16589}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md16589}
