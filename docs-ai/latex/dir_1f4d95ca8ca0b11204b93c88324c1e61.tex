This package is a \href{https://github.com/unifiedjs/unified}{\texttt{ unified}} (\href{https://github.com/mdx-js/recma}{\texttt{ recma}}) plugin that defines how to take Java\+Script as input and turn it into a syntax tree. When it’s used, Java\+Script can be parsed and other recma plugins can be used.

See \href{https://github.com/mdx-js/recma}{\texttt{ the monorepo readme}} for info on what the recma ecosystem is.\hypertarget{readme.md_autotoc_md27972}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{readme.md_autotoc_md27972}
This plugin adds support to unified for parsing Java\+Script. If you also need to serialize Java\+Script, you can alternatively use \href{https://github.com/mdx-js/recma/tree/main/packages/recma}{\texttt{ {\ttfamily recma}}}, which combines unified, this plugin, and \href{https://github.com/mdx-js/recma/tree/main/packages/recma-stringify}{\texttt{ {\ttfamily recma-\/stringify}}}.

If you don’t use plugins and want to access the syntax tree, you can directly use \href{https://github.com/syntax-tree/esast-util-from-js}{\texttt{ {\ttfamily esast-\/util-\/from-\/js}}}, which is used inside this plugin. recma focusses on making it easier to transform code by abstracting such internals away.\hypertarget{readme.md_autotoc_md27973}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md27973}
