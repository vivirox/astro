\doxysubsection{Detailed Description}
Data in Java\+Script is often represented by an iterable (such as an \href{https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Array}{\texttt{ array}}, \href{https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Set}{\texttt{ set}} or \href{https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Generator}{\texttt{ generator}}), and so iterable manipulation is a common task when analyzing or visualizing data. For example, you might take a contiguous slice (subset) of an array, filter an array using a predicate function, or map an array to a parallel set of values using a transform function. Before looking at the methods that d3-\/array provides, familiarize yourself with the powerful \href{https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Array}{\texttt{ array methods built-\/in to Java\+Script}}.

Java\+Script includes {\bfseries{mutation methods}} that modify the array\+:

\item \href{https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Array/push}{\texttt{ {\itshape array}.push}} -\/ Add one or more elements to the end of the array.
\item \href{https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Array/reverse}{\texttt{ {\itshape array}.reverse}} -\/ Reverse the order of the elements of the array.
\item \href{https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Array/shift}{\texttt{ {\itshape array}.shift}} -\/ Remove the first element from the array.
\item \href{https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Array/sort}{\texttt{ {\itshape array}.sort}} -\/ Sort the elements of the array.
\item \href{https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Array/splice}{\texttt{ {\itshape array}.splice}} -\/ Add or remove elements from the array.
\item \href{https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/Array/unshift}{\texttt{ {\itshape array}.unshift}} -\/ Add one or more elements to the front of the array.

\# d3.{\bfseries{range}}(\mbox{[}{\itshape start}, \mbox{]}{\itshape stop}\mbox{[}, {\itshape step}\mbox{]}) · \href{https://github.com/d3/d3-array/blob/main/src/range.js}{\texttt{ Source}}, \href{https://observablehq.com/@d3/d3-range}{\texttt{ Examples}}

Returns an array containing an arithmetic progression, similar to the Python built-\/in \href{http://docs.python.org/library/functions.html\#range}{\texttt{ range}}. This method is often used to iterate over a sequence of uniformly-\/spaced numeric values, such as the indexes of an array or the ticks of a linear scale. (See also \doxylink{README.md_ticks}{d3.ticks} for nicely-\/rounded values.)

If {\itshape step} is omitted, it defaults to 1. If {\itshape start} is omitted, it defaults to 0. The {\itshape stop} value is exclusive; it is not included in the result. If {\itshape step} is positive, the last element is the largest {\itshape start} + {\itshape i} \texorpdfstring{$\ast$}{*} {\itshape step} less than {\itshape stop}; if {\itshape step} is negative, the last element is the smallest {\itshape start} + {\itshape i} \texorpdfstring{$\ast$}{*} {\itshape step} greater than {\itshape stop}. If the returned array would contain an infinite number of values, an empty range is returned.

The arguments are not required to be integers; however, the results are more predictable if they are. The values in the returned array are defined as {\itshape start} + {\itshape i} \texorpdfstring{$\ast$}{*} {\itshape step}, where {\itshape i} is an integer from zero to one minus the total number of elements in the returned array. For example\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{d3.range(0,\ 1,\ 0.2)\ //\ [0,\ 0.2,\ 0.4,\ 0.6000000000000001,\ 0.8]}

\end{DoxyCode}

Binning groups discrete samples into a smaller number of consecutive, non-\/overlapping intervals. They are often used to visualize the distribution of numerical data as histograms.

\label{README.md_bin}%
\Hypertarget{README.md_bin}%
\# d3.{\bfseries{bin}}() · \href{https://github.com/d3/d3-array/blob/main/src/bin.js}{\texttt{ Source}}, \href{https://observablehq.com/@d3/d3-bin}{\texttt{ Examples}}

Constructs a new bin generator with the default settings.

\label{README.md__bin}%
\Hypertarget{README.md__bin}%
\# {\itshape bin}({\itshape data}) · \href{https://github.com/d3/d3-array/blob/main/src/bin.js}{\texttt{ Source}}, \href{https://observablehq.com/@d3/d3-bin}{\texttt{ Examples}}

Bins the given iterable of {\itshape data} samples. Returns an array of bins, where each bin is an array containing the associated elements from the input {\itshape data}. Thus, the {\ttfamily length} of the bin is the number of elements in that bin. Each bin has two additional attributes\+:


\begin{DoxyItemize}
\item {\ttfamily x0} -\/ the lower bound of the bin (inclusive).
\item {\ttfamily x1} -\/ the upper bound of the bin (exclusive, except for the last bin).
\end{DoxyItemize}

Any null or non-\/comparable values in the given {\itshape data}, or those outside the \doxylink{README.md_bin_domain}{domain}, are ignored.

\label{README.md_bin_value}%
\Hypertarget{README.md_bin_value}%
\# {\itshape bin}.{\bfseries{value}}(\mbox{[}{\itshape value}\mbox{]}) · \href{https://github.com/d3/d3-array/blob/main/src/bin.js}{\texttt{ Source}}, \href{https://observablehq.com/@d3/d3-bin}{\texttt{ Examples}}

If {\itshape value} is specified, sets the value accessor to the specified function or constant and returns this bin generator. If {\itshape value} is not specified, returns the current value accessor, which defaults to the identity function.

When bins are \doxylink{README.md__bin}{generated}, the value accessor will be invoked for each element in the input data array, being passed the element {\ttfamily d}, the index {\ttfamily i}, and the array {\ttfamily data} as three arguments. The default value accessor assumes that the input data are orderable (comparable), such as numbers or dates. If your data are not, then you should specify an accessor that returns the corresponding orderable value for a given datum.

This is similar to mapping your data to values before invoking the bin generator, but has the benefit that the input data remains associated with the returned bins, thereby making it easier to access other fields of the data.

\label{README.md_bin_domain}%
\Hypertarget{README.md_bin_domain}%
\# {\itshape bin}.{\bfseries{domain}}(\mbox{[}{\itshape domain}\mbox{]}) · \href{https://github.com/d3/d3-array/blob/main/src/bin.js}{\texttt{ Source}}, \href{https://observablehq.com/@d3/d3-bin}{\texttt{ Examples}}

If {\itshape domain} is specified, sets the domain accessor to the specified function or array and returns this bin generator. If {\itshape domain} is not specified, returns the current domain accessor, which defaults to \doxylink{README.md_extent}{extent}. The bin domain is defined as an array \mbox{[}{\itshape min}, {\itshape max}\mbox{]}, where {\itshape min} is the minimum observable value and {\itshape max} is the maximum observable value; both values are inclusive. Any value outside of this domain will be ignored when the bins are \doxylink{README.md__bin}{generated}.

For example, if you are using the bin generator in conjunction with a \href{https://github.com/d3/d3-scale/blob/main/README.md\#linear-scales}{\texttt{ linear scale}} {\ttfamily x}, you might say\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var\ bin\ =\ d3.bin()}
\DoxyCodeLine{\ \ \ \ .domain(x.domain())}
\DoxyCodeLine{\ \ \ \ .thresholds(x.ticks(20));}

\end{DoxyCode}


You can then compute the bins from an array of numbers like so\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{var\ bins\ =\ bin(numbers);}

\end{DoxyCode}


If the default \doxylink{README.md_extent}{extent} domain is used and the \doxylink{README.md_bin_thresholds}{thresholds} are specified as a count (rather than explicit values), then the computed domain will be \doxylink{README.md_nice}{niced} such that all bins are uniform width.

Note that the domain accessor is invoked on the materialized array of \doxylink{README.md_bin_value}{values}, not on the input data array.

\# {\itshape bin}.{\bfseries{thresholds}}(\mbox{[}{\itshape thresholds}\mbox{]})

If {\itshape thresholds} is specified, sets the threshold generator to the specified function or array and returns this bin generator. If {\itshape thresholds} is not specified, returns the current threshold generator, which by default implements \doxylink{README.md_thresholdSturges}{Sturges’ formula}. (Thus by default, the values to be binned must be numbers!) Thresholds are defined as an array of values \mbox{[}{\itshape x0}, {\itshape x1}, …\mbox{]}. Any value less than {\itshape x0} will be placed in the first bin; any value greater than or equal to {\itshape x0} but less than {\itshape x1} will be placed in the second bin; and so on. Thus, the \doxylink{README.md__bin}{generated bins} will have {\itshape thresholds}.length + 1 bins. See bin thresholds for more information.

Any threshold values outside the \doxylink{README.md_bin_domain}{domain} are ignored. The first {\itshape bin}.x0 is always equal to the minimum domain value, and the last {\itshape bin}.x1 is always equal to the maximum domain value.

If a {\itshape count} is specified instead of an array of {\itshape thresholds}, then the \doxylink{README.md_bin_domain}{domain} will be uniformly divided into approximately {\itshape count} bins; see \doxylink{README.md_ticks}{ticks}.\hypertarget{README.md_autotoc_md11215}{}\doxysubsubsubsection{\texorpdfstring{Bin Thresholds}{Bin Thresholds}}\label{README.md_autotoc_md11215}
