\+::\+:danger Deprecated

The old {\ttfamily ban-\/types} rule encompassed multiple areas of functionality, and so has been split into several rules.

{\bfseries{\href{./no-restricted-types.mdx}{\texttt{ {\ttfamily no-\/restricted-\/types}}}}} is the new rule for banning a configurable list of type names. It has no options enabled by default and is akin to rules like \href{https://eslint.org/docs/latest/rules/no-restricted-globals}{\texttt{ {\ttfamily no-\/restricted-\/globals}}}, \href{https://eslint.org/docs/latest/rules/no-restricted-properties}{\texttt{ {\ttfamily no-\/restricted-\/properties}}}, and \href{https://eslint.org/docs/latest/rules/no-restricted-syntax}{\texttt{ {\ttfamily no-\/restricted-\/syntax}}}.

The default options from {\ttfamily ban-\/types} are now covered by\+:


\begin{DoxyItemize}
\item {\bfseries{\href{./no-empty-object-type.mdx}{\texttt{ {\ttfamily no-\/empty-\/object-\/type}}}}}\+: banning the built-\/in {\ttfamily \{\}} type in confusing locations
\item {\bfseries{\href{./no-unsafe-function-type.mdx}{\texttt{ {\ttfamily no-\/unsafe-\/function-\/type}}}}}\+: banning the built-\/in {\ttfamily Function}
\item {\bfseries{\href{./no-wrapper-object-types.mdx}{\texttt{ {\ttfamily no-\/wrapper-\/object-\/types}}}}}\+: banning {\ttfamily Object} and built-\/in class wrappers such as {\ttfamily Number}
\end{DoxyItemize}

{\ttfamily ban-\/types} itself is removed in typescript-\/eslint v8. See \href{/blog/announcing-typescript-eslint-v8-beta}{\texttt{ Announcing typescript-\/eslint v8 Beta}} for more details. \+::\+:
