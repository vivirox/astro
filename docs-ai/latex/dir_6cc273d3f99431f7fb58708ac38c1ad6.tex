When {\ttfamily term} is {\ttfamily undefined}, it means the search term input is empty. You can use this to return default choices, or return an empty array.

Aside from returning the choices\+:


\begin{DoxyEnumerate}
\item An {\ttfamily Abort\+Signal} is passed in to cancel ongoing network calls when the search term change.
\item {\ttfamily Separator}s can be used to organize the list.

Here\textquotesingle{}s each property\+:


\begin{DoxyItemize}
\item {\ttfamily value}\+: The value is what will be returned by {\ttfamily await search()}.
\item {\ttfamily name}\+: This is the string displayed in the choice list.
\item {\ttfamily description}\+: Option for a longer description string that\textquotesingle{}ll appear under the list when the cursor highlight a given choice.
\item {\ttfamily short}\+: Once the prompt is done (press enter), we\textquotesingle{}ll use {\ttfamily short} if defined to render next to the question. By default we\textquotesingle{}ll use {\ttfamily name}.
\item {\ttfamily disabled}\+: Disallow the option from being selected. If {\ttfamily disabled} is a string, it\textquotesingle{}ll be used as a help tip explaining why the choice isn\textquotesingle{}t available.
\end{DoxyItemize}

Choices can also be an array of string, in which case the string will be used both as the {\ttfamily value} and the {\ttfamily name}.\hypertarget{README.md_autotoc_md3811}{}\doxysubsubsubsection{\texorpdfstring{Validation \& autocomplete interaction}{Validation \& autocomplete interaction}}\label{README.md_autotoc_md3811}
The validation within the search prompt acts as a signal for the autocomplete feature.

When a list value is submitted and fail validation, the prompt will compare it to the search term. If they\textquotesingle{}re the same, the prompt display the error. If they\textquotesingle{}re not the same, we\textquotesingle{}ll autocomplete the search term to match the value. Doing this will trigger a new search.

You can rely on this behavior to implement progressive autocomplete searches. Where you want the user to narrow the search in a progressive manner.

Pressing {\ttfamily tab} also triggers the term autocomplete.

You can see this behavior in action in \href{https://github.com/SBoudrias/Inquirer.js/blob/main/packages/demo/demos/search.mjs}{\texttt{ our search demo}}.\hypertarget{README.md_autotoc_md3812}{}\doxysubsubsection{\texorpdfstring{Theming}{Theming}}\label{README.md_autotoc_md3812}
