Problems reported by rules which have a wrench \+:wrench\+: below can be fixed automatically by running ESLint on the command line with {\ttfamily -\/-\/fix} option.

Note that a number of fixable rules have an {\ttfamily enable\+Fixer} option which can be set to {\ttfamily false} to disable the fixer (or in the case of {\ttfamily check-\/param-\/names}, {\ttfamily check-\/property-\/names}, and {\ttfamily no-\/blank-\/blocks}, set to {\ttfamily true} to enable a non-\/default-\/recommended fixer).
