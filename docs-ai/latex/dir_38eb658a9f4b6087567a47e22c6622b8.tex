{\ttfamily @octokit/webhooks} helps to handle webhook events received from Git\+Hub.

\href{https://docs.github.com/webhooks/}{\texttt{ Git\+Hub webhooks}} can be registered in multiple ways


\begin{DoxyEnumerate}
\item In repository or organization settings on \href{https://github.com/}{\texttt{ github.\+com}}.
\item Using the REST API for \href{https://docs.github.com/rest/reference/repos\#webhooks}{\texttt{ repositories}} or \href{https://docs.github.com/rest/reference/orgs\#webhooks/}{\texttt{ organizations}}
\item By \href{https://docs.github.com/developers/apps/creating-a-github-app}{\texttt{ creating a Git\+Hub App}}.
\end{DoxyEnumerate}

Note that while setting a secret is optional on Git\+Hub, it is required to be set in order to use {\ttfamily @octokit/webhooks}. Content Type must be set to {\ttfamily application/json}, {\ttfamily application/x-\/www-\/form-\/urlencoded} is not supported.\hypertarget{README.md_autotoc_md5385}{}\doxysubsection{\texorpdfstring{Usage}{Usage}}\label{README.md_autotoc_md5385}

Returns a promise. Runs all handlers set with \`{}webhooks.on()\`{} in parallel and waits for them to finish. If one of the handlers rejects or throws an error, then {\ttfamily webhooks.\+receive()} rejects. The returned error has an {\ttfamily .errors} property which holds an array of all errors caught from the handlers. If no errors occur, {\ttfamily webhooks.\+receive()} resolves without passing any value.

If a webhook event handler throws an error or returns a promise that rejects, an error event is triggered. You can use this handler for logging or reporting events. The passed error object has a .event property which has all information on the event.

Asynchronous {\ttfamily error} event handler are not blocking the {\ttfamily .receive()} method from completing.
