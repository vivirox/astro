\hypertarget{md_node__modules_2fast-equals_2_c_h_a_n_g_e_l_o_g_autotoc_md15003}{}\doxysubsubsection{\texorpdfstring{{\ttfamily constructor} equality now required}{{\ttfamily constructor} equality now required}}\label{md_node__modules_2fast-equals_2_c_h_a_n_g_e_l_o_g_autotoc_md15003}
To align with other implementations common in the community, but also to be more functionally correct, the two objects being compared now must have equal {\ttfamily constructor}s.\hypertarget{md_node__modules_2fast-equals_2_c_h_a_n_g_e_l_o_g_autotoc_md15004}{}\doxysubsubsection{\texorpdfstring{{\ttfamily Map} / {\ttfamily Set} comparisons no longer support IE11}{{\ttfamily Map} / {\ttfamily Set} comparisons no longer support IE11}}\label{md_node__modules_2fast-equals_2_c_h_a_n_g_e_l_o_g_autotoc_md15004}
In previous verisons, {\ttfamily .for\+Each()} was used to ensure that support for {\ttfamily Symbol} was not required, as IE11 did not have {\ttfamily Symbol} and therefore both {\ttfamily Map} and {\ttfamily Set} did not have iterator-\/based methods such as {\ttfamily .values()} or {\ttfamily .entries()}. Since IE11 is no longer a supported browser, and support for those methods is present in all browsers and Node for quite a while, the comparison has moved to use these methods. This results in a \texorpdfstring{$\sim$}{\string~}20\% performance increase.\hypertarget{md_node__modules_2fast-equals_2_c_h_a_n_g_e_l_o_g_autotoc_md15005}{}\doxysubsubsection{\texorpdfstring{{\ttfamily create\+Custom\+Equal} contract has changed}{{\ttfamily create\+Custom\+Equal} contract has changed}}\label{md_node__modules_2fast-equals_2_c_h_a_n_g_e_l_o_g_autotoc_md15005}
To better facilitate strict comparisons, but also to allow for {\ttfamily meta} use separate from caching, the contract for {\ttfamily create\+Custom\+Equal} has changed. See the \href{./README.md\#createcustomequal}{\texttt{ README documentation}} for more details, but froma high-\/level\+:


\begin{DoxyItemize}
\item {\ttfamily meta} is no longer passed through to equality comparators, but rather a general {\ttfamily state} object which contains {\ttfamily meta}
\item {\ttfamily cache} now also lives on the {\ttfamily state} object, which allows for use of the {\ttfamily meta} property separate from but in parallel with the circular cache
\item {\ttfamily equals} is now on {\ttfamily state}, which prevents the need to pass through the separate {\ttfamily is\+Equal} method for the equality comparator
\end{DoxyItemize}\hypertarget{md_node__modules_2fast-equals_2_c_h_a_n_g_e_l_o_g_autotoc_md15006}{}\doxysubsubsection{\texorpdfstring{{\ttfamily create\+Custom\+Circular\+Equal} has been removed}{{\ttfamily create\+Custom\+Circular\+Equal} has been removed}}\label{md_node__modules_2fast-equals_2_c_h_a_n_g_e_l_o_g_autotoc_md15006}
You can create a custom circular equality comparator through {\ttfamily create\+Custom\+Equal} now by providing {\ttfamily circular\+: true} to the options.\hypertarget{md_node__modules_2fast-equals_2_c_h_a_n_g_e_l_o_g_autotoc_md15007}{}\doxysubsubsection{\texorpdfstring{Custom {\ttfamily meta} values are no longer passed at callsite}{Custom {\ttfamily meta} values are no longer passed at callsite}}\label{md_node__modules_2fast-equals_2_c_h_a_n_g_e_l_o_g_autotoc_md15007}
To use {\ttfamily meta} properties for comparisons, they must be returned in a {\ttfamily create\+State} method.\hypertarget{md_node__modules_2fast-equals_2_c_h_a_n_g_e_l_o_g_autotoc_md15008}{}\doxysubsubsection{\texorpdfstring{Deep links have changed}{Deep links have changed}}\label{md_node__modules_2fast-equals_2_c_h_a_n_g_e_l_o_g_autotoc_md15008}
If you were deep-\/linking into a specific asset type (ESM / CJS / UMD), they have changed location.

{\bfseries{NOTE}}\+: You may no longer need to deep-\/link, as the build resolution has improved.\hypertarget{md_node__modules_2fast-equals_2_c_h_a_n_g_e_l_o_g_autotoc_md15009}{}\doxysubsection{\texorpdfstring{Enhancements}{Enhancements}}\label{md_node__modules_2fast-equals_2_c_h_a_n_g_e_l_o_g_autotoc_md15009}
\hypertarget{md_node__modules_2fast-equals_2_c_h_a_n_g_e_l_o_g_autotoc_md15010}{}\doxysubsubsection{\texorpdfstring{New "{}strict"{} comparators available}{New "{}strict"{} comparators available}}\label{md_node__modules_2fast-equals_2_c_h_a_n_g_e_l_o_g_autotoc_md15010}
The following new comparators are available\+:


\begin{DoxyItemize}
\item {\ttfamily strict\+Deep\+Equal}
\item {\ttfamily strict\+Shallow\+Equal}
\item {\ttfamily strict\+Circular\+Deep\+Equal}
\item {\ttfamily strict\+Circular\+Shallow\+Equal}

This will perform the same comparisons as their non-\/strict counterparts, but will verify additional properties (non-\/enumerable properties on objects, keyed objects on {\ttfamily Array} / {\ttfamily Map} / {\ttfamily Set}) and that the descriptors for the properties align.\hypertarget{md_node__modules_2fast-equals_2_c_h_a_n_g_e_l_o_g_autotoc_md15011}{}\doxysubsubsection{\texorpdfstring{{\ttfamily Typed\+Array} support}{{\ttfamily Typed\+Array} support}}\label{md_node__modules_2fast-equals_2_c_h_a_n_g_e_l_o_g_autotoc_md15011}
Support for comparing all typed array values is now supported, and you can provide a custom comparator via the new {\ttfamily are\+Typed\+Arrays\+Equal} option in the {\ttfamily create\+Custom\+Equal} configuration.\hypertarget{md_node__modules_2fast-equals_2_c_h_a_n_g_e_l_o_g_autotoc_md15012}{}\doxysubsubsection{\texorpdfstring{Better build system resolution}{Better build system resolution}}\label{md_node__modules_2fast-equals_2_c_h_a_n_g_e_l_o_g_autotoc_md15012}
