Programmatic access to the \href{https://www.w3.org/TR/wai-aria-1.2/\#roles}{\texttt{ WAI-\/\+ARIA 1.\+2 Roles Model}}. This package tracks the W3C Recommendation (last update\+: 6 June 2023).

The files under {\ttfamily src/etc} are generated by the {\ttfamily break\+Up\+Aria\+JSON} script.

To change them, edit the file {\ttfamily scripts/roles.\+json} then run\+:

The map of elements to roles is keyed by an HTML concept. An HTML concept corresponds to the {\ttfamily base\+Concepts} and {\ttfamily related\+Concepts} of an ARIA role. Concepts exist in the context of a {\ttfamily module}\+: HTML, XForms, Dublin Core, for example. The concept representation is an object literal with a name property (the element name) and an optional attributes array.

The roles are provided in a \href{https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Set}{\texttt{ Set}}.\hypertarget{README.md_autotoc_md6214}{}\doxysubsubsection{\texorpdfstring{Role to element}{Role to element}}\label{README.md_autotoc_md6214}

ARIA roles are mapped to the HTML Elements with the same inherent role. Some roles, such as {\ttfamily columnheader} are only mapped to an HTML element that expresses specific attributes. In the case of {\ttfamily \texorpdfstring{$<$}{<}input\texorpdfstring{$>$}{>}}, the element often requires a {\ttfamily type} attribute to map to an ARIA role.
