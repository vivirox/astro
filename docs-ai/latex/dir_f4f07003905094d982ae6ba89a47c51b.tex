\doxysubsection{Detailed Description}
\hypertarget{README.md_autotoc_md9329}{}\doxysubsubsection{\texorpdfstring{Helper module for loading your native module\textquotesingle{}s {\ttfamily .node} file}{Helper module for loading your native module\textquotesingle{}s {\ttfamily .node} file}}\label{README.md_autotoc_md9329}
This is a helper module for authors of Node.\+js native addon modules. It is basically the "{}swiss army knife"{} of {\ttfamily require()}ing your native module\textquotesingle{}s {\ttfamily .node} file.

Throughout the course of Node\textquotesingle{}s native addon history, addons have ended up being compiled in a variety of different places, depending on which build tool and which version of node was used. To make matters worse, now the {\ttfamily gyp} build tool can produce either a {\bfseries{Release}} or {\bfseries{Debug}} build, each being built into different locations.

This module checks {\itshape all} the possible locations that a native addon would be built at, and returns the first one that loads successfully.\hypertarget{README.md_autotoc_md9330}{}\doxysubsection{\texorpdfstring{Installation}{Installation}}\label{README.md_autotoc_md9330}
Install with {\ttfamily npm}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\ bash}
\DoxyCodeLine{\$\ npm\ install\ -\/-\/save\ bindings}

\end{DoxyCode}


Or add it to the {\ttfamily "{}dependencies"{}} section of your {\ttfamily package.\+json} file.\hypertarget{README.md_autotoc_md9331}{}\doxysubsection{\texorpdfstring{Example}{Example}}\label{README.md_autotoc_md9331}
