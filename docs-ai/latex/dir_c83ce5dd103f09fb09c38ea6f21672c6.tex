This package provides a virtual file format. It exposes an API to access the file value, path, metadata about the file, and specifically supports attaching lint messages and errors to certain places in these files.\hypertarget{readme.md_autotoc_md555}{}\doxysubsubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{readme.md_autotoc_md555}
The virtual file format is useful when dealing with the concept of files in places where you might not be able to access the file system. The message API is particularly useful when making things that check files (as in, linting).

vfile is made for \href{https://github.com/unifiedjs/unified}{\texttt{ unified}}, which amongst other things checks files. However, vfile can be used in other projects that deal with parsing, transforming, and serializing data, to build linters, compilers, static site generators, and other build tools.

This is different from the excellent \href{https://github.com/gulpjs/vinyl}{\texttt{ {\ttfamily vinyl}}} in that vfile has a smaller API, a smaller size, and focuses on messages.\hypertarget{readme.md_autotoc_md556}{}\doxysubsubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md556}
