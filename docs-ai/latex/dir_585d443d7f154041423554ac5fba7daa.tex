By default it will only looks for {\ttfamily .gitignore} but NOT {\ttfamily .eslintignore}, as we would recommended move away from {\ttfamily .eslintignore} to declare directly in {\ttfamily eslint.\+config.\+js} for single source of truth. If you still want it, you can pass the {\ttfamily files} option to specify the files to look for.

By default, this plugin throws if any of the ignore files are missing. This can be disabled by passing setting the {\ttfamily strict} option to {\ttfamily false}.

By default, this plugin will try to look up the directory tree and match the first {\ttfamily .gitignore} file. You can disable this by setting the {\ttfamily root} option to {\ttfamily true}, or specify the {\ttfamily files} option to a specific path.

This plugin will also automatically detect git submodule file {\ttfamily .gitmodules} and ignore all files under the submodule dirs. If you want to disable or customize this behavior, you can pass the {\ttfamily files\+Git\+Modules} option.
