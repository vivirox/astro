\cellcolor{\tableheadbgcolor}\textbf{ 

Browsers

}&

{\ttfamily @octokit/oauth-\/methods} is not meant for browser usage.

Some of the methods will work, but others do not have CORS headers enabled and will fail ({\ttfamily exchange\+Web\+Flow\+Code()}, {\ttfamily create\+Device\+Code()}, {\ttfamily exchange\+Device\+Code()}, {\ttfamily refresh\+Token()}). Also the Client Secret should not be exposed to clients as it can be used for a \href{https://en.wikipedia.org/wiki/Person-in-the-middle_attack}{\texttt{ Person-\/in-\/the-\/middle attack}}.

After a user granted access to an OAuth App or Git\+Hub App on \href{https://docs.github.com/en/developers/apps/identifying-and-authorizing-users-for-github-apps\#1-request-a-users-github-identity}{\texttt{ Step 1 of Git\+Hub\textquotesingle{}s OAuth Web Flow}}, they get redirected to a URL controlled by your app with a {\ttfamily ?code=...} query parameter.

You can exchange that code for a user access token as described in \href{https://docs.github.com/en/developers/apps/authorizing-oauth-apps\#2-users-are-redirected-back-to-your-site-by-github}{\texttt{ Step 2 of Git\+Hub\textquotesingle{}s OAuth Web Flow}}.

Setting {\ttfamily client\+Type} is required because there are slight differences between {\ttfamily "{}oauth-\/app"{}} and {\ttfamily "{}github-\/app"{}}. Most importantly, Git\+Hub Apps do not support scopes.
