Enforce that all elements that require alternative text have meaningful information to relay back to the end user. This is a critical component of accessibility for screen reader users in order for them to understand the content\textquotesingle{}s purpose on the page. By default, this rule checks for alternative text on the following elements\+: {\ttfamily \texorpdfstring{$<$}{<}img\texorpdfstring{$>$}{>}}, {\ttfamily \texorpdfstring{$<$}{<}area\texorpdfstring{$>$}{>}}, {\ttfamily \texorpdfstring{$<$}{<}input type="{}image"{}\texorpdfstring{$>$}{>}}, and {\ttfamily \texorpdfstring{$<$}{<}object\texorpdfstring{$>$}{>}}.\hypertarget{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2alt-text_autotoc_md13714}{}\doxysection{\texorpdfstring{How to resolve}{How to resolve}}\label{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2alt-text_autotoc_md13714}
\hypertarget{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2alt-text_autotoc_md13715}{}\doxysubsection{\texorpdfstring{{\ttfamily \texorpdfstring{$<$}{<}img\texorpdfstring{$>$}{>}}}{{\ttfamily \texorpdfstring{$<$}{<}img\texorpdfstring{$>$}{>}}}}\label{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2alt-text_autotoc_md13715}
An {\ttfamily \texorpdfstring{$<$}{<}img\texorpdfstring{$>$}{>}} must have the {\ttfamily alt} prop set with meaningful text or as an empty string to indicate that it is an image for decoration.

For images that are being used as icons for a button or control, the {\ttfamily alt} prop should be set to an empty string ({\ttfamily alt="{}"{}}).

The content of an {\ttfamily alt} attribute is used to calculate the accessible label of an element, whereas the text content is used to produce a label for the element. For this reason, adding a label to an icon can produce a confusing or duplicated label on a control that already has appropriate text content.\hypertarget{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2alt-text_autotoc_md13716}{}\doxysubsection{\texorpdfstring{{\ttfamily \texorpdfstring{$<$}{<}object\texorpdfstring{$>$}{>}}}{{\ttfamily \texorpdfstring{$<$}{<}object\texorpdfstring{$>$}{>}}}}\label{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2alt-text_autotoc_md13716}
Add alternative text to all embedded {\ttfamily \texorpdfstring{$<$}{<}object\texorpdfstring{$>$}{>}} elements using either inner text, setting the {\ttfamily title} prop, or using the {\ttfamily aria-\/label} or {\ttfamily aria-\/labelledby} props.\hypertarget{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2alt-text_autotoc_md13717}{}\doxysubsection{\texorpdfstring{{\ttfamily \texorpdfstring{$<$}{<}input type="{}image"{}\texorpdfstring{$>$}{>}}}{{\ttfamily \texorpdfstring{$<$}{<}input type="{}image"{}\texorpdfstring{$>$}{>}}}}\label{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2alt-text_autotoc_md13717}
All {\ttfamily \texorpdfstring{$<$}{<}input type="{}image"{}\texorpdfstring{$>$}{>}} elements must have a non-\/empty {\ttfamily alt} prop set with a meaningful description of the image or have the {\ttfamily aria-\/label} or {\ttfamily aria-\/labelledby} props set.\hypertarget{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2alt-text_autotoc_md13718}{}\doxysubsection{\texorpdfstring{{\ttfamily \texorpdfstring{$<$}{<}area\texorpdfstring{$>$}{>}}}{{\ttfamily \texorpdfstring{$<$}{<}area\texorpdfstring{$>$}{>}}}}\label{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2alt-text_autotoc_md13718}
All clickable {\ttfamily \texorpdfstring{$<$}{<}area\texorpdfstring{$>$}{>}} elements within an image map have an {\ttfamily alt}, {\ttfamily aria-\/label} or {\ttfamily aria-\/labelledby} prop that describes the purpose of the link.\hypertarget{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2alt-text_autotoc_md13719}{}\doxysection{\texorpdfstring{Rule options}{Rule options}}\label{md_node__modules_2eslint-plugin-jsx-a11y_2docs_2rules_2alt-text_autotoc_md13719}

The {\ttfamily elements} option is a whitelist for DOM elements to check for alternative text. If an element is removed from the default set of elements (noted above), any custom components for that component will also be ignored. In order to indicate any custom wrapper components that should be checked, you can map the DOM element to an array of JSX custom components. This is a good use case when you have a wrapper component that simply renders an {\ttfamily img} element, for instance (like in React)\+:
