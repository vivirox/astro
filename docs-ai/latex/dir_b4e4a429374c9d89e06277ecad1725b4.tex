\doxysubsection{Detailed Description}
\begin{quote}
Map over promises concurrently \end{quote}


Useful when you need to run promise-\/returning \& async functions multiple times with different inputs concurrently.

This is different from {\ttfamily Promise.\+all()} in that you can control the concurrency and also decide whether or not to stop iterating when there\textquotesingle{}s an error.\hypertarget{readme.md_autotoc_md25377}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md25377}

Type\+: {\ttfamily Async\+Iterable\texorpdfstring{$<$}{<}Promise\texorpdfstring{$<$}{<}unknown\texorpdfstring{$>$}{>} \texorpdfstring{$\vert$}{|} unknown\texorpdfstring{$>$}{>} \texorpdfstring{$\vert$}{|} Iterable\texorpdfstring{$<$}{<}Promise\texorpdfstring{$<$}{<}unknown\texorpdfstring{$>$}{>} \texorpdfstring{$\vert$}{|} unknown\texorpdfstring{$>$}{>}}

Synchronous or asynchronous iterable that is iterated over concurrently, calling the {\ttfamily mapper} function for each element. Each iterated item is {\ttfamily await}\textquotesingle{}d before the {\ttfamily mapper} is invoked so the iterable may return a {\ttfamily Promise} that resolves to an item.

Asynchronous iterables (different from synchronous iterables that return {\ttfamily Promise} that resolves to an item) can be used when the next item may not be ready without waiting for an asynchronous process to complete and/or the end of the iterable may be reached after the asynchronous process completes. For example, reading from a remote queue when the queue has reached empty, or reading lines from a stream.\hypertarget{readme.md_autotoc_md25383}{}\doxysubsubsubsection{\texorpdfstring{mapper(element, index)}{mapper(element, index)}}\label{readme.md_autotoc_md25383}

Number of concurrently pending promises returned by {\ttfamily mapper}.\hypertarget{readme.md_autotoc_md25386}{}\doxysubsubsubsubsection{\texorpdfstring{backpressure}{backpressure}}\label{readme.md_autotoc_md25386}
{\bfseries{Only for {\ttfamily p\+Map\+Iterable}}}

Type\+: {\ttfamily number} {\itshape (Integer)}\textbackslash{} Default\+: {\ttfamily options.\+concurrency}\textbackslash{} Minimum\+: {\ttfamily options.\+concurrency}

Maximum number of promises returned by {\ttfamily mapper} that have resolved but not yet collected by the consumer of the async iterable. Calls to {\ttfamily mapper} will be limited so that there is never too much backpressure.

Useful whenever you are consuming the iterable slower than what the mapper function can produce concurrently. For example, to avoid making an overwhelming number of HTTP requests if you are saving each of the results to a database.\hypertarget{readme.md_autotoc_md25387}{}\doxysubsubsubsubsection{\texorpdfstring{stop\+On\+Error}{stop\+On\+Error}}\label{readme.md_autotoc_md25387}
{\bfseries{Only for {\ttfamily p\+Map}}}

Type\+: {\ttfamily boolean}\textbackslash{} Default\+: {\ttfamily true}

When {\ttfamily true}, the first mapper rejection will be rejected back to the consumer.

When {\ttfamily false}, instead of stopping when a promise rejects, it will wait for all the promises to settle and then reject with an \href{https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/AggregateError}{\texttt{ {\ttfamily Aggregate\+Error}}} containing all the errors from the rejected promises.

Caveat\+: When {\ttfamily true}, any already-\/started async mappers will continue to run until they resolve or reject. In the case of infinite concurrency with sync iterables, {\itshape all} mappers are invoked on startup and will continue after the first rejection. \href{https://github.com/sindresorhus/p-map/issues/51}{\texttt{ Issue \#51}} can be implemented for abort control.\hypertarget{readme.md_autotoc_md25388}{}\doxysubsubsubsubsection{\texorpdfstring{signal}{signal}}\label{readme.md_autotoc_md25388}
{\bfseries{Only for {\ttfamily p\+Map}}}

Type\+: \href{https://developer.mozilla.org/en-US/docs/Web/API/AbortSignal}{\texttt{ {\ttfamily Abort\+Signal}}}

You can abort the promises using \href{https://developer.mozilla.org/en-US/docs/Web/API/AbortController}{\texttt{ {\ttfamily Abort\+Controller}}}.
