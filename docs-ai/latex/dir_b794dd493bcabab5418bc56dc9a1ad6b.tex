This package is a \href{https://github.com/unifiedjs/unified}{\texttt{ unified}} (\href{https://github.com/rehypejs/rehype}{\texttt{ rehype}}) plugin that defines how to take a syntax tree as input and turn it into serialized HTML. When it‚Äôs used, HTML is serialized as the final result.

See \href{https://github.com/rehypejs/rehype}{\texttt{ the monorepo readme}} for info on what the rehype ecosystem is.\hypertarget{readme.md_autotoc_md28951}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{readme.md_autotoc_md28951}
This plugin adds support to unified for serializing HTML. If you also need to parse HTML, you can alternatively use \href{../rehype/}{\texttt{ {\ttfamily rehype}}}, which combines unified, \href{../rehype-parse/}{\texttt{ {\ttfamily rehype-\/parse}}}, and this plugin.

When you are in a browser, trust your content, don‚Äôt need formatting options, and value a smaller bundle size, you can use \href{https://github.com/rehypejs/rehype-dom/tree/main/packages/rehype-dom-stringify}{\texttt{ {\ttfamily rehype-\/dom-\/stringify}}} instead.

If you don‚Äôt use plugins and have access to a syntax tree, you can directly use \href{https://github.com/syntax-tree/hast-util-to-html}{\texttt{ {\ttfamily hast-\/util-\/to-\/html}}}, which is used inside this plugin. rehype focusses on making it easier to transform content by abstracting such internals away.

A different plugin, \href{https://github.com/rehypejs/rehype-format}{\texttt{ {\ttfamily rehype-\/format}}}, improves the readability of HTML source code as it adds insignificant but pretty whitespace between elements. There is also the preset \href{https://github.com/rehypejs/rehype-minify}{\texttt{ {\ttfamily rehype-\/minify}}} for when you want the inverse\+: minified and mangled HTML.\hypertarget{readme.md_autotoc_md28952}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md28952}
