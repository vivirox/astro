{\bfseries{Description}}\+: Do not strip backslashes that were used for escaping from the result.\hypertarget{README.md_autotoc_md9579}{}\doxysubsection{\texorpdfstring{What is "{}brace expansion"{}?}{What is "{}brace expansion"{}?}}\label{README.md_autotoc_md9579}
Brace expansion is a type of parameter expansion that was made popular by unix shells for generating lists of strings, as well as regex-\/like matching when used alongside wildcards (globs).

In addition to "{}expansion"{}, braces are also used for matching. In other words\+:


\begin{DoxyItemize}
\item brace expansion is for generating new lists
\item brace matching is for filtering existing lists
\end{DoxyItemize}



{\bfseries{{\bfseries{More about brace expansion}} (click to expand)}}\begin{adjustwidth}{1em}{0em}


There are two main types of brace expansion\+:


\begin{DoxyEnumerate}
\item {\bfseries{lists}}\+: which are defined using comma-\/separated values inside curly braces\+: {\ttfamily \{a,b,c\}}
\item {\bfseries{sequences}}\+: which are defined using a starting value and an ending value, separated by two dots\+: {\ttfamily a\{1..3\}b}. Optionally, a third argument may be passed to define a "{}step"{} or increment to use\+: {\ttfamily a\{1..100..10\}b}. These are also sometimes referred to as "{}ranges"{}.
\end{DoxyEnumerate}

Here are some example brace patterns to illustrate how they work\+:

{\bfseries{Sets}}


\begin{DoxyCode}{0}
\DoxyCodeLine{\{a,b,c\}\ \ \ \ \ \ \ =>\ a\ b\ c}
\DoxyCodeLine{\{a,b,c\}\{1,2\}\ \ =>\ a1\ a2\ b1\ b2\ c1\ c2}

\end{DoxyCode}


{\bfseries{Sequences}}


\begin{DoxyCode}{0}
\DoxyCodeLine{\{1..9\}\ \ \ \ \ \ \ \ =>\ 1\ 2\ 3\ 4\ 5\ 6\ 7\ 8\ 9}
\DoxyCodeLine{\{4..-\/4\}\ \ \ \ \ \ \ =>\ 4\ 3\ 2\ 1\ 0\ -\/1\ -\/2\ -\/3\ -\/4}
\DoxyCodeLine{\{1..20..3\}\ \ \ \ =>\ 1\ 4\ 7\ 10\ 13\ 16\ 19}

\DoxyCodeLine{\{a..z..3\}\ \ \ \ \ =>\ a\ d\ g\ j\ m\ p\ s\ v\ y}

Sets and sequences can be mixed together or used along with any other strings.

\DoxyCodeLine{\{a,b,c\}\{1..3\}\ \ \ =>\ a1\ a2\ a3\ b1\ b2\ b3\ c1\ c2\ c3}

\hypertarget{README.md_autotoc_md9581}{}\doxysubsection{\texorpdfstring{Brace matching pitfalls}{Brace matching pitfalls}}\label{README.md_autotoc_md9581}
Although brace patterns offer a user-\/friendly way of matching ranges or sets of strings, there are also some major disadvantages and potential risks you should be aware of.\hypertarget{README.md_autotoc_md9582}{}\doxysubsubsection{\texorpdfstring{tldr}{tldr}}\label{README.md_autotoc_md9582}
{\bfseries{"{}brace bombs"{}}}


\begin{DoxyItemize}
\item brace expansion can eat up a huge amount of processing resources
\item as brace patterns increase {\itshape linearly in size}, the system resources required to expand the pattern increase exponentially
\item users can accidentally (or intentially) exhaust your system\textquotesingle{}s resources resulting in the equivalent of a DoS attack (bonus\+: no programming knowledge is required!)
\end{DoxyItemize}

For a more detailed explanation with examples, see the geometric complexity section.\hypertarget{README.md_autotoc_md9583}{}\doxysubsubsection{\texorpdfstring{The solution}{The solution}}\label{README.md_autotoc_md9583}
Jump to the performance section to see how Braces solves this problem in comparison to other libraries.\hypertarget{README.md_autotoc_md9584}{}\doxysubsubsection{\texorpdfstring{Geometric complexity}{Geometric complexity}}\label{README.md_autotoc_md9584}
At minimum, brace patterns with sets limited to two elements have quadradic or {\ttfamily O(n\texorpdfstring{$^\wedge$}{\string^}2)} complexity. But the complexity of the algorithm increases exponentially as the number of sets, {\itshape and elements per set}, increases, which is {\ttfamily O(n\texorpdfstring{$^\wedge$}{\string^}c)}.

For example, the following sets demonstrate quadratic ({\ttfamily O(n\texorpdfstring{$^\wedge$}{\string^}2)}) complexity\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\{1,2\}\{3,4\}\ \ \ \ \ \ =>\ (2X2)\ \ \ \ =>\ 13\ 14\ 23\ 24}
\DoxyCodeLine{\{1,2\}\{3,4\}\{5,6\}\ =>\ (2X2X2)\ \ =>\ 135\ 136\ 145\ 146\ 235\ 236\ 245\ 246}

\end{DoxyCode}


But add an element to a set, and we get a n-\/fold Cartesian product with {\ttfamily O(n\texorpdfstring{$^\wedge$}{\string^}c)} complexity\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\{1,2,3\}\{4,5,6\}\{7,8,9\}\ =>\ (3X3X3)\ =>\ 147\ 148\ 149\ 157\ 158\ 159\ 167\ 168\ 169\ 247\ 248}
\DoxyCodeLine{\ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ 249\ 257\ 258\ 259\ 267\ 268\ 269\ 347\ 348\ 349\ 357}
\DoxyCodeLine{\ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ \ 358\ 359\ 367\ 368\ 369}

\end{DoxyCode}


Now, imagine how this complexity grows given that each element is a n-\/tuple\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\{1..100\}\{1..100\}\ \ \ \ \ \ \ \ \ =>\ (100X100)\ \ \ \ \ =>\ 10,000\ elements\ (38.4\ kB)}
\DoxyCodeLine{\{1..100\}\{1..100\}\{1..100\}\ =>\ (100X100X100)\ =>\ 1,000,000\ elements\ (5.76\ MB)}

\end{DoxyCode}


Although these examples are clearly contrived, they demonstrate how brace patterns can quickly grow out of control.

Braces is not only screaming fast, it\textquotesingle{}s also more accurate the other brace expansion libraries.\hypertarget{README.md_autotoc_md9586}{}\doxysubsubsection{\texorpdfstring{Better algorithms}{Better algorithms}}\label{README.md_autotoc_md9586}
Fortunately there is a solution to the "{}brace bomb"{} problem\+: {\itshape don\textquotesingle{}t expand brace patterns into an array when they\textquotesingle{}re used for matching}.

Instead, convert the pattern into an optimized regular expression. This is easier said than done, and braces is the only library that does this currently.
