This package contains two extensions that add support for GFM syntax in markdown to \href{https://github.com/syntax-tree/mdast}{\texttt{ mdast}}\+: autolink literals ({\ttfamily www.\+x.\+com}), footnotes ({\ttfamily \mbox{[}\texorpdfstring{$^\wedge$}{\string^}1\mbox{]}}), strikethrough ({\ttfamily \texorpdfstring{$\sim$}{\string~}\texorpdfstring{$\sim$}{\string~}stuff\texorpdfstring{$\sim$}{\string~}\texorpdfstring{$\sim$}{\string~}}), tables ({\ttfamily \texorpdfstring{$\vert$}{|} cell \texorpdfstring{$\vert$}{|}…}), and tasklists ({\ttfamily \texorpdfstring{$\ast$}{*} \mbox{[}x\mbox{]}}). These extensions plug into \href{https://github.com/syntax-tree/mdast-util-from-markdown}{\texttt{ {\ttfamily mdast-\/util-\/from-\/markdown}}} (to support parsing GFM in markdown into a syntax tree) and \href{https://github.com/syntax-tree/mdast-util-to-markdown}{\texttt{ {\ttfamily mdast-\/util-\/to-\/markdown}}} (to support serializing GFM in syntax trees to markdown).\hypertarget{readme.md_autotoc_md22508}{}\doxysubsection{\texorpdfstring{When to use this}{When to use this}}\label{readme.md_autotoc_md22508}
This project is useful when you want to support the same features that Git\+Hub does in files in a repo, Gists, and several other places. Users frequently believe that some of these extensions, specifically autolink literals and tables, are part of normal markdown, so using {\ttfamily mdast-\/util-\/gfm} will help match your implementation to their understanding of markdown. There are several edge cases where Git\+Hub’s implementation works in unexpected ways or even different than described in their spec, so {\itshape writing} in GFM is not always the best choice.

You can use these extensions when you are working with {\ttfamily mdast-\/util-\/from-\/markdown} and {\ttfamily mdast-\/util-\/to-\/markdown} already.

When working with {\ttfamily mdast-\/util-\/from-\/markdown}, you must combine this package with \href{https://github.com/micromark/micromark-extension-gfm}{\texttt{ {\ttfamily micromark-\/extension-\/gfm}}}.

Instead of this package, you can also use the extensions separately\+:


\begin{DoxyItemize}
\item \href{https://github.com/syntax-tree/mdast-util-gfm-autolink-literal}{\texttt{ {\ttfamily mdast-\/util-\/gfm-\/autolink-\/literal}}} — support GFM autolink literals
\item \href{https://github.com/syntax-tree/mdast-util-gfm-footnote}{\texttt{ {\ttfamily mdast-\/util-\/gfm-\/footnote}}} — support GFM footnotes
\item \href{https://github.com/syntax-tree/mdast-util-gfm-strikethrough}{\texttt{ {\ttfamily mdast-\/util-\/gfm-\/strikethrough}}} — support GFM strikethrough
\item \href{https://github.com/syntax-tree/mdast-util-gfm-table}{\texttt{ {\ttfamily mdast-\/util-\/gfm-\/table}}} — support GFM tables
\item \href{https://github.com/syntax-tree/mdast-util-gfm-task-list-item}{\texttt{ {\ttfamily mdast-\/util-\/gfm-\/task-\/list-\/item}}} — support GFM tasklists
\end{DoxyItemize}

A different utility, \href{https://github.com/syntax-tree/mdast-util-frontmatter}{\texttt{ {\ttfamily mdast-\/util-\/frontmatter}}}, adds support for frontmatter. Git\+Hub supports YAML frontmatter for files in repos and Gists but they don’t treat it as part of GFM.

All these packages are used in \href{https://github.com/remarkjs/remark-gfm}{\texttt{ {\ttfamily remark-\/gfm}}}, which focusses on making it easier to transform content by abstracting these internals away.

This utility does not handle how markdown is turned to HTML. That’s done by \href{https://github.com/syntax-tree/mdast-util-to-hast}{\texttt{ {\ttfamily mdast-\/util-\/to-\/hast}}}. If your content is not in English, you should configure that utility.\hypertarget{readme.md_autotoc_md22509}{}\doxysubsection{\texorpdfstring{Install}{Install}}\label{readme.md_autotoc_md22509}
