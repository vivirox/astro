A \href{https://github.com/rehypejs/rehype}{\texttt{ rehype}} plugin for processing and rendering blockquote-\/based callouts.\hypertarget{README.md_autotoc_md28727}{}\doxysubsection{\texorpdfstring{What is this?}{What is this?}}\label{README.md_autotoc_md28727}
This plugin adds support for callouts (admonitions/alerts), allowing you to use \href{https://help.obsidian.md/Editing+and+formatting/Callouts}{\texttt{ Obsidian\textquotesingle{}s callout syntax}} to achieve the following features\+:


\begin{DoxyItemize}
\item Includes default callout types for various themes.
\item Supports collapsible callouts with {\ttfamily -\//+} and nestable callouts.
\item Optionally import stylesheets for corresponding themes.
\item Allows custom titles with markdown syntax.
\item Customizable default callout types.
\item Configurable new callout types.
\item Configurable aliases for callout types.
\item Configurable icon display.
\item Configurable element attributes.
\end{DoxyItemize}\hypertarget{README.md_autotoc_md28728}{}\doxysubsection{\texorpdfstring{When should I use this?}{When should I use this?}}\label{README.md_autotoc_md28728}
This plugin helps render markdown callouts, ideal for blogs built with frameworks like Astro or Next.\+js. It processes HTML directly without needing {\ttfamily allow\+Dangerous\+Html} in \href{https://github.com/remarkjs/remark-rehype}{\texttt{ remark-\/rehype}} and supports collapsible callouts with the {\ttfamily details} tag, all without Java\+Script.\hypertarget{README.md_autotoc_md28729}{}\doxysubsection{\texorpdfstring{Installation}{Installation}}\label{README.md_autotoc_md28729}
